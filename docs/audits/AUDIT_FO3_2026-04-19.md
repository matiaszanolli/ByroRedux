# Fallout 3 Compatibility Audit — 2026-04-19

Audit of ByroRedux readiness for Fallout 3 content across six dimensions:
NIF parser correctness, BSA archive reader, ESM record coverage, rendering
path, real-data validation, and end-to-end CLI plumbing.

Game context: NIF v20.2.0.7 (BSVER 34), BSA v104, ESM shared with FNV.
Parse-rate baseline 10989/10989 (100%). Reference data from
`/mnt/data/SteamLibrary/steamapps/common/Fallout 3 goty/Data/`.

---

## Executive Summary

**Compatibility level**: Tier-1 candidate — static interior cell geometry
loads correctly end-to-end; parser is stable across all 10,989 vanilla NIFs
at 100%; all six vanilla BSAs (64,361 files) open and extract cleanly;
Fallout3.esm parses to 18,007 structured records in 962 ms.

**Delta vs FNV baseline**:
- *Same surface*: BSA code path, CELL/XCLL/XCWT/XCLR/XESP layout, NIF
  block vocabulary, shader-property parse (PPLighting/NoLighting/TextureSet),
  skinning chain (NiSkinData/NiSkinPartition/BSDismemberSkinInstance),
  interior lighting template path, CLI (`--esm`/`--cell`/`--bsa`).
- *FO3-specific gaps*: (1) GameKind misclassification from HEDR=0.94 routing
  to Fallout4; (2) CREA group unparsed — FO3's dominant bestiary record
  (FNV migrated combat to NPC_, masking the gap); (3) LVLC unparsed — FO3
  actor-spawn lists; (4) SCPT pre-Papyrus bytecode unparsed; (5) FaceGen
  head `NiTriShapeData` over-reads 4,024 B and silently drops geometry,
  making every FO3 NPC face invisible; (6) renderer flag-bit vocabulary
  collision on BSShaderPPLighting double-sided detection; (7) exterior
  worldspace auto-pick excludes FO3's `Wasteland`.

**Critical path to ship FO3**: fix FO3-3-01 (HEDR band, one-line), FO3-5-01
(tangent mask, one-line), FO3-REN-H1 (flag bit per-game dispatch,
small refactor), then FO3-3-02/06 (CREA/LVLC dispatch). Everything else
is cosmetic or DLC-blocking only.

## Dimension Findings

### Dim 1 — NIF v20.2.0.7 Parser Correctness

**Verified**: `BSShaderPPLightingProperty` field completeness (refraction,
parallax, bump map tiling captured with bsver-gated reads + regression tests
at `crates/nif/src/blocks/shader.rs:1184-1213`); `BSShaderTextureSet` stores
raw Vec<String>; normal slot 1 correctly routed by the importer;
`BSSegmentedTriShape` dispatches through `NiTriShape::parse_segmented` with
regression test for #146 (`blocks/tri_shape.rs:1359-1390`); legacy
NiParticleSystem stack reachable per ROADMAP N23 closeout.

**Medium — FO3-NIF-M1** `crates/nif/src/blocks/properties.rs:287-293`.
`NiTexturingProperty` parallax slot 7 is read via `Self::read_tex_desc(stream)`
and the returned `TexDesc` is immediately dropped — there is no
`parallax_texture` field on the struct. `_parallax_offset` at line 291 also
discarded. FO3 meshes that keep NiTexturingProperty alongside PPLighting lose
their parallax height map. Fix: add `parallax_texture: Option<TexDesc>` +
`parallax_offset: f32`, wire through `import/material.rs`.

**Medium — FO3-NIF-M2** `crates/nif/src/import/material.rs:653-656`.
`BSShaderNoLightingProperty` path reads `file_name` → `info.texture_path`
but silently drops the four falloff fields that the parser captures at
`shader.rs:102-150`. FO3 HUD/VATS/scope overlays lose soft-alpha cone.
Fix: extend `MaterialInfo` with the same falloff fields already mirrored
for `BSEffectShaderProperty` (`material.rs:316-320`).

**Low — FO3-NIF-L1** `shader.rs:175`. `num_textures` read as `i32` with
`max(0) as u32` clamp; negative count becomes silently empty set. Mask for
upstream drift. Fix: use `u32` or return parse error on negative.

**Low — FO3-NIF-L2** `shader.rs:109-112`. Accessor asymmetry — `shader_flags_1()`
exists on NoLighting only, and not at all on PPLighting. Readability only.

### Dim 2 — BSA v104 Archive Reader

**Verified**: All 6 vanilla archives open, list, and extract cleanly
(64,361 files). Code path identical to verified-FNV baseline, zero
FO3-specific regressions. DDS parser covers full FO3 BC1/BC3/BC5 + RGBA.

**Medium — FO3-2-01** `crates/bsa/src/archive.rs:86-99`. `genhash_file`
folds the extension bytes on top of the stem-middle rolling hash, but
BSArch/libbsarch reference computes them independently and **adds** the
results: `high = rolling(stem[1..len-2]) + rolling(ext_full)`. Low 32 bits
match (HashMap lookup by path, not hash), but 119k debug warnings per FO3
archive open (125k on FNV Meshes.bsa). Verified via Python port:
`rolling("lov") + rolling(".nif") = 0xc86aec30` matches stored,
sequential fold gives `0xd91bd930`. The `#[cfg(debug_assertions)]`
validation check never passed for a single real file. **Pre-existing
and shared with FNV**, not FO3-specific. Fix inline:
```rust
let hash_low = (hash as u32) ^ ext_xor;
let mut hash_ext = 0u32;
for &c in ext_bytes {
    hash_ext = hash_ext.wrapping_mul(0x1003f).wrapping_add(c as u32);
}
let hash_high = ((hash >> 32) as u32).wrapping_add(hash_ext);
hash = ((hash_high as u64) << 32) | (hash_low as u64);
```
Pin regression test against a known FNV stored hash.

**Info — FO3-2-02** `.egm`/`.egt`/`.tri`/`.dtl` not in `ext_xor` privileged
table. Listing + extraction still work. Only matters if BSA-writing is
added.

**Info — FO3-2-03** No hash collisions — file counts match header
`file_count` exactly for all 6 archives.

### Dim 3 — ESM Record Coverage (Fallout3.esm)

**Verified**: Fallout3.esm (288 MB GOTY) parses in 962 ms release, zero
errors. 18,007 structured records: 421 cells, 11,450 statics, 1,762 items,
535 containers, 972 LVLI, 89 LVLN, 1,647 NPCs, 22 RACE, 53 CLAS, 326 FACT,
155 GLOB, 530 GMST, 27 WTHR, 18 CLMT.

**Critical — FO3-3-01** `crates/plugin/src/esm/reader.rs:110-137`.
Fallout3.esm ships `HEDR=0.94f32` (bytes `d7 a3 70 3f` at 0x1e, all 6 FO3
masters). Band `(0.94..=0.955)` routes to `GameKind::Fallout4`. Fallout4.esm
actually ships HEDR=1.0 — the existing band matches no real FO4 file.
Comments at lines 88/95/115 are stale. Silent today only because WEAP/ARMO/
AMMO DATA arms bucket Fallout4 with FO3NV/Oblivion; any future divergence
corrupts FO3 data. Fix: reorder bands, `(0.935..=0.945)` → Fallout3NV,
`(0.96..=1.04)` → Fallout4; add per-master regression tests.

**High — FO3-3-02** `records/mod.rs:128-204`. CREA top-level group dropped
at the catch-all skip. FO3's core bestiary (super mutants, deathclaws,
radroaches, mirelurks, robots) lives here; FNV migrated most combat to
NPC_ (3,816 NPCs vs FO3's 1,647), masking the gap. Fix: add `b"CREA"` arm
reusing `parse_npc` (overlapping sub-records), new `index.creatures`.

**High — FO3-3-03** `records/common.rs:97`, `records/mod.rs`. SCPT unparsed.
FO3/FNV predate Papyrus — scripts are SCHR+SCDA bytecode. NPC/item `SCRI`
form refs dangle. M30's .psc parser does not cover this. Fix: add `b"SCPT"`
arm extracting EDID + SCHR (5×u32) + SCDA (bytecode) + SCTX.

**Medium — FO3-3-04/05/06** Multiple top-level groups hit the catch-all:
PACK (AI packages — 30-procedure memo has producer gap), DIAL/INFO/QUST/
MESG/PERK/SPEL/MGEF (dialogue + quest + popups + effects), LVLC (FO3-dominant
spawn tables — 972 LVLI parsed, only 89 LVLN, LVLC needed for encounter
zones).

**Low — FO3-3-07** WATR/NAVI/NAVM/REGN/ECZN/LGTM/HDPT/EYES/HAIR skipped.
WATR breaks XCWT cell pointer (`cell.rs:521`). LGTM ties to #379 (cell
lighting template fallback).

**Info — FO3-3-08** CELL parser (`cell.rs:486-620`) game-agnostic. XCLL
40-byte identical to FNV; XCLR parsed (528); XCWT/XCLW collected
unconsumed; XESP unparsed. Existing issues #378/#379/#380 + FNV audit
F3-03/F3-07 apply to FO3 for free.

### Dim 4 — Rendering Path for FO3 Shaders

**Verified**: Normal map slot 1 correctly routed at
`crates/nif/src/import/material.rs:628-634` + frag shader normal sampling
at `triangle.frag:350-365`. Decal z-bias reads ECS `Decal` component
(populated from NIF flag, no heuristic) at `byroredux/src/render.rs:272`.
`find_decal_bs` (BsTriShape path, line 903) vs inline decal checks at
646-650 (FO3 path) cover the same flag trio — no bug.

**High — FO3-REN-H1** `crates/nif/src/import/material.rs:22,643,657`.
`SF_DOUBLE_SIDED = 0x1000` (bit 12) is applied indiscriminately to both
`BSShaderPPLightingProperty.shader_flags_1` (643) and
`BSShaderNoLightingProperty.shader_flags_1` (657). On FO3/FNV,
**flags1 bit 12 is `Unknown_3`** (nif.xml `Fallout3ShaderPropertyFlags1`;
flagging crashes original game). The real FO3 Double_Sided bit lives on
**flags2 bit 4** (0x10) — but on FO3 that same bit is `Refraction_Tint`,
only Skyrim/FO4 reuse it as Double_Sided. Every FO3 PPLighting mesh that
set Unknown_3 renders back-face-culled; foliage/hair/glass falls through
to NiStencilProperty at 666-672 only if exporter added one. Fix:
per-game `fn is_two_sided(bs_version, flags1, flags2)`, never
`flags1 & 0x1000`. Related to open issues #437 (GameVariant enum), #414
(FO4 flag naming) but a distinct FO3 miscompute.

**Medium — FO3-REN-M1** `material.rs:617-651`. BSShaderTextureSet slots
3 (parallax/height), 4 (env cube), 5 (env mask) never read on PPLighting.
`parallax_max_passes`/`parallax_height_scale` captured only from
`ShaderTypeData::ParallaxOcc` (BSLightingShaderProperty, Skyrim+), never
from FO3 PPLighting `shader_type=3` (Parallax_Shader_Index_15, flag1 bit
11) or `shader_type=7` (Parallax_Occlusion, flag1 bit 28). Pitt/Point
Lookout brick walls render flat; Hoover Dam concrete flat; env reflections
never bind. Related-but-distinct from open #353 (Skyrim env slot).

**Medium — FO3-REN-M2** `crates/renderer/shaders/triangle.frag:28-60`.
`GpuInstance` has no `parallaxIndex`/`parallaxHeightScale`/`parallaxMaxPasses`
and no env cube sampler binding. `MaterialInfo` at `material.rs:182-192`
parses values but they die at the renderer boundary. Grep of
`crates/renderer/shaders/` for `parallax|heightmap|envmap` returns zero
matches. Every shader_type=7 ParallaxOcc surface renders as base-color
only. Fix touches triangle.vert/frag + ui.vert per Shader Struct Sync
memory.

**Medium — FO3-REN-M3** `material.rs:660`. NoLighting decal detection
omits `ALPHA_DECAL_F2` (0x00200000). PPLighting branch at 646-650 checks
both flag1 decal bits and flag2 alpha-decal; NoLighting branch only
checks flag1. Rare on FO3 (NoLighting is mostly skybox/UI) but real for
blood-splat meshes.

**Low — FO3-REN-L1** `material.rs:19-22`. Named flag constants
module-private, inconsistent naming (`DECAL_SINGLE_PASS` vs
`SF_DOUBLE_SIDED`). Combined with #414, #437. Promote to
`crates/nif/src/shader_flags.rs` with game-prefixed aliases.

**Non-issues verified** (cross-ref dim_1): Normal slot 1 routed, decal
flag not heuristic, refraction is screen-space (frag:468-471; not P0 for
FO3).

### Dim 5 — NIF Real-Data Validation

**Verified**: 10/10 canonical FO3 probes (airtimer, deathclaw,
deathclaw_skeleton, dogskin, hairdefaultfacegenf, headfemalefacegen,
pipboyarm, scope, skeleton_male, stealthindicator) parse with 100% clean
via `target/release/examples/nif_stats`. Covers creature-skinned,
UI-NoLighting, FaceGen head, FaceGen hair, skeleton+ragdoll, HUD element.
Parse-rate baseline 10989/10989 reused from prior telemetry. Megaton
cell-load skipped (exceeds 5-min budget). The 100% parse rate masks three
silent-data-loss sites hidden behind `block_sizes` self-heal in
`lib.rs:202-213`.

**High — FO3-5-01** `crates/nif/src/blocks/tri_shape.rs:755` and
`826-867`. `headfemalefacegen.nif` block 6 NiTriShapeData over-consumes
4,024 B, fails recovery, demotes to `NiUnknown`. Root cause: `data_flags
& 0xF000 != 0` triggers tangent+bitangent skips of `array_count * 24`
bytes, but nif.xml says tangents/bitangents are present only when bit 12
specifically is set (NBT_METHOD == 0x1000). FaceGen heads set bit 13/14
for morph pointers (VAS_MATERIAL_DATA / VAS_MORPH_DATA), not NBT. Impact:
every FO3 NPC head invisible today — the ROADMAP Megaton claim covered
props, not NPCs. Fix one character: `& 0x1000 != 0`.

**Medium — FO3-5-02** `blocks/mod.rs:264`, `shader.rs:56-86`.
`TileShaderProperty` aliased to `BSShaderPPLightingProperty::parse`,
drops 20–28 B per block (`expected 82, consumed 54` × 16 in
stealthindicator.nif; `expected 74, consumed 54` × 17 in airtimer.nif).
Tile shaders carry FO3-specific `UnknownByte` + `TextureTransforms` +
per-slot uses trailer. HUD overlays (stealth, airtimer, quest markers)
render with uninitialized texture transforms — usually invisible or
mislocated. Fix: dedicated `TileShaderProperty::parse` extending
PPLighting, gated on `user_version_2 == 34`.

**Low — FO3-5-03** `collision.rs:1186-1336` + `lib.rs:204-211`. Havok
constraint stubs emit 45 `warn!` lines per skeleton NIF. Intentional
per #117 but drowns real drift during cell loads. Fix: downgrade to
`trace!` for known-stub types.

**Info — FO3-5-04** UI path clean: `pipboyarm.nif` (13× PPLighting + 6×
NoLighting + 4× NiTexturingProperty + 1× NiStencilProperty, 100% clean);
`scope.nif` (6× NoLighting + 6× NiTriShape + 3× NiAlphaProperty, 100%
clean).

**Info — FO3-5-05** Skinning path intact: `deathclaw.nif` 5× of
NiTriShape/NiTriShapeData/NiSkinData/NiSkinPartition/BSDismemberSkinInstance;
`dogskin.nif` same; `headfemalefacegen.nif` 1× of each (FO3-5-01 is in
NiTriShapeData, not skinning).

### Dim 6 — Blockers & Quirks vs FNV

**Verified**: CLI dispatch (`byroredux/src/main.rs:43-97`,
`scene.rs:40-245`) never inspects the ESM filename. `load_cell`/
`load_exterior_cells` (`cell_loader.rs:233,313`) route through shared
`parse_esm`/`parse_esm_cells`. `parse_esm` derives GameKind from HEDR
once (`records/mod.rs:110-114`) — no `FalloutNV.esm` string match anywhere.
WTHR/CLMT parsed from shared modules. Climate TNAM read raw per ESM, not
hardcoded. Grep for `Fallout3|fallout3|FO3|Capital` across
`byroredux/src/` and `crates/plugin/src/esm/` returns only doc comments
and `GameKind::Fallout3NV` — no runtime branches.

**High — FO3-6-01** `cell_loader.rs:346-360`. Preferred worldspace list
`["wastelandnv", "tamriel", "skyrim"]` excludes FO3's `Wasteland` EDID.
Falls through to `max_by_key(cell_count)` fallback that lands on Wasteland
today, but DLC masters (PointLookout, Zeta, Pitt, Anchorage) could tie or
outvote. Fix: add `"wasteland"` to preferred list in front of `"tamriel"`;
expose `--wrld <name>` override in `scene.rs:40`; prefer worldspace
containing the requested grid.

**High — FO3-6-02** `crates/plugin/src/esm/cell.rs:320-329`,
`records/mod.rs:97`, `reader.rs:175,398-401`. `read_file_header`
collects `master_files: Vec<String>`, logs count, then discards it. Every
REFR/CELL/STAT FormID stored as raw u32. Top byte is a mod-index into
containing plugin's MASTERS array. Fallout3.esm (0 masters) works by
coincidence — mod-index 0x00 == itself. Any multi-master load
(Fallout3.esm + DLC or + modded ESP) silently collides in the same
`statics: HashMap<u32, StaticObject>`. `LegacyLoadOrder::resolve`
(`legacy/mod.rs:170`) has the machinery but no caller invokes it from
the cell loader. Fix: thread `FileHeader.master_files` into
`parse_esm_cells`/`parse_esm`, store as `FormIdPair` (plugin + local ID),
normalize on insert.

**Medium — FO3-6-03** `ROADMAP.md:30-31`, `docs/engine/game-compatibility.md:60-65`.
"Megaton 42 FPS / 199 textures / 1609 entities" repeated in two docs
with identical phrasing; no checked-in bench log or CI target. 42 FPS
predates TAA (M37.5), SVGF (M37), BLAS batching (M31), RIS (M31.5).
Likely off by 2× today in either direction. Fix: add Megaton to
`--bench-frames` CI alongside sweetroll + Prospector.

**Medium — FO3-6-04** `ROADMAP.md:874`. Tier-1 table: `Fallout 3 |
Interior ✓` — no exterior column entry. FNV row says "Interior + exterior
✓". `load_exterior_cells` is game-agnostic; FO3 exterior blocked only by
FO3-6-01's hardcoded list. Fix: run one exterior load, update ROADMAP
row.

**Low — FO3-6-05** `climate.rs:89-95`, `byroredux/src/systems.rs`.
ClimateRecord exposes sunrise_begin/end, sunset_begin/end (10-minute-unit
bytes) but `weather_system` interpolates sky_colors through hardcoded TOD
slots. Capital Wasteland ships earlier sunrise than Mojave — minor
aesthetic drift.

**Info — FO3-6-06** No FO3-specific CLI/loader code paths exist. Design
goal hit.

## FNV-Shared Surface

Paths ByroRedux inherits from verified FNV support with zero FO3
divergence (verified: no FO3-specific gap found):

- **BSA v104 reader** — entire archive code path identical. Meshes, Textures,
  Sound, Misc, MenuVoices, Voices all open + extract without per-game
  dispatch. Latent `genhash_file` bug shared with FNV (FO3-2-01).
- **ESM header + GRUP framework** — `reader.rs` shared (modulo FO3-3-01
  HEDR band). CELL record parser (`cell.rs:486-620`) game-agnostic.
  XCLL/XCWT/XCLR/XESP layout identical.
- **NIF shader property family** — `BSShaderPPLightingProperty`,
  `BSShaderNoLightingProperty`, `BSShaderTextureSet`, `BSShaderPropertyData`
  all share FO3/FNV code paths with bsver-gated field reads.
- **Skinning chain** — `NiSkinData` / `NiSkinPartition` /
  `BSDismemberSkinInstance` identical across FO3/FNV.
- **Interior lighting template path** — XCLL 40-byte layout, existing
  open issues #378/#379/#380 apply to FO3 without modification.
- **Item records** — WEAP/ARMO/AMMO/MISC/KEYM/ALCH/INGR/BOOK/NOTE/CONT/
  LVLI/LVLN/NPC_/RACE/CLAS/FACT/GLOB/GMST all parse from shared
  `parse_esm` with zero FO3 divergence (counts lower due to no DLC).
- **CLI + cell loader** — `--esm`/`--cell`/`--grid`/`--bsa`/`--textures-bsa`
  all route through the same `cell_loader::load_cell` /
  `load_exterior_cells`. No `esm_name == "..."` gates.

**FO3-only gaps in shared paths**:
- BSShader flag-bit vocabulary differs from Skyrim/FO4 for
  `Double_Sided`/`Refraction_Tint`/`Unknown_3` (FO3-REN-H1). This is the
  single biggest renderer risk because the current code silently computes
  wrong backface culling for every FO3 PPLighting mesh.
- `BSShaderPPLightingProperty` parallax/env map texture slots (3/4/5)
  never read (FO3-REN-M1). Pre-Skyrim parallax uses PPLighting paths that
  the modern BSLightingShaderProperty code doesn't touch.
- `NiTexturingProperty` parallax slot dropped (FO3-NIF-M1) — rarer on FO3
  but affects ported/mixed assets.

## Validation Status

**Interior cell load** — ✓ works end-to-end via `--esm Fallout3.esm
--cell <interior>`. 10/10 canonical probe meshes parse 100% clean. Megaton
Player House baseline (1609 entities, 199 textures) predates renderer
modernization — FO3-6-03 flags the need for a fresh bench pass.

**Exterior cell load** — ⚠ blocked by two issues:
- FO3-6-01: `Wasteland` worldspace name not in `cell_loader.rs` preferred
  list; relies on `max_by_key(cell_count)` fallback.
- FO3-6-02: DLC multi-master FormID remapping absent across the whole
  plugin stack — single-master (vanilla Fallout3.esm) works; any load
  with DLC or ESPs collides form IDs.

**Creature / NPC rendering** — ⚠ mixed:
- Creatures (deathclaw, dog) parse with full skinning chain; CREA ESM
  metadata unreachable (FO3-3-02) — no race/faction/inventory attached.
- NPCs render with invisible faces (FO3-5-01) — `NiTriShapeData` tangent
  mask bug drops every FaceGen head to zero-geometry. Trivial one-character
  fix but has to land before any NPC work is meaningful.

**UI / HUD overlays** — ⚠ partial:
- NoLighting shader paths parse clean (FO3-5-04).
- `TileShaderProperty` drops 20–28 B of texture transforms per block
  (FO3-5-02) — HUD elements usually mispositioned.

**Latent correctness risk** — FO3-3-01 (HEDR=0.94 → Fallout4) is silent
today only because the item DATA arms happen to bucket Fallout4 with
FO3NV/Oblivion. The first FO3↔FO4 schema split after this lands (BGSM
refs, dual-weapon SCOL, BOD2) corrupts FO3 data.

## Recommended Merge Order

1. **FO3-3-01** — HEDR band reorder, add per-master regression tests.
   One-line data fix; prevents silent corruption on future FO4 divergence.
