## Dimensions confirmed clean (with evidence)

### 1. Lock ordering & deadlock (`world.rs`, `lock_tracker.rs`, `query.rs`)

- **Same-thread reentrancy**: `track_read`/`track_write`
  (`lock_tracker.rs:58,99`) panic with a clear message on read-while-write,
  write-while-write, and write-while-read; the thread-local check runs in BOTH
  debug and release (no `cfg` gate on the panic path). Every acquisition site in
  `world.rs` arms a `TrackedRead`/`TrackedWrite` scope, `defuse()`s it only
  AFTER the real lock is acquired, and the query/resource wrapper `Drop` impls
  (`query.rs:185-195,276-280`) untrack.
- **TypeId-sorted multi-lock**: `query_2_mut` / `query_2_mut_mut`
  (`world.rs:390,456`) and `resource_2_mut` / `try_resource_2_mut`
  (`world.rs:592,704`) branch on `id_a < id_b` and arm the tracker scopes in the
  SAME order the locks are acquired — the #313 fix is intact (scopes are not
  armed in generic-parameter order).
- **Same-type double-lock**: all four pair APIs `assert_ne!` on `A == B` with a
  clear message (`world.rs:393,459,595,707`).
- **Cross-thread ABBA graph**: `global_order::record_and_check`
  (`lock_tracker.rs:229`) is `#[cfg(debug_assertions)]` AND opt-in via
  `BYRO_LOCK_ORDER_CHECK`; read-fast-path / write-on-novel-edge; panics on a
  cycle-closing edge. (The skill calls this "debug-only"; it is additionally
  env-gated by design — see the module doc — not a finding.)
- **Poison resolution**: every `lock.read()/.write()/.into_inner()` resolves
  `PoisonError` through `storage_lock_poisoned::<T>()` /
  `storage_lock_poisoned_erased(type_name)` / `resource_lock_poisoned::<R>()`
  (`world.rs:22-51`). `despawn` (`world.rs:121`) uses the erased variant fed by
  the `type_names` side-table (#466).

### 2. Storage correctness (`sparse_set.rs`, `packed.rs`)

- `SparseSetStorage::remove` (`sparse_set.rs:77`) swap-removes and fixes the
  moved entity's sparse pointer (`self.sparse[moved_entity] = Some(dense_idx)`,
  line 107); last-element path takes the no-swap branch (line 92-96); insert into
  an existing entity overwrites in place (line 65-67). `structural_gen` bumps on
  insert + remove only when `TRACK_CHANGES` (lines 55-57, 83-85).
- `PackedStorage::insert/remove` (`packed.rs:94,109`) maintain the sorted
  invariant via `binary_search`; `insert_bulk` (`packed.rs:152`) appends + single
  stable index-sort + last-writer-wins dedup, matching serial `insert` (pinned by
  `insert_bulk_matches_serial_insert_for_unique_ids`,
  `insert_bulk_duplicate_ids_last_writer_wins`).
- Change tracking: `mark_dirty` fires on insert/remove/get_mut and bulk/iter_mut
  conservatively mark the whole set; `drain_dirty_into` clears `out` then appends
  and preserves `self.dirty` capacity (#1371, pinned by
  `drain_dirty_into_preserves_storage_capacity`); `take_dirty` hands capacity
  away. `TRACK_CHANGES` const-folds to zero overhead when off (pinned by
  `untracked_component_never_accumulates_dirty`).
- `World::insert_batch` (`world.rs:187`) wraps the iterator so the
  `entity < next_entity` `debug_assert` still fires per item.

### 3. Query borrow safety (`query.rs`)

- `QueryRead` / `QueryWrite` / `ComponentRef` hold the `RwLock*Guard` for the
  wrapper lifetime and cache the downcast pointer ONCE in `new()` (#1367). Each
  `unsafe { &*self.storage }` / `&mut *self.storage` carries a SAFETY comment
  tying validity to the live guard (`query.rs:58-65,130-144,282-291`). The
  `#[allow(dead_code)] guard` field is never dropped early.
- `ComponentRef` (the sound replacement for the unsound #35 pattern) retains the
  guard; `DerefMut` requires `&mut self`, so `&`/`&mut` into the same storage
  cannot coexist.
- `query`/`query_mut` return `None` for never-created storage (`world.rs:349,367`
  — `self.storages.get(&type_id)?`), no lazy empty-storage creation on the read
  path.

### 4. Resource lifetimes (`world.rs`, `resource.rs`)

- `resource()`/`resource_mut()` panic with the type name on miss
  (`world.rs:540,565`); `try_resource`/`try_resource_mut` return `None`.
- `insert_resource` returns the prior value (downcast back out, `world.rs:508`);
  `remove_resource` resolves poison (`world.rs:521`).
- `try_resource_2_mut` does BOTH existence checks before acquiring EITHER lock
  (`world.rs:714-718`, #465).

### 5 / 5b. System & scheduler (`scheduler.rs`, `access.rs`, `system.rs`, `main.rs`)

- `Stage` enum discriminants `0..=4` in order `Early → Update → PostUpdate →
  Physics → Late` (`scheduler.rs:27-37`); no `ParallelUpdate`/`LateExclusive`
  stage. `AccessConflict` has EXACTLY `None` / `Unknown` / `Conflict` — no
  `Parallel` variant (`access.rs:139-152`); `analyze_pair` returns `Unknown` when
  either side is undeclared (`access.rs:162-171`).
- All 12 parallel-batch systems on the engine binary register via
  `add_to_with_access` (`main.rs:621,637,651,722,770,826,846,864,878,903,916`);
  **zero** plain `scheduler.add_to(` call sites exist. Exclusives register via
  `add_exclusive` (by design — `undeclared_exclusive_count()` non-zero is
  expected).
- Boot guard present: `debug_assert_eq!(report_snapshot.undeclared_parallel_count(),
  0, "undeclared parallel system detected …")` after schedule build
  (`main.rs:934-938`, #1394).

### 6. Unsafe code review

- The only `unsafe` in the ECS core is the three cached-pointer derefs in
  `query.rs` (#1367), each with a guard-lifetime-bounded SAFETY comment. No new
  unsafe block. `World::spawn` uses `checked_add` and panics on overflow
  (`world.rs:85`, #36); `despawn` does not reclaim IDs (#372, documented).

### 7. Component lifecycles (cross-crate)

All nine sub-items verified against live code:

- **Streaming** (`cell_loader/unload.rs:32,191-193`) drains the
  `CellRootIndex` victim set and despawns every owned entity after releasing
  item-instances, **Rapier bodies** (see #1520 note below), skin slots, and GPU
  handles — no component row survives a load/unload cycle.
- **NPC/REFR spawn** idempotent: `compute_streaming_deltas` only emits a cell
  absent from `state.loaded` (`streaming.rs:707`) and dispatch gates on
  `state.pending` (`main.rs:1170`) — no double-spawn without an intervening
  unload.
- **Scripting transient markers** drained by `event_cleanup_system`
  (`scripting/src/cleanup.rs:28-31`), registered last as
  `add_exclusive(Stage::Late, …)` (`main.rs:927`) — single-frame lifetime.
- **ScriptTimer** decrements then removes-and-fires on `<= 0.0`
  (`scripting/src/timer.rs:36-57`) — no negative-time accumulation.
- **AnimationClipRegistry** lowercases the path before lookup/insert
  (`animation/registry.rs:73-78,104-110`) — #790 dedupe intact.
- **DebugDrainSystem** `add_exclusive(Stage::Late, …)`
  (`debug-server/src/lib.rs:33`); all World access inside `run()`; listener
  thread only enqueues; evaluator's sole production write is the deferred
  `PendingDebugLoadSlot` resource.
- **AudioWorld** `audio_system` `add_exclusive(Stage::Late, …)` (`main.rs:902`);
  `OneShotSound`/`AudioEmitter` pruned on `PlaybackState::Stopped`
  (`audio/src/lib.rs`); field drop order drops spatial sub-tracks before the
  listener handle.
- **Particle emitter** `apply_emitter_params` sets size =
  `initial_radius * base_scale.unwrap_or(1.0)` and does NOT clobber color
  (`systems/particle.rs:39-41`).
- **Character / light-anim** `CharacterController` (CellRoot-stamped) and
  `LightFlicker`/`LightSource` rows are caught by the unload CellRoot sweep — no
  orphans.

### 8. Hot-path perf guards

All five guards still in place:

- #823 — `held_others` Vec build is `#[cfg(debug_assertions)]`-gated in both
  `track_read` and `track_write` (`lock_tracker.rs:83-92,125-134`).
- #824 — `NameIndex.map` refilled in place (`clear` + `reserve` + reinsert),

  `(Transform::len(), Parent-len-or-0, next_entity_id())`, folds in
  `Parent`/`Children` `structural_generation()`, uses `drain_dirty_into` (not

- #828 — `events`/`seen_labels` scratches hoisted above the per-entity loop and
  emitted via `clone`, not `mem::take` (`systems/animation.rs:554-555,685`).
- #932 — `footstep_system` drains/restores `FootstepScratch.triggers` via
  `mem::take` + restore on both exit branches (`systems/audio.rs:166,175,198`).
