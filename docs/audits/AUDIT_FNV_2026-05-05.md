## Dimension Findings

### Dim 3 ‚Äî Cell Loading End-to-End

#### MEDIUM ‚Äî `CellLoadResult.weather` / `.climate` are dead fields
- **Location**: `byroredux/src/cell_loader.rs:43-72` (struct), `cell_loader.rs:379-388` (sole producer).
- **Evidence**: `load_cell_with_masters` always emits `weather: None, climate: None`. Three workspace hits for `CellLoadResult\b` (struct + signature + producer); no consumer. `scene.rs:705-755` ignores both fields when destructuring.
- **Impact**: Misleading type signature. Future caller will see `None` and silently render with engine-default fog/ambient. Interior storm-cell or scripted weather override paths would hit this.
- **Fix**: Either populate from `index.weathers` / `index.climates` keyed off cell's worldspace, or drop both fields and update the audit-spec wording. Add a doctest pinning the contract.

#### MEDIUM ‚Äî XCLL extended fog and ambient-cube fields are dropped at the renderer layer
- **Location**: `byroredux/src/components.rs:108-123` (`CellLightingRes`), `byroredux/src/scene.rs:728-739` (only XCLL‚ÜíCellLightingRes conversion site).
- **Evidence**: Plugin layer's `CellLighting` carries 9 optional Skyrim/FNV-extended fields (`directional_fade`, `fog_clip`, `fog_power`, `directional_ambient` cube, `light_fade_*`, `specular_*`, `fog_max`, `fog_far_color`). `CellLightingRes` only carries 3 fog fields (`fog_color`, `fog_near`, `fog_far`). `grep` for any dropped name returns hits only inside parser tests and synthetic LGTM-fallback fixtures ‚Äî zero hits in the renderer.
- **Impact**: FNV interior fog renders as plain linear (clip-at-`fog_far`, no power curve). Skyrim cells with authored ambient cubes (Solitude inn cluster, Dragonsreach, Markarth) render as flat single-channel ambient. M44+ Skyrim SE / FO4 visual fidelity will look subtly wrong against Bethesda reference.
- **Fix**: Extend `CellLightingRes` with `Optional<f32>` / `Optional<[f32;3]>` fields, propagate them through both the XCLL conversion site and the LGTM fallback (`cell_loader.rs:429-441`), then add the corresponding shader uniforms. Pin with a fixture-cell regression test.

#### MEDIUM ‚Äî Streaming worker bypasses `NifImportRegistry`
- **Location**: `byroredux/src/streaming.rs:288-361` (`pre_parse_cell`).
- **Evidence**: Worker has no access to `World` and therefore can't read `NifImportRegistry`. `into_par_iter` over the unique-path set runs with no membership check; `finish_partial_import` then `reg.insert(cache_key, Some(cached))` overwriting any prior cache entry.
- **Impact**: At 7√ó7 grid traversal in WastelandNV, worker pays the full BSA-extract + NIF-parse cost on every cell crossing despite >95% of statics being cached. `mesh.cache` `hits` counter under-reports. Phase 2 hot-reload (scripted teleport) gets punished worst.
- **Fix**: Pass an `Arc<RwLock<HashSet<String>>>` snapshot of currently-cached keys into the worker request alongside `wctx`; skip extraction for paths already present. Refresh the snapshot on the main thread after each `finish_partial_import` insert.

#### MEDIUM ‚Äî `AnimationClipRegistry` is grow-only; leaks under LRU eviction cycles
- **Location**: `crates/core/src/animation/registry.rs:8-89`, `byroredux/src/cell_loader.rs:1212-1217`, `cell_loader_nif_import_registry.rs:262-265`.
- **Evidence**: Registry comment at `registry.rs:9-14` documents the grow-only invariant. LRU eviction in `NifImportRegistry::insert` correctly drops `clip_handles[victim_key]` (line 265) but the underlying `AnimationClipRegistry.clips[handle]` slot is never freed ‚Äî registry has no `remove()` API. Re-parsing the evicted NIF allocates a fresh handle.
- **Impact**: Bounded under default `BYRO_NIF_CACHE_MAX=0` (no eviction). With a finite cap and long doorwalking sessions, every evict-then-revisit cycle adds one keyframe-array's worth of memory. M40 Phase 2 doorwalking is exactly the workload that triggers it.
- **Fix**: Add `AnimationClipRegistry::release(handle: u32)` (clear keyframes + mark slot reusable, or use generational handles). Call it from the LRU eviction site. Update the registry comment.

#### LOW ‚Äî `finish_partial_import` re-runs `convert_nif_clip` on already-cached NIFs
- **Location**: `byroredux/src/cell_loader.rs:1591-1599`.
- **Evidence**: `finish_partial_import` does not check `reg.clip_handle_for(&cache_key).is_some()` before re-converting ‚Äî appends another clip to `AnimationClipRegistry` and overwrites `clip_handles[key]` with the new handle.
- **Impact**: Compounds the streaming-worker-bypasses-cache MEDIUM above. Each re-parse leaks the previous clip handle.
- **Fix**: Early-out at the top of `finish_partial_import` when `reg.get(&cache_key).is_some()`. Best fix is the upstream MEDIUM (worker doesn't re-parse cached models in the first place).

#### LOW ‚Äî `fog_clip` / `fog_power` parsed for FNV but no shader code consumes them
- **Location**: `crates/plugin/src/esm/cell/tests.rs:1683-1740` (parser test); no consumer in `crates/renderer/src/vulkan/shaders/`.
- **Evidence**: Both fields parsed since #379 and pinned by tests; zero references in renderer shaders.
- **Impact**: FNV's authored interior fog renders as plain linear fog. Visible as too-much fog on close camera approaches in cells like Doc Mitchell's House and the Goodsprings Source Pump. "Subtly off" rather than broken.
- **Fix**: Plumb through with the umbrella XCLL MEDIUM above; add a `fog_curve` uniform on geometry / sky-composite shaders.

#### INFO ‚Äî `CellRoot` stamp range covers all spawn-during-load entities
Verified by static analysis. `placement_root` / `AnimationPlayer` / NPC skeleton+body+head / terrain mesh / light-only / effect-sprite-light entities all bracketed by `world.next_entity_id()` snapshots in `stamp_cell_root`. No orphan-leak hypothesis confirmed for the cell-load entity path.
