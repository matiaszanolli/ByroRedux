## Executive Summary

| Severity  | Count |

| CRITICAL  | 5     |

| **Total** | **39** |

**Key picture**:

1. **Oblivion is effectively unsupported at the block level**, despite archive-level "100%" parse rate reported in CLAUDE.md. A v20.0.0.5 header has no `block_sizes` array, so every unhandled RTTI type in the dispatch table raises a fatal `InvalidData` error (mod.rs:380-388). That means the following are all **CRITICAL cascade failures** in Oblivion: `NiBillboardNode`, `NiUVController`/`NiUVData`, `NiKeyframeController`, `NiSequenceStreamHelper`, the entire legacy particle stack (`NiParticleSystemController` + modifier chain), lights (`NiPointLight`/`NiAmbientLight`/`NiSpotLight`/`NiDirectionalLight`), specialized BS shader variants (`SkyShaderProperty`/`WaterShaderProperty`/`TallGrassShaderProperty`/`Lighting30ShaderProperty`), `NiTextureEffect`, `NiSwitchNode`/`NiLODNode`, and `NiStringsExtraData`/`NiIntegersExtraData`. The 100% parse-rate claim reflects archive enumeration tolerance, not shipped mesh coverage.

2. **Skinning is completely unwired in the import pipeline** (N26-4-02). NIF parsers for `NiSkinInstance`, `NiSkinData`, and `NiSkinPartition` landed in N23.5, but `extract_mesh` / `extract_bs_tri_shape` never read `skin_instance_ref`. Every NPC, creature, and biped armor piece across every game renders in bind pose at the parent bone origin. This is the largest single gap between parser capability and end-to-end rendering.

3. **Alpha test is never extracted** (N26-4-03). Only bit 0 of `NiAlphaProperty.flags` (alpha-blend) is read; bit 9 (alpha-test) and the `threshold` byte are both dropped. Every foliage, fence cutout, hair plane, and eyelash surface in every game is visibly broken.

4. **BSOrderedNode / BSValueNode child subtrees are silently dropped** during scene walk (N26-4-01) because `walk.rs` only downcasts to `NiNode`. Common FO3/FNV weapon/effect containers vanish.

5. **NiTexturingProperty "1-byte shortfall" root cause identified** (N26-3-01): the parser reads `Num Shader Textures` as u32 without first reading the gating `Has Shader Textures` bool per nif.xml.

6. **The ECS/stream infrastructure is solid** — version dispatch, RwLock ordering, Arc<str> string table, lock-tracker deadlock detection, block-size mismatch recovery, and BSVER-based binary-layout decisions all verified clean.

## Findings — grouped by severity

### CRITICAL (5)

#### N26-5-01: Oblivion has no fallback path for unknown RTTI
- **Dimension**: Coverage  **Game**: Morrowind, Oblivion
- **Location**: `crates/nif/src/blocks/mod.rs:380-388`
- **Description**: When `block_size` is `None` and dispatch falls through, the parser returns `Err`. Oblivion NIFs use header version 20.0.0.5 with no per-block size array, so any unhandled RTTI aborts the entire mesh. The CLAUDE.md "Oblivion 100%" claim almost certainly reflects archive enumeration tolerance rather than actual block coverage.
- **Suggested Fix**: Add dedicated parsers for every type below (N26-5-02 through N26-5-09). Alternatively, a "best-effort skip" pre-pass computing byte-level seek arithmetic, but dedicated parsers are cleaner.

#### N26-5-02: NiBillboardNode (+ NiSwitchNode, NiLODNode, NiSortAdjustNode, AvoidNode, BSBlastNode, BSDamageStage, BSDebrisNode, BSRangeNode, BSFurnitureMarkerNode, NiBSAnimationNode, NiBSParticleNode) unhandled
- **Dimension**: Coverage  **Game**: All (Oblivion CRITICAL)
- **Location**: `crates/nif/src/blocks/mod.rs:108` (NiNode arm)
- **Description**: 13 `NiNode` subtypes with trivial trailing fields are absent from dispatch. Used pervasively: foliage, magic FX, fire effects, LOD nodes, architecture, furniture interaction markers.
- **Suggested Fix**: Extend the `NiNode` arm with all 13 names mapped to either a generic `NiNode::parse_with_trailing` or individual subtypes. Most need 1-2 extra fields.

#### N26-5-04: Legacy Oblivion particle stack entirely missing
- **Dimension**: Coverage  **Game**: Morrowind, Oblivion, some FO3 legacy meshes
- **Location**: `crates/nif/src/blocks/mod.rs` (no arms for legacy particle types)
- **Description**: Oblivion uses the pre-NiPSys particle architecture: `NiParticleSystemController` / `NiBSPArrayController` drives `NiAutoNormalParticles`/`NiRotatingParticles` with a chain of `NiParticleModifier` subclasses (`NiParticleColorModifier`, `NiParticleGrowFade`, `NiParticleRotation`, `NiParticleBomb`, `NiGravity`, `NiPlanarCollider`, `NiSphericalCollider`). None dispatched → every Oblivion particle effect fatally aborts its mesh.
- **Suggested Fix**: Create `legacy_particle.rs` mirroring the existing `particle.rs`. Reference layouts in Gamebryo 2.3 source `CoreLibs/NiParticle/`.

#### N26-5-05: NiKeyframeController + NiSequenceStreamHelper missing
- **Dimension**: Coverage  **Game**: Oblivion KF files, animated Morrowind meshes
- **Location**: `crates/nif/src/blocks/mod.rs`
- **Description**: Pre-Skyrim KF animation files use `NiSequenceStreamHelper` as root and `NiKeyframeController` as per-bone driver. Redux only handles the modern `NiControllerSequence` / `NiTransformInterpolator` path.
- **Suggested Fix**: `NiKeyframeController` is a `NiSingleInterpController` subclass with a single trailing `data: ref<NiKeyframeData>` (aliases to NiTransformData). Add to the existing `NiSingleInterpController` match arm. `NiSequenceStreamHelper` is a `NiObjectNET` with no extra fields.

#### N26-5-09: Oblivion specialized BS shader variants unhandled
- **Dimension**: Coverage  **Game**: Oblivion exteriors
- **Location**: `crates/nif/src/blocks/mod.rs:124-128`
- **Description**: `SkyShaderProperty`, `WaterShaderProperty`, `TallGrassShaderProperty`, `Lighting30ShaderProperty`, `TileShaderProperty`, `HairShaderProperty`, `VolumetricFogShaderProperty`, `DistantLODShaderProperty`, `BSDistantTreeShaderProperty`, `BSSkyShaderProperty`, `BSWaterShaderProperty` all derive from `BSShaderLightingProperty` but are absent.
- **Impact**: Oblivion exterior cells (sky, water, grass, distant LOD trees) hard-fail. Most cells in the world rely on at least one of these.
- **Suggested Fix**: Alias all of them to `BSShaderPPLighting::parse` as a starting point — they share the texture-set + flags layout. Specialize differences in follow-up.

### HIGH (14)

#### N26-1-01: BSSegmentedTriShape dispatched as plain NiTriShape
- **Dimension**: Block Parsing  **Game**: FO3/FNV/Skyrim LE/SE
- **Location**: `crates/nif/src/blocks/mod.rs:116-118`
- **Description**: `BSSegmentedTriShape` extends NiTriShape with Num Segments (u32) and BSGeometrySegmentData array. Dispatch aliases to plain NiTriShape, dropping segment data. Block-size skip hides the misalignment but segmentation info (LOD chunking, biped body parts) is lost.
- **Suggested Fix**: Add dedicated `BsSegmentedTriShape` wrapper that delegates to `NiTriShape::parse` then reads the segment array.

#### N26-1-02: BSMeshLODTriShape / BSSubIndexTriShape dispatched as plain BSTriShape
- **Dimension**: Block Parsing  **Game**: Skyrim SE, FO4, FO76
- **Location**: `crates/nif/src/blocks/mod.rs:119-121`
- **Description**: `BSMeshLODTriShape` adds three uint LOD sizes; `BSSubIndexTriShape` adds full segmentation block. Both dispatch to `BsTriShape::parse` without reading extensions. BSSubIndexTriShape is ubiquitous in Skyrim SE DLC and all FO4 actor meshes (clothing segmentation).
- **Suggested Fix**: Add distinct structs (or `BsTriShape::parse_with_extra` with extension callback).

#### N26-1-04: BSMultiBoundNode dispatched as plain NiNode, multi_bound ref + culling_mode dropped
- **Dimension**: Block Parsing  **Game**: FO3/FNV/Skyrim LE/SE
- **Location**: `crates/nif/src/blocks/mod.rs:108-109`
- **Description**: `BSMultiBoundNode` inherits NiNode and adds `Multi Bound` (Ref → BSMultiBound) field and (Skyrim+) `Culling Mode` u32. Currently aliased to plain NiNode.
- **Impact**: Multi-bound culling volumes (large interior cell optimization) never wired up; target ref dropped.
- **Suggested Fix**: Add dedicated `BsMultiBoundNode` struct that delegates to `NiNode::parse` then reads `multi_bound_ref` + Skyrim `culling_mode`.

#### N26-3-01: NiTexturingProperty reads Num Shader Textures without the gating Has Shader Textures bool
- **Dimension**: Stream Position  **Game**: Oblivion, FO3, FNV, Skyrim LE
- **Location**: `crates/nif/src/blocks/properties.rs:238-254`
- **Status**: NEW — **plausible root cause of the long-standing 1-byte shortfall**
- **Description**: nif.xml specifies:
  ```
  Has Shader Textures : bool      (ver >= 10.0.1.0)
  Num Shader Textures : uint      (cond="Has Shader Textures")
  Shader Textures     : array[N]  (cond="Has Shader Textures")
  ```
  The parser unconditionally reads a `u32` as the count, skipping the leading bool. When the file has `has_shader_textures=0x00` (the common case), the parser swallows that byte + 3 trailing bytes as a bogus count, leaving the stream ~3 bytes past block end.
- **Suggested Fix**: Insert the missing `has_shader_textures` bool read before the count, gating both. Add a regression test.

#### N26-4-01: BSOrderedNode and BSValueNode subtrees silently dropped during import
- **Dimension**: Import Pipeline  **Game**: FO3, FNV (BSOrderedNode is alpha-sort container)
- **Location**: `crates/nif/src/import/walk.rs:18-90` and `96-166`
- **Description**: `walk_node_hierarchical` and `walk_node_flat` only downcast to `NiNode`, `NiTriShape`, `BsTriShape`. `BsOrderedNode` and `BsValueNode` are parsed as distinct concrete types containing a `base: NiNode`. The downcast fails, function returns without recursing, every child is dropped.
- **Impact**: Transparent geometry pre-sorted via BSOrderedNode (very common in FO3/FNV weapon/effect NIFs, architecture) never reaches the renderer.
- **Suggested Fix**: Add `downcast_ref::<BsOrderedNode>()` and `downcast_ref::<BsValueNode>()` branches that operate on `node.base`. Better: helper `as_ni_node(&dyn NiObject) -> Option<&NiNode>` or a `HasNiNode` trait.

#### N26-4-02: Skinning data completely dropped during import
- **Dimension**: Import Pipeline  **Game**: ALL — every animated character/creature
- **Location**: `crates/nif/src/import/mesh.rs:26-118` and `126-258`
- **Description**: `extract_mesh` and `extract_bs_tri_shape` ignore `skin_instance_ref` entirely. `ImportedMesh` has no skin fields; no reference to `NiSkinInstance`/`NiSkinPartition`/`BsSkinInstance` anywhere in import/. Parsers exist (landed in N23.5) but are never read.
- **Impact**: All skinned meshes (NPCs, creatures, biped armor, hands) render in bind pose at parent bone origin. Animation can move parent bone but cannot deform mesh. Effectively no skeletal animation end-to-end despite N23 parsing all 6 skinning block types.
- **Suggested Fix**: Add `skin: Option<ImportedSkin>` to `ImportedMesh`. Walk `NiSkinInstance → bones[] + NiSkinData (bind transforms, vertex weights) + NiSkinPartition (GPU partitioned weights)` and emit a bind-pose-relative skin component. Wire to renderer skin uniform path.

#### N26-4-03: NiAlphaProperty alpha-test bit and threshold never extracted
- **Dimension**: Import Pipeline  **Game**: ALL — foliage, fences, hair, eyelashes
- **Location**: `crates/nif/src/import/material.rs:159-179`
- **Description**: NiAlphaProperty.flags packs both alpha-blend (bit 0) and alpha-test (bit 9, 0x200). Importer only checks bit 0. Parser already reads `threshold: u8` at `properties.rs:94`, but `MaterialInfo` has no `alpha_test`/`alpha_threshold` field.
- **Suggested Fix**: Add `alpha_test: bool` and `alpha_threshold: f32` to `MaterialInfo`/`ImportedMesh`. Set `alpha_test = (alpha.flags & 0x200) != 0`, `alpha_threshold = alpha.threshold as f32 / 255.0`. Renderer branches on alpha_test for fragment shader discard.

#### N26-5-03: Embedded NiCamera not handled
- **Dimension**: Coverage  **Game**: All (Oblivion CRITICAL)
- **Description**: Animated cinematic NIFs ship with a `NiCamera` block (frustum + view transform). Required for cinematic/cutscene playback.
- **Suggested Fix**: Add `NiCamera` parser: NiAVObject base + 4× frustum planes + near/far + ortho flag + viewport rect + LOD_adjust + scene_ref + screen_polygons/textures.

#### N26-5-06: NiUVController / NiUVData missing
- **Dimension**: Coverage  **Game**: Oblivion (water, fire, banners), all earlier
- **Description**: Scrolling-texture animation. `NiUVController` is `NiTimeController` subclass with `target_attribute: ushort` and `data: ref<NiUVData>`. `NiUVData` contains four key groups (offset U/V, tiling U/V).
- **Suggested Fix**: New parser pair; both layouts version-stable.

#### N26-5-07: NiBSpline* compressed animation family missing
- **Dimension**: Coverage  **Game**: Skyrim LE/SE, FO4
- **Description**: 16 related types. KF files for actor body/face animation use compressed-transform variant almost exclusively. Currently load as `NiUnknown`; transforms collapse to bind pose.
- **Suggested Fix**: Implement at least `NiBSplineCompTransformInterpolator` + `NiBSplineData` + `NiBSplineBasisData`. Decompression formula in `NiAnimation/NiBSplineCompInterpolator.cpp` in the Gamebryo 2.3 source; output is same TQS keyframe set we already evaluate.

#### N26-5-08: NiPointLight / NiAmbientLight / NiSpotLight / NiDirectionalLight missing
- **Dimension**: Coverage  **Game**: All (Oblivion CRITICAL)
- **Description**: `NiLight` subclasses for per-mesh light sources. Currently lighting is sourced only from cell XCLL — meshes that carry their own emitter are silently dark. Visible holes in night scenes (torches, candles).
- **Suggested Fix**: 4 small parsers sharing the `NiDynamicEffect` base. Each needs dimmer, ambient/diffuse/specular color, point/spot constant/linear/quadratic attenuation; spot adds cutoff + exponent.

#### N26-5-11: BSDynamicTriShape & BSLODTriShape missing
- **Dimension**: Coverage  **Game**: Skyrim LE/SE, FO4
- **Location**: `crates/nif/src/blocks/mod.rs:119-121`
- **Description**: Only `BSTriShape | BSMeshLODTriShape | BSSubIndexTriShape` handled. Skyrim facegen uses `BSDynamicTriShape`; FO4 uses `BSLODTriShape` for LOD meshes. Silently routed to `NiUnknown`, dropping geometry.
- **Impact**: Faces invisible in Skyrim/SE/FO4 character meshes. Distant LOD missing in FO4.
- **Suggested Fix**: Add both to existing `BsTriShape::parse` arm — layouts near-identical except for BSDynamicTriShape's extra `vertex_size` and BSLODTriShape's trailing per-LOD count.

#### N26-5-12: BSPackedCombined[Shared]GeomDataExtra missing
- **Dimension**: Coverage  **Game**: Skyrim+/FO4 distant LOD
- **Description**: Packed geometry batch extras attached to BSMultiBoundNode roots in cell LOD NIFs. Without them the engine can't reconstruct merged distant meshes.
- **Suggested Fix**: Implement both as `NiExtraData` subtypes; contain object lists + per-object transform stride + vertex/triangle pools.

### MEDIUM (7)

#### N26-1-03: BSTreeNode dispatched as plain NiNode, two ref-lists dropped
- **Dimension**: Block Parsing  **Game**: Skyrim LE/SE SpeedTree nodes
- **Location**: `crates/nif/src/blocks/mod.rs:108-109`
- **Description**: `BSTreeNode` adds `Num Bones 1`/`Bones 1[]` and `Num Bones 2`/`Bones[]`. Block-size skip prevents downstream corruption but bone structures are silently lost.
- **Suggested Fix**: Add dedicated `BsTreeNode { base: NiNode, bones_1, bones_2 }` wrapper.

#### N26-2-02: has_properties_list misses pre-Skyrim Unknown variants
- **Dimension**: Version Handling  **Game**: Non-Bethesda Gamebryo (Civ IV, Freedom Force, etc.)
- **Location**: `crates/nif/src/version.rs:178-180`, `blocks/base.rs:73`
- **Description**: nif.xml gates `Num Properties`/`Properties` on `#NI_BS_LTE_FO3#` = `BSVER <= 34`, covering all pre-Skyrim files. `detect()` only classifies Bethesda variants; any other pre-Skyrim NIF falls to `NifVariant::Unknown` → `has_properties_list` returns false → NiAVObject parser silently skips the properties list and corrupts the block stream.
- **Suggested Fix**: Replace variant-based check with direct `stream.bsver() <= 34` in `base.rs:73`, mirroring the pattern already used for `flags` at line 63.

#### N26-3-02: parse_emitter_ctlr_data silently falls back to key_size=8 for unknown interpolation
- **Dimension**: Stream Position  **Game**: FO3, Oblivion (animated particle emitters)
- **Location**: `crates/nif/src/blocks/particle.rs:672-688`
- **Description**: Unknown interpolation modes use default 8 bytes/key, misaligning the stream. block_size recovery catches it elsewhere, but invisible corruption in Oblivion.
- **Suggested Fix**: Return `InvalidData` on unknown interpolation, or at minimum log warning.

#### N26-3-03: NifStream::skip doesn't clamp to cursor length; raw pos+n may wrap
- **Dimension**: Stream Position  **Game**: All
- **Location**: `crates/nif/src/stream.rs:63-66`
- **Description**: `skip(n)` calls `cursor.set_position(pos + n)` with no bounds check, no overflow check. In release builds `u64` addition wraps silently. Called from ~30 sites in particle/skin/texture parsers.
- **Suggested Fix**: Make `skip` return `io::Result<()>`, use `checked_add`, clamp to buffer length, return `UnexpectedEof` on overshoot. Propagate errors from call sites (each already returns `io::Result`).

#### N26-4-04: NiTexturingProperty multi-slot textures (normal/glow/detail) dropped
- **Dimension**: Import Pipeline  **Game**: Oblivion
- **Location**: `crates/nif/src/import/material.rs:193-205`
- **Description**: NiTexturingProperty has 7 slots (base, dark, detail, gloss, glow, bump, normal). Importer only reads `base_texture`. For Oblivion there is no BSShaderTextureSet, so `info.normal_map` is ALWAYS None.
- **Impact**: Oblivion has no normal mapping, no glow maps, no detail textures. Related to closed #130 which fixed FO3/FNV; Oblivion was missed.
- **Suggested Fix**: Also extract `normal_texture`/`bump_texture` and `glow_texture` via the `NiSourceTexture` lookup pattern.

#### N26-4-05: BsTriShape never resolves BSEffectShaderProperty for material data
- **Dimension**: Import Pipeline  **Game**: Skyrim SE, FO4, FO76, Starfield
- **Location**: `crates/nif/src/import/mesh.rs:184-230`, `mesh.rs:269-292`
- **Description**: `extract_bs_tri_shape` only tries `BSLightingShaderProperty`. BsTriShape geometry with `BSEffectShaderProperty` (water, decals, force fields, holograms, Pip-Boy screens) gets default material defaults instead.
- **Suggested Fix**: After BSLightingShaderProperty branch, try BSEffectShaderProperty. Better: refactor `extract_material_info` to take generic shader_property_ref so there's one pipeline.

#### N26-5-10: NiTextureEffect missing
- **Dimension**: Coverage  **Game**: Oblivion (decal/env-map projector)
- **Description**: `NiTextureEffect` is `NiDynamicEffect` for projected textures (sphere maps, env maps, projected shadows). Common in Oblivion magic FX.
- **Suggested Fix**: New parser with ~12 fields including projection matrix and clamp mode.

#### N26-5-13: NiStringsExtraData / NiIntegersExtraData unhandled
- **Dimension**: Coverage  **Game**: All (Oblivion CRITICAL)
- **Location**: `crates/nif/src/blocks/mod.rs:158-159`
- **Description**: Only scalar variants handled. Array variants ship in Bethesda meshes (e.g. material override lists).
- **Suggested Fix**: Trivial extension of existing `NiExtraData::parse` match.

- **Description**: When a block parser returns Err on an Oblivion NIF (no block_size), the loop breaks silently and returns a truncated `NifScene` as `Ok`. Consumers get incomplete scenes with dangling BlockRefs that resolve to None — no crash but silent data loss.
- **Suggested Fix**: Either return Err from parse_nif on Oblivion truncation (callers opt in via ParseOptions), or add `parse_truncated: bool` field to NifScene with consumer warning.

#### N26-4-06: Editor marker culling misses MapMarker and Skyrim EditorMarker flag bit
- **Dimension**: Import Pipeline  **Game**: Skyrim+, all games for MapMarker
- **Location**: `crates/nif/src/import/walk.rs:169-176`
- **Description**: `is_editor_marker` doesn't match `MapMarker`, and Skyrim's bit-5 `bEditorMarker` flag (0x20) is never checked.
- **Suggested Fix**: Add `"mapmarker"` to name match. In each shape/node arm, add `if av.flags & 0x20 != 0 { return; }` gated on user_version2 >= 12 (Skyrim+).

#### N26-4-07: Skyrim SLSF2 decal bits never checked on BSLightingShaderProperty

- **Location**: `crates/nif/src/import/material.rs:123-125`
- **Description**: Skyrim SLSF2 decal bits in `shader_flags_2` (Skyrim's Decal/Dynamic_Decal bits) are never checked; only inherited SF1 bits 26/27. Some Skyrim decal meshes z-fight.
- **Suggested Fix**: Verify Skyrim SLSF2 decal bit positions in nif.xml's `SkyrimShaderPropertyFlags2`, add SF2 check to the existing condition.
