# NIF Parser Audit — 2026-04-18

**Scope**: Full NIF binary format parser — block correctness, version dispatch, stream position integrity, import pipeline, coverage gaps.
**Method**: 5 parallel dimension agents, static analysis (no live BSA sweep this pass — would add ~40 min for full FNV/Oblivion pass).
**Baseline**: Post-#343 (ShaderTypeData exhaustive match), post-#347 (GameKind for ESM items), post-#388 (allocate_vec helper in progress).

---

## Executive Summary

**~30 unique findings** across 5 dimensions — 5 Critical, ~13 High, ~8 Medium, rest Low. Two themes dominate:

1. **BsTriShape import pipeline has a Critical regression from the #343 landing**: the NiTriShape path got an `apply_shader_type_data` helper that writes 9 variants onto `MaterialInfo`, but the BsTriShape path (mesh.rs:317-327) still only reads `EnvironmentMap.env_map_scale` and silently drops SkinTint, HairTint, Parallax, SparkleSnow, EyeEnvmap — the exact variants #343 was supposed to address. `material_kind` is set on `ImportedMesh` but its payload is not. **D4-N01 re-opens half of SK-D3-01 for BsTriShape.**

2. **Silent block-stream drift on Oblivion**: a new Critical finding (**NIF-D3-C1**) for `NiTexturingProperty` over-consuming 1 byte on Oblivion meshes with `texture_count > 7`. This is the inverse of the long-standing "1-byte shortfall" symptom and explains the intermittent Oblivion drift not covered by #388/#395. Fix is a version gate: `has_normal_texture` and `has_parallax_texture` are `since="20.2.0.5"`, absent at Oblivion v20.0.0.5.

Plus **five trivial coverage additions** (~5 LOC of dispatch-table wiring) that unblock animation events + emissive-color controllers engine-wide.

## Findings (Deduplicated, Grouped by Severity)

### Critical

**NIF-D3-C1** [Dim 3] — `NiTexturingProperty` over-consumes 1 byte on Oblivion meshes with `texture_count > 7`
`crates/nif/src/blocks/properties.rs:259-288`. `Has Normal Texture` and `Has Parallax Texture` are `since="20.2.0.5"` — **absent at Oblivion v20.0.0.5**. Parser reads them unconditionally. Fixes the "drift on decal-heavy Oblivion clutter" symptom that #388/#395 didn't cover. Oblivion has no block_sizes → every subsequent block misaligned.
**Fix**: gate `normal_texture` + `parallax_texture` reads on `version >= 0x14020005`; pre-20.2.0.5 branch enters decal loop directly with `num_decals = texture_count.saturating_sub(6)`.

**NIF-D1-C1** [Dim 1, unverified] — `NiPSysData` skips NiGeometryData-base fields
`crates/nif/src/blocks/particle.rs:137-189`. Reads extra `has_rotation_speeds_2` bool that doesn't exist in spec, never emits `NumAddedParticles`/`AddedParticlesBase`. Every FO3/FNV/Skyrim particle-bearing NIF desyncs. **Needs line-verification vs Gamebryo 2.3 `NiPSysData.cpp` before filing.**

**NIF-D1-C2** [Dim 1, unverified] — `NiMaterialProperty` flags u16 not user_version-gated
`crates/nif/src/blocks/properties.rs:48-56`. Always reads u16 flags; nif.xml says absent when user_version >= 26 (FO4). FO4 SCOL precombined meshes desync 2 bytes. **Needs verification**.

**NIF-D2-C1** [Dim 2] — Version gate ignores game variant for shared version 20.2.0.7
`crates/nif/src/version.rs`. Oblivion-final / FO3 / FNV / Skyrim LE all share v20.2.0.7 but disambiguate via user_version/bs_version. Predicates key off version + single bs_version threshold rather than a `GameVariant` enum. Underlying cause of several lower-severity findings.
**Fix**: Introduce `GameVariant { Oblivion, Fo3, Fnv, SkyrimLE, SkyrimSE, Fo4, Fo76, Starfield, StockGamebryo }` at header parse time, re-express `bs_version >= N` thresholds as `variant.is_at_least(Fo4)`.

**D4-N01** [Dim 4] — BsTriShape drops ShaderTypeData payload (re-opens half of #343)
`crates/nif/src/import/mesh.rs:317-327`. After recent `material_kind` capture addition, the `match shader.shader_type_data` only reads `env_map_scale` and collapses every other variant to 1.0 fallback. SkinTint RGB, HairTint, EyeEnvmap centers, Parallax height scale, MultiLayer thickness, SparkleSnow params — all silently dropped on Skyrim SE/FO4/FO76/Starfield BsTriShape meshes. `material_kind` set on ImportedMesh without its payload.
**Fix**: Reuse `apply_shader_type_data` helper (currently private to material.rs) on the BsTriShape path, or refactor `extract_bs_tri_shape` to build a full `MaterialInfo` first.

### High

**NIF-D1-H2** [Dim 1, unverified] — BSShaderPPLightingProperty missing NiShadeProperty 2-byte flags base. FO3/FNV universal if confirmed.

**NIF-D2-C2/H3** [Dim 2] — Bethesda-only allowlist; endian byte unread. Non-Bethesda Gamebryo + big-endian console ports mis-parse. Subsumed by C1 GameVariant refactor except for endian which is independent.

**NIF-D2-H1/H2** [Dim 2] — BSStreamHeader threshold duplicated across parsers; raw bs_version thresholds (>= 34, >= 83, etc.) without named constants.

**D4-N02** [Dim 4] — BsTriShape `find_material_path_bs_tri_shape` only looks at BSLightingShaderProperty, drops BSEffectShaderProperty `.bgem` paths. FO4/FO76 effect shaders lose material pointer.

**D4-N03** [Dim 4] — `BSShaderNoLightingProperty` never reads normal_map or uv_transform though the shader carries them. FO3/FNV/Oblivion skybox/UI/map panels miss scroll/scale.

**D4-N04** [Dim 4] — Defensive: change second `if let` to `else if let` for BSLightingShader/BSEffectShader mutual exclusion at material.rs:437-465.

**NIF-COV-01** [Dim 5] — `NiColorInterpolator` missing. Every animated emissive-color surface (muzzle flashes, plasma, glow pulses) has a controller pointing to a NiUnknown ref. **Extremely high frequency**. ~30 LOC fix.

**NIF-COV-04** [Dim 5] — `BSAnimNote`/`BSAnimNotes` missing. Every `.kf` weapon/creature idle loses animation-event hooks (footsteps, weapon impact frames, fX triggers) on KF→ECS conversion. ~20 LOC; text-event channel already exists in `crates/core/src/animation/text_events.rs`.

**NIF-COV-02** [Dim 5] — 4× Ni*LightController variants. Every animated point/spot light (lantern flicker, torch pulse, magic glow) land as NiUnknown. ~4 LOC.

**NIF-COV-07/09** [Dim 5] — `BSBoneLODExtraData` (every Skyrim actor + FO4 NPC), `BSPackedAdditionalGeometryData` (Skyrim facegen masks).

**NIF-COV-06/08/10** [Dim 5] — `BSMultiBoundSphere`/Data, `BSDistantObjectLargeRefExtraData` (SSE), `BSExtraData` family (FO4+).

### Medium

**NIF-D3-H1** [Dim 3] — `NiTextKeyExtraData` doesn't route through shared NiExtraData dispatcher. Morrowind-only impact; flag for M22.

**NIF-D3-H2** [Dim 3] — `NiLegacyParticles::parse` reads `has_shader` bool unconditionally. Morrowind-only.

**NIF-D3-M1** [Dim 3] — `NiSkinPartition::has_bone_indices` not version-gated. Morrowind-only.

**D4-N05** [Dim 4] — `extract_vertex_colors` ignores `_mat: &MaterialInfo` parameter. 3× redundant property scan per NiTriShape.

**D4-N06** [Dim 4] — NiTriShape UV transform dropped when NiMaterialProperty comes first in property list. Oblivion/FO3/FNV tapestries + scrolling signs lose authored UV scroll/scale.

**D4-N07** [Dim 4] — `as_ni_node` BSFadeNode coverage needs line-verification.

**NIF-COV-11/12/14/15** [Dim 5] — `BSTreadTransfInterpolator`, `BSRefractionFirePeriodController`, Havok pose/ragdoll templates, `NiPSysTrailEmitter`.

## Regression Guard List (Confirmed Still Holding)

- `NiTexturingProperty` trailing `num_shader_textures` reads raw u32 (regression-pinned).
- `user_version` threshold `>= 10.0.1.8` (not 10.0.1.0).
- `BSStreamHeader` dual condition `version == 10.0.1.2 || user_version >= 3`.
- Strings table gate `>= 20.1.0.1` (Oblivion 20.0.0.5 correctly inlines).
- `NiTexturingProperty` flags absent in 10.0.1.3–20.1.0.1 gap.
- `stream.bsver()` is in-file `user_version_2`, not variant's hardcoded value.
- No `srgb_to_linear` on NiMaterialProperty colors.
- `#343` ShaderTypeData exhaustive match on NiTriShape path — verified landed, but **NOT** mirrored on BsTriShape path (D4-N01).
- `#347` GameKind dispatch in `esm::records::mod.rs` — landed.
