### Findings

#### FO4-D4-NEW-01: TXST DODT decal-data sub-record silently dropped (207/382 vanilla TXSTs)

**Severity**: HIGH
**Location**: [crates/plugin/src/esm/cell/support.rs:230-258](crates/plugin/src/esm/cell/support.rs#L230-L258)
**Status**: NEW

`parse_txst_group` matches TX00..TX07 + MNAM but has no `b"DODT"` arm.
`TextureSet` (`cell/mod.rs:514-530`) has no `decal_data` field. DODT is
the texture-set decal-data sub-record (per UESP / xEdit
`wbDefinitionsFO4`): fixed-layout struct with min/max width, depth,
shininess, parallax scale + passes, RGBA color, flags. Affects every
decal-bearing TXST authoring (blood splatters, scorch marks, posters,
graffiti, signs).

**Fix**: Add `decal_data: Option<DecalData>` to `TextureSet` + DODT
arm. Mirror the `LightData` / `AddonData` pattern (length-gate +
defensive). Renderer-side decal RenderLayer wiring is M28-extension
follow-up.

#### FO4-D4-NEW-02: TXST DNAM flag bits dropped on 100 % of vanilla records (382/382)

**Severity**: MEDIUM
**Location**: [crates/plugin/src/esm/cell/support.rs:230-258](crates/plugin/src/esm/cell/support.rs#L230-L258), [crates/plugin/src/esm/cell/mod.rs:514-530](crates/plugin/src/esm/cell/mod.rs#L514-L530)
**Status**: NEW

Every vanilla `Fallout4.esm` TXST (382/382) carries a DNAM (u16 on FO4
per UESP/xEdit, was u8 on Skyrim). Same `_ => {}` catch-all swallows
it. Two specific consequences: `HasModelSpaceNormals` flips
normal-map decoding (object-space vs tangent-space) — face TXSTs read
garbage normals today; `NoSpecular` should clip the specular slot at
TXST level.

**Fix**: Add `flags: u16` (FO4) / `u8` (Skyrim) on `TextureSet`, DNAM
arm gated on `game`. Renderer-side `RefrTextureOverlay` carries the
flags forward; BSP normal-map upload branches on `HasModelSpaceNormals`.

#### FO4-D4-NEW-03: PKIN FLTR sub-record silently dropped (230/872 vanilla PKINs)

**Severity**: MEDIUM
**Location**: [crates/plugin/src/esm/records/pkin.rs:80-99](crates/plugin/src/esm/records/pkin.rs#L80-L99)
**Status**: NEW

`parse_pkin` has CNAM/VNAM/FNAM arms but no FLTR. The sister parser
`parse_scol` (records/scol.rs:158-175) DOES collect FLTR — PKIN
diverged. FLTR drives FO4 Workshop build-mode filtering.

**Fix**: Add `filter: Vec<u32>` to `PkinRecord` and a `b"FLTR"` arm
mirroring `parse_scol` exactly (same flat u32 array layout).

#### FO4-D4-NEW-04: SCOL `FULL` display name silently dropped (124/2617 vanilla SCOLs)

**Severity**: LOW
**Location**: [crates/plugin/src/esm/records/scol.rs:111-198](crates/plugin/src/esm/records/scol.rs#L111-L198)
**Status**: NEW

`parse_scol` walks EDID/MODL/ONAM/DATA/FLTR but no FULL arm. No
render impact today; consistency gap with every other record type
that captures display name.

**Fix**: Add `full_name: String` to `ScolRecord`, route through
`read_lstring_or_zstring` to match the localised-plugin handling
established in #348.

#### FO4-D4-NEW-05: Architecture-record categories invisible to `EsmIndex::total()` / category-pin test

**Severity**: MEDIUM
**Location**: [crates/plugin/src/esm/records/mod.rs:360-450](crates/plugin/src/esm/records/mod.rs#L360-L450) (`categories()`); [crates/plugin/src/esm/records/mod.rs:2262-2280](crates/plugin/src/esm/records/mod.rs#L2262-L2280) (`categories_table_row_count_pinned`)
**Status**: NEW

The `EsmIndex::categories()` table — single source of truth for
`total()` / `category_breakdown()` and the
`categories_table_row_count_pinned == 82` regression test — does NOT
include rows for the 5 FO4-architecture maps:

```
texture_sets    (379 / 382 captured)
scols           (2617 records, 40,330 placements)
packins         (872 records)
movables        (0 vanilla, DLC-relevant)
material_swaps  (2537 records)
```

These live on `EsmCellIndex`, not `EsmIndex`. A regression that wipes
`cells.scols` (e.g. a future `parse_scol_group` rewrite that drops the
insert) won't trip any test or CI alert.

**Fix**: Add 5 rows referencing `s.cells.*.len()` (pattern already
established for `cells` and `statics`). Bump pin from 82 → 87.

#### FO4-D4-NEW-06: TXST records with DODT-only (3/382) silently dropped from `texture_sets` map

**Severity**: LOW
**Location**: [crates/plugin/src/esm/cell/support.rs:265-267](crates/plugin/src/esm/cell/support.rs#L265-L267)
**Status**: NEW

The final guard `if set != TextureSet::default()` drops 3 vanilla
TXSTs that ship only DODT + DNAM (no TX00..TX07, no MNAM). Once
NEW-01 / NEW-02 land and `TextureSet` carries `decal_data` + `flags`,
the equality guard naturally re-admits these records.

**Fix**: Naturally fixed once NEW-01 + NEW-02 land. Until then,
extend the guard to admit records with DODT or DNAM present.

#### FO4-D4-NEW-07: No FO4 ESM parse-rate harness — FNV / FO3 / Oblivion covered, FO4 silent

**Severity**: HIGH
**Location**: [crates/plugin/tests/parse_real_esm.rs](crates/plugin/tests/parse_real_esm.rs) (FNV/FO3/Oblivion harnesses; no FO4 equivalent)
**Status**: NEW

`parse_real_esm.rs` defines `parse_rate_fnv_esm`, `parse_rate_fo3_esm`,
`parse_rate_oblivion_esm` — `#[ignore]`-gated on `BYROREDUX_<GAME>_DATA`
env vars with per-game total + per-category floors. There is no
`parse_rate_fo4_esm`. Live measurement (`parse_esm("Fallout4.esm")`
succeeds): 964 cells / 31,989 statics / 2,617 SCOLs / 872 PKINs /
2,537 MSWPs / 379 TXSTs / `idx.game == GameKind::Fallout4`. Parser is
clean today; no regression floor.

**Fix**: Add `parse_rate_fo4_esm` mirroring the FNV harness. Env
`BYROREDUX_FO4_DATA` (already used by `bsa/bgsm/nif` tests). Floors
with 5 % buffer below live measurements: `cells.cells >= 900`,
`scols >= 2500`, `packins >= 850`, `material_swaps >= 2400`,
`texture_sets >= 370`, `cells.statics >= 30000`. Pin
`idx.game == GameKind::Fallout4`.
