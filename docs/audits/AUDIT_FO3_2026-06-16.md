## Findings by Severity

### HIGH (1)

#### D7-4: FO3 SCPT runtime gap — 1 257 scripts parse, none execute (KNOWN BLOCKER)
- **Severity**: HIGH (functional gap, by design)
- **Dimension**: 7 — Animation/NPC/Scripting
- **Location**: `crates/plugin/src/esm/records/script.rs:13-16`; `crates/scripting/src/translate/source.rs:33-37`; no runtime driver in `byroredux/src/cell_loader/` or main loop
- **Status**: NEW (record as KNOWN BLOCKER, not a bug to file)
- **Description**: FO3 SCPT records parse into structured `ScriptRecord` (SCHR/SCDA/SCTX/SLSD/SCVR/SCRV/SCRO) but the parser is explicitly extraction-only ("Bytecode runtime is out of scope"). The translate layer defines an `Obscript(&ScriptRecord)` source variant documented as "Designed-for; the bytecode disassembler is a later phase." No call site drives Obscript translation or SCDA execution — the M30/M47 ECS-native path runs only on hand-written recognizers and Papyrus `.psc` AST, never on FO3 SCDA bytecode.
- **Impact**: Largest FO3-specific functional gap. SCRI cross-references resolve (linkage intact) but no runtime executes the bytecode — FO3 quest progression, terminal/door scripts, trap triggers, and magic-effect callbacks are inert. Static rendering, equip, animation, and FaceGen are unaffected.
- **Suggested Fix**: Not an audit defect. Closure is the SCDA-disassembler → Obscript-recognizer phase feeding the existing `translate::ScriptSource::Obscript` arm (M47.2). Track as a milestone deliverable.

### LOW (3)

#### D6-1: One FO3 trigger mesh is v20.0.0.4/bsver=11 → routed as Oblivion (#1219 ambiguity sample)
- **Severity**: LOW
- **Dimension**: 6 — BSA + Real Data
- **Location**: NifVariant detection; sample file `meshes\triggers\collisionboxstatic.nif` in `Fallout - Meshes.bsa`
- **Status**: NEW
- **Description**: The single FO3 mesh `collisionboxstatic.nif` is authored at `v20.0.0.4 / uv=11 / bsver=11` and trips the documented #1219 `NifVariant` ambiguity — it is routed as Oblivion rather than FO3. It parses 100% clean (it is an invisible collision/trigger box with no renderable geometry), so there is no functional impact today. FO3 version histogram: 10 961 @ bsver=34, 18 @ bsver=21, 8 @ bsver=14, 1 @ bsver=26, 1 @ bsver=11.
- **Impact**: None today (invisible trigger box). Flagged because this is the exact real-content sample #1219 was waiting on to validate flipping the variant-routing heuristic.
- **Suggested Fix**: Use as the regression fixture when #1219's variant disambiguation lands; no standalone fix warranted now.

#### D3-1: SCPT SCHR `flags` never extracted on FO3 (u32 read where field is u16)
- **Severity**: LOW
- **Dimension**: 3 — ESM Records
- **Location**: `crates/plugin/src/esm/records/script.rs:118-138`
- **Status**: NEW
- **Description**: The parser reads the trailing SCHR `flags` as a `u32` gated on `len >= 20`, but the FO3/FNV/TES5 SCHR is exactly 20 bytes with a **u16** flag. After the 16-byte prefix + `type` u16 (offset 18), only 2 bytes remain, so the strict `r.u32()` fails and `unwrap_or(0)` sets flags to 0 for every FO3 script. Verified three ways: openmw `esm4/script.hpp` (`type u16, flag u16`); raw byte-scan (`SCHR histogram {20: 57730}` — all 20 bytes); parsed-index probe (1 257 scripts, `nonzero_flags=0`). The inline comment claiming "FO3+ stores u32 tail" is itself wrong.
- **Impact**: Low — `ScriptRecord.flags` is parsed-but-unused (no downstream consumer). All other SCHR fields decode correctly.
- **Suggested Fix**: Read `r.u16()` instead of `r.u32()`; correct the comment.

#### D4-1: Worldspace selection runs preferred-default list before grid-containment
- **Severity**: LOW
- **Dimension**: 4 — Cell Loading
- **Location**: `byroredux/src/cell_loader/exterior.rs:100-148`
- **Status**: NEW
- **Description**: Worldspace selection runs the preferred-default list (step 2) before grid-coord containment (step 3). With `Fallout3.esm` + a DLC master loaded, `--grid X,Y` targeting an FO3 DLC worldspace (Anchorage, Point Lookout, Zeta, etc.) without `--wrld` silently lands on Capital Wasteland. Base-game-only loads are unaffected; `--wrld` is a workaround. FNV-shared.
- **Impact**: Low — affects only multi-master DLC exterior loads invoked via `--grid` without `--wrld`.
- **Suggested Fix**: Run grid-containment before the preferred list when `--grid` is supplied.

## FNV-Shared Surface (inherited coverage, verified on FO3)

- **NIF v20.2.0.7 parser** — `BSShaderPPLightingProperty`/`NoLighting` field order + version gates match nif.xml exactly (refraction `>14`, parallax `>24` per #774, NoLighting falloff `>26` per #1331). `BSSegmentedTriShape` (biped parts) is dispatched to `NiTriShape::parse_segmented` and the 9-byte/segment table is consumed structurally, not skipped. 96 distinct block types, all dispatched, 0 unknown.
- **Typed particle emitters** — `NiPSysEmitter`/`Ctlr`/`CtlrData`/`GrowFadeModifier` resolve identically for FO3 and FNV at bsver 34; `extract_emitter_params`/`extract_emitter_rate` operate on already-decoded typed structs (game-agnostic). The flagged "UNVERIFIED FO3 gap" is resolved by version-gate reasoning — `parse_grow_fade_modifier` base_scale gate (`version==V20_2_0_7 && bsver>=34`) matches nif.xml exactly; FO3 dev-band (<34) correctly yields `None`→1.0.
- **Inline shader render path** — single-u32 flag layout (`fo3nv_f1`/`fo3nv_f2`) distinct from Skyrim/FO4; normal map read from dedicated `BSShaderTextureSet` slot 1 (not the Oblivion bump-slot); NoLighting → `MATERIAL_KIND_NO_LIGHTING` fullbright with decal flags honored. `EmissiveSource::Material` for FO3; `MAT_FLAG_PBR_BSDF` provably 0 (gated on `mesh.is_pbr`, hardcoded `false` for FO3 NiTriShape). Single NIFAL boundary (`translate_material`) emits plain finite f32 metalness/roughness.
- **Collision** — FO3 `bhkCollisionObject`→`BhkRigidBody`→shape classifies as `CollisionAuthoring::Classic`; havok scale `Fallout3 → 7.0`. #1277 `BhkMultiSphereShape`+`BhkConvexListShape` translate. Full dispatch↔resolve parity, CI-enforced by `every_dispatched_bhk_shape_has_resolve_arm`. No FO3-reachable concrete shape skips via block_size.
- **ESM** — XCLL/RCLR shared 40-byte Fallout-era layout; DIAL/INFO TRDT (`response_number`@12, `emotion_type`@0) correct for all FO3 forms; WATR 196-byte DATA (53 records); XATO/XTNM/XTXR dispatch clean (0 `unreachable_patterns`, but vanilla FO3 ships 0 such overrides — path inert).
- **Cell loading** — interior CLI shared, live log reports real `cell.references.len()` (Megaton 929; 0 grep hits for stale 1609). Exterior wired and game-agnostic; Capital Wasteland fully record-driven. `NifImportRegistry` Arc cache keyed on lowercased path, LRU-bounded (`BYRO_NIF_CACHE_MAX`=2048), releases evicted clip handles in lockstep.
- **Animation/NPC** — B-spline FLT_MAX pose-fallback gate (#772) holds on every TRS axis; `AnimationClipRegistry` case-insensitive dedup (#790) collapses duplicates (no per-cell keyframe leak); NPC hand-mesh load (#793) returns `[upperbody, lefthand, righthand]` for `Fallout3NV` and FO3 `skeleton.nif` resolves.
