## Executive Summary

FNV is the reference title — the most-validated end-to-end path and the reference
realization for the canonical translation layers (NIFAL material/physics, PHYSAL
ragdoll). This audit found **no regressions and no defects above LOW severity**.
Every high-blast-radius checklist premise — cell load + unload hygiene, NIFAL
single-boundary material translation, RT lighting gates, ESM record coverage, NIF
parse rate, animation/particle/skin pipeline, and the newly-shipped PHYSAL ragdoll
slice — was verified correct against current code, with the cited prior fixes present
and regression-tested.

A **live bench ran on the real GPU** and the engine is materially *faster* than the
(201-commit-stale) ROADMAP record: **87.6 FPS / 11.41 ms / fence=9.02 / 3624 ent /
1224 draws** vs the recorded 76.2 / 13.11 / fence=11.12 / 3516 / 1224. No GPU-side
regression; the favorable delta is a stale-baseline artifact (D8-01). NIF parse holds
at **14881/14881 clean (100.00%)**.

**Totals: 10 findings — 0 CRITICAL, 0 HIGH, 0 MEDIUM, 10 LOW.** Most LOW items are
documentation drift or out-of-FNV-scope cross-game notes; the substantive ones are a
narrow FO4-only clip-leak (D1-01) and two diagnostic/telemetry gaps (D7-01, D8-02).

### LOW

#### D1-01: FO4 precombined NIF-cache insert discards `#[must_use]` evicted clip handles
- **Severity**: LOW · **Dimension**: Cell Loading
- **Location**: `byroredux/src/cell_loader/precombined.rs:220-223`
- **Status**: NEW
- **Description**: `NifImportRegistry::insert` is `#[must_use]` and returns the clip
  handles of LRU-evicted cache entries, which must be forwarded to
  `AnimationClipRegistry::release` (#863 contract). Every other production insert site
  honours this (`references.rs`, `partial.rs`, `world_setup.rs`); the precombined path
  binds the return to `_freed` and drops it.
- **Evidence**: `let _freed = reg.insert(path.clone(), parsed.clone());` — contrast the
  `partial.rs:146-167` capture-and-release block.
- **Impact**: One keyframe-array set leaks per evicted clip-bearing entry. Reachable only
  on FO4+ cells (precombines don't exist pre-FO4) with `BYRO_NIF_CACHE_MAX > 0` (default
  2048) when the insert crosses the cap. Bounded, slow, FO4-only — out of strict FNV
  scope, reported as a #863-contract regression on the one site that skips it.
- **Suggested Fix**: Mirror `partial.rs:155-166` — capture the freed vec and release each
  handle.

#### D1-02: NifImportRegistry default-cap doc rot (comments claim `=0` unlimited is the default)
- **Severity**: LOW · **Dimension**: Cell Loading (docs)
- **Location**: `byroredux/src/cell_loader/nif_import_registry.rs:277-279`; `byroredux/src/cell_loader/partial.rs:153-154`
- **Status**: NEW
- **Description**: `new()` defaults the LRU cap to **2048** (`.unwrap_or(2048)`), but two
  doc comments describe `BYRO_NIF_CACHE_MAX=0` (unlimited) as "the default", inverting the
  expected eviction behaviour (eviction + clip-release IS live by default).
- **Impact**: Misleading comment only; code is correct.
- **Suggested Fix**: "No-op when `BYRO_NIF_CACHE_MAX=0` (explicit unlimited mode); active on
  the default cap of 2048."

#### D4-02: FO4/Skyrim prebaked equip path skips TPLT-inherited inventory
- **Severity**: LOW · **Dimension**: ESM Parser / equip (cross-game)
- **Location**: `byroredux/src/npc_spawn.rs:290-331` (`build_npc_equip_state`) vs `:497-528,985-1016` (`spawn_npc_entity`)
- **Status**: NEW
- **Description**: The kf-era path (`spawn_npc_entity`, used by FNV/FO3/Oblivion) resolves
  TPLT inventory inheritance via `resolve_inherited_inventory` → `effective_inventory`
  before LVLI expansion. The prebaked path (`build_npc_equip_state`, FO4/Starfield only)
  iterates `npc.inventory` directly and never resolves TPLT inheritance, so a FO4/Skyrim
  template NPC inheriting CNTO via TPLT would spawn without inherited gear.
- **Impact**: Out of FNV scope (FNV always routes through `spawn_npc_entity`). Noted for
  cross-game consistency; verify under FO4/Skyrim audit before changing.
- **Suggested Fix**: Have `build_npc_equip_state` call `resolve_inherited_inventory` before
  the CNTO loop.

#### D7-01: Ragdoll body/joint resolution drops silently with zero telemetry
- **Severity**: LOW · **Dimension**: PHYSAL Ragdoll
- **Location**: `byroredux/src/ragdoll.rs:81-122` (`template_from_imported`)
- **Status**: NEW (partial overlap with OPEN #1539, which covers the constraint-*kind* drop at the importer; this is the *name-resolution* drop at template build)
- **Description**: A `bhk` body whose `bone_name` fails to resolve (line 88-90 `continue`),
  or a constraint referencing a dropped body (line 109-111 `continue`), is silently removed
  with no `log`/`tracing` call anywhere in `ragdoll.rs` (grep-confirmed empty). On a real
  FNV creature/NPC skeleton with a bone-naming quirk, a partially-resolved chain activates
  with missing joints (it still passes the ≥2-body / ≥1-joint gate) as a fragmented blob,
  indistinguishable at runtime from a correct ragdoll.
- **Impact**: Diagnostic-only; no crash, no UB. A mismatch produces a wrong-looking ragdoll
  with no breadcrumb.
- **Suggested Fix**: `log::debug!`/`warn!` the count of dropped bodies/constraints with the
  offending bone names.

#### D7-02: `bhkRigidBody` (non-T) local translation/rotation read unconditionally
- **Severity**: LOW · **Dimension**: PHYSAL Ragdoll
- **Location**: `crates/nif/src/blocks/collision/rigid_body.rs` (T/non-T collapse); consumed at `byroredux/src/ragdoll.rs:190-191,287`
- **Status**: NEW
- **Description**: Both `bhkRigidBody` and `bhkRigidBodyT` map to one struct and the body-
  local transform is applied unconditionally. Per Havok semantics the local fields are only
  meaningful for the `T` variant; for non-T they should be identity. In authored FNV content
  the CK writes identity for non-T bodies, so the seed/writeback inverse cancels to a no-op
  — a latent defense-in-depth gap, not a demonstrable FNV bug (FNV ragdoll bodies are
  predominantly `bhkRigidBodyT`).
- **Impact**: None on vanilla FNV. A malformed/modded non-T body with a non-identity offset
  would be displaced.
- **Suggested Fix**: Defense-in-depth only — gate the local-offset on the `T` discriminator
  or assert identity for non-T at import. Not worth churning the shipped reference path
  absent a real failing asset.

#### D8-02: Missing `--bsa`/`--textures-bsa` only WARN-logs; scene silently loads near-empty
- **Severity**: LOW · **Dimension**: Real-Data Validation
- **Location**: `byroredux/src/asset_provider.rs:374-415` (`open_with_numeric_siblings`), call sites `:481,:488`
- **Status**: NEW
- **Description**: `open_with_numeric_siblings` passes the archive path verbatim to
  `Archive::open` with no CWD/`--esm`-folder fallback. On failure it WARN-logs and returns;
  the engine proceeds with the placeholder provider. With a wrong CWD and bare names all
  three archives fail → ~35 entities / 0 draws / spurious ~460 FPS. This is the documented
  CWD gotcha, behaving as designed, but the only signal is three WARN lines and the bench
  still prints a healthy-looking (meaningless) FPS.
- **Impact**: No correctness bug — intended soft-fail. Cost is diagnostic: the exact trap
  that produces false "renders empty" / "regression" reports.
- **Suggested Fix**: Defense-in-depth — when `--bench-frames` is set and an archive open
  fails, escalate that WARN to ERROR, or annotate near-empty/`draws=0` bench summaries as
  `(suspect: archives not opened — check CWD)`.
