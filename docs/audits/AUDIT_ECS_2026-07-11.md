## Per-Dimension Results

### 1. Lock Ordering & Deadlock — CLEAN
- Every `query` / `query_mut` / `resource` / `resource_mut` / `get` /
  `try_resource*` site in `world.rs` arms a `TrackedRead` / `TrackedWrite` scope,
  `defuse()`s it only **after** the real lock is acquired, and the wrapper's
  `Drop` (`query.rs` / `resource.rs`) untracks. `World::get` (world.rs:283) keeps
  the scope armed on the `None` branch (no double-untrack).
- `query_2_mut` / `query_2_mut_mut` / `resource_2_mut` acquire in `id_a < id_b`
  TypeId order **and** arm the tracker scopes in the same order via the explicit
  `if id_a < id_b { … } else { … }` branches (world.rs:457, 521, 695) — the #313
  fix; scopes are not armed in generic-parameter order.
- Same-type double-lock: `assert_ne!(A, B)` present on all three paired APIs
  (world.rs:440, 508, 672) plus `try_resource_2_mut` (world.rs:796).
- `try_resource_2_mut` does BOTH existence checks (`contains_key`) before
  acquiring EITHER lock (world.rs:803–807) — #465 partial-acquire window closed.
- Poison resolution: every acquisition resolves `PoisonError` through
  `storage_lock_poisoned::<T>()` / `storage_lock_poisoned_erased()` /
  `resource_lock_poisoned::<R>()`. `despawn` (world.rs:132) and `clear_entities`
  (world.rs:235) use the type-erased variant fed by the `type_names` side-table
  (#466); `insert_resource` (world.rs:573) re-panics loud on a poisoned prior
  value rather than swallowing it into `None` (#1837).
- Cross-thread ABBA graph (`lock_tracker::global_order`) is `cfg(debug_assertions)`
  gated; the same-thread reentrancy check runs in both debug and release.

### 2. Storage Correctness — CLEAN
- `SparseSetStorage` (sparse_set.rs): swap-remove fixes the moved entity's sparse
  pointer (`self.sparse[moved_entity] = Some(dense_idx)`, line 107); last-element
  removal takes the no-swap path (line 92); insert into an existing entity
  overwrites in place (line 66, len unchanged). `structural_gen` bumps on insert
  (incl. overwrite/reparent), remove, and clear — const-false no-op for
  non-tracked components.
- `PackedStorage` (packed.rs): `binary_search` maintains the sorted invariant on
  every insert/remove; `insert_bulk` (line 152) uses append + single stable
  index-sort + last-wins dedup (#467). `drain_dirty_into` clears `out` then
  drains while preserving `self.dirty` capacity (#1371); `take_dirty` hands the
  allocation away (documented). `mark_dirty` fires on insert / remove / `get_mut`
  / `iter_mut`.

### 3. Query Borrow Safety — CLEAN
- `QueryRead` / `QueryWrite` / `ComponentRef` (query.rs) each hold their
  `RwLock*Guard` for the wrapper lifetime (`#[allow(dead_code)] guard`) and cache
  the raw-pointer downcast once in `new()` (#1367). All three `unsafe { &*… }` /
  `&mut *…` derefs (query.rs:64, 135, 143, 289) carry a SAFETY comment tying
  validity to the still-held guard.
- `ComponentRef` retains the guard rather than returning a raw pointer to dropped
  storage — the sound replacement for the unsound #35 pattern.
- `QueryWrite::DerefMut` routes through `storage_mut()` on `&mut self`, so the
  borrow checker forbids a simultaneous `&`/`&mut` into the same storage.
- `query` / `query_mut` return `None` for never-created storage (no lazy
  empty-storage creation on the read path).

### 4. Resource Lifetimes — CLEAN
- `resource()` / `resource_mut()` panic with the type name on a missing resource
  (world.rs:609, 638); `try_resource*` return `None`.
- `ResourceRead` / `ResourceWrite` (resource.rs) downcast through the guard on
  each `Deref` (NOT cached — documented as not the #1367 hot path); the `expect`
  can't fire because `TypeId` keys the map.
- `insert_resource` returns the prior value; `remove_resource` and
  `insert_resource` both resolve poison via `resource_lock_poisoned`.

### 5 / 5b. Scheduler & Access Declarations — CLEAN
- `Stage` enum (scheduler.rs:27) is `Early=0 → Update=1 → PostUpdate=2 →
  Physics=3 → Late=4`, iterated via `BTreeMap` `Ord`. No `ParallelUpdate` /
  `LateExclusive` stage.
- Fail-fast panic policy documented (scheduler.rs:425–453, TS-08/#1412); no
  `catch_unwind` recovery added. `run` takes `&mut self`; `Scheduler` is not a
  `Resource`.
- `AccessConflict` (access.rs:139) has exactly three variants — `None`,
  `Unknown { left_undeclared, right_undeclared }`, `Conflict { pairs }`; no
  `Parallel` variant (#1521). `analyze_pair` returns `Unknown` when either side
  is undeclared.
- Boot guard (`byroredux/src/boot.rs:820–843`, moved out of `main.rs`) runs
  `debug_assert_eq!` on `undeclared_parallel_count()`, `known_conflict_count()`,
  AND `unknown_pair_count()` == 0 (strengthened by #1602 beyond the original
  #1394 single-counter check). All parallel-batch systems register via
  `add_to_with_access` (boot.rs:434+).

### 6. Unsafe Code Review — CLEAN
- The only `unsafe` in the ECS core is the four cached-pointer derefs in
  `query.rs` (`QueryRead::storage`, `QueryWrite::storage`/`storage_mut`,
  `ComponentRef::Deref`) — all #1367, all with SAFETY comments. No new unsafe
  block introduced.
- `World::spawn` uses `checked_add` and panics on `EntityId` overflow (#36);
  `despawn` does not reclaim IDs (#372) — documented, correct.

### 7. Component Lifecycles — CLEAN
- The two newer core resources (`debug_load.rs` `PendingDebugLoadSlot`,
  `game_profiles.rs` `GameProfileRegistry`, `metrics.rs` `MetricsSnapshot`) are
  pure-data resources with FIFO-drain / read-only accessors — no World mutation,
  no orphan-component risk, no unsound patterns.
- Prior audits' lifecycle guards (streaming orphan invariant, NPC-spawn
  idempotency, transient-marker single-frame lifetime, `AnimationClipRegistry`
  case-folded dedup #790) were confirmed unchanged since 2026-07-06.

### 8. Hot-Path Performance Invariants — CLEAN
- `make_transform_propagation_system` (systems.rs:41): keys the cached `roots`
  set on `(Transform::len(), Parent-len, next_entity_id)` AND tracks
  `Parent`/`Children` `structural_generation()` plus the drained `Transform`
  dirty set. Fast path skips the whole BFS when the dirty set is empty and full
  state is unchanged (systems.rs:120). Uses `drain_dirty_into(&mut transform_dirty)`
  to keep scratch capacity across frames (#1371), NOT `take_dirty`. Incremental
  path dedups the dirty list and reseeds only moved subtrees.
- `lock_tracker` held-set collection (`held_others: Vec`) is
  `#[cfg(debug_assertions)]`-gated in both `track_read` (line 83) and
  `track_write` (line 125) — release builds skip the alloc (#823).

### 9. NIFAL Canonical Material — CLEAN
- `Material` (components/material.rs) carries `metalness: f32` / `roughness: f32`
  as fully-resolved plain fields (not `Option`); defaults `0.0` / `0.5` match the
  renderer's no-Material fallback.
- `Material::resolve_pbr` (line 686) is idempotent, preserves upstream translator
  values, fills only the NaN-sentinel slot, and clamps to `[0,1]` / `[0.04,1]` —
  pinned by `resolve_pbr_is_idempotent`,
  `resolve_pbr_preserves_upstream_translator_values`,
  `resolve_pbr_fills_only_missing_slot`, `resolve_pbr_clamps_authored_out_of_range`.
- **#1925 verified**: the scrap arm (line 482) matches `metalscrap` specifically,
  not the bare `scrap` substring, and is checked BEFORE the `metal` arm so
  megaton painted-tin cladding stays matte (metalness 0.0) while genuine
  `scrapmetal`-token clutter reaches the conductive-metal arm. Pinned by
  `classify_pbr_bare_scrap_reaches_metal_arm` and
  `classify_pbr_scrap_metal_is_not_chrome`.
