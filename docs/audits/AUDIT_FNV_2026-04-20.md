# Fallout New Vegas Compatibility Audit — 2026-04-20

Audit of ByroRedux readiness for Fallout: New Vegas content across six
dimensions: NIF parser regression guard, ESM record coverage, cell loading,
RT lighting pipeline, real-data validation, and animation+skinning.

FNV is the **reference title** — the most validated end-to-end path in the
engine. Regressions against roadmap baselines are Critical by default.

---

## Executive Summary

**Compatibility level**: Tier-1 reference — no regressions against the M24
Phase 1 or N23.4 baselines. 14,881 / 14,881 vanilla NIFs parse at 100%;
FalloutNV.esm produces 13,684 structured records matching the documented
baseline exactly; every named regression guard (#108, #119, #148, #149, #354,
#355, #400, #429) verified intact. Interior + exterior cells load end-to-end.
RT pipeline (M22 + M31 + M31.5 + M36 + M37 + M37.5) is wired correctly —
TAA is actually in the pipeline (despite a stale "once TAA is wired" doc
comment in composite.rs).

**Two previously-open medium findings** (#376 CONT flags, #377 NPC
disposition_base i8) remain unresolved — both one-line fixes.

**Eight new findings** surfaced, led by:
1. (HIGH) WTHR cloud textures resolved without `textures\` prefix → every
   FNV exterior renders with clouds disabled silently.
2. (HIGH) `AnimationClip.weight` loaded from NIF but never consumed in
   sampling — per-sequence default weights silently dropped.
3. (MEDIUM) Exterior terrain uses only quadrant[0].base; ATXT/VTXT splat
   layers parsed but unused → all roads/grass look like solid dirt.
4. (MEDIUM) `EnableParent::default_disabled()` over-hides every XESP-gated
   REFR regardless of parent state.
5. (MEDIUM) Caustic scatter splats AFTER SVGF but BEFORE TAA — neighborhood
   clamp will flicker caustic energy.

## Dimension 2 — ESM Record Coverage

**Verified**: record counts match M24 baseline exactly. HEDR=1.34 → Fallout3NV
via pinned test (`reader.rs:724-728`). No `unreachable_patterns` warnings.

**FNV-2-M1** `crates/plugin/src/esm/records/actor.rs:50,172`. NPC ACBS
disposition_base declared `u8`, read as raw byte. UESP spec is `i8`.
Observable: **0 NPCs report negative disposition out of 3,816** — vanilla
ships Legion, Fiends, Powder Gangers with negative base disposition.
Combat-trigger logic collapses silently. Prior art: #377 (unresolved).

**FNV-2-M2** `crates/plugin/src/esm/records/container.rs:86-89`. FNV CONT
DATA is 5 bytes (`f32 weight + u8 flags`), parser reads only the f32.
`ContainerRecord` has no `flags` field. Respawn / showOwner semantics lost.
Prior art: #376 (unresolved).

**FNV-2-L1** `crates/plugin/src/esm/records/actor.rs:280-282`. FACT DATA
reads u32 into `record.flags`; FNV DATA is 1 byte. `NCRFactionNV` reports
`flags=0x00000100` (high byte is padding/neighbor bytes). Only bit 0..2
authoritative today.

**FNV-2-L2** `crates/plugin/src/esm/records/actor.rs:288`. FACT XNAM reads
`combat_reaction` as `u8` but the field is `u32`. Behaviorally correct
(values 0-3 fit) but width-mismatched.

**FNV-2-L3** `crates/plugin/src/esm/cell.rs:587`. Tautological
`d.len() >= 20` inside outer `>= 28` block. Dead conditional, no functional
bug.

**Info**: #378 (unreachable_patterns at cell.rs:211) appears resolved — no
warning in release build. FO4 WEAP DATA mis-bucketing is tracked in
FO3/FO4 audit, not here. Varmint Rifle FormID in audit brief was wrong
(`0x0007EA24`, not `0x000086A8`).

## Dimension 3 — Cell Loading End-to-End

**Verified**: `CachedNifImport` Arc cache process-lifetime, correct hit rate.
`CellLoadResult.weather` populated via WRLD.CNAM → CLMT → WLST (max-chance)
→ WTHR. `wastelandnv` worldspace priority first. `unload_cell` teardown
ordering correct (BLAS before mesh, MAX_FRAMES_IN_FLIGHT overlap). No
missing-optional early-exits.

**FNV-3-H1** `byroredux/src/scene.rs:166-204`. `wthr.cloud_textures[0]` is
textures-root-relative (e.g. `"sky\cloudsnoon.dds"`). Passed directly to
`tex_provider.extract(path)`, which requires `textures\` prefix per
`archive.rs:487 normalize_path`. Lookup always misses →
`cloud_tile_scale = 0.0` → every FNV exterior cloud path dead end-to-end.
Contrast: `cell_loader.rs:700` correctly prepends `textures\landscape\`
before `resolve_texture`.

**FNV-3-M1** `byroredux/src/cell_loader.rs:694-716`. `spawn_terrain_mesh`
does `quadrants.iter().find_map(|q| q.base)` — one texture per 4096-unit
tile. LAND parser correctly decodes per-quadrant BTXT + all ATXT+VTXT
splat layers with 17×17 alpha grids; `TerrainTextureLayer::alpha` never
read. Every FNV road/grass tile needs ≥2 layers.

**FNV-3-M2** `byroredux/src/cell_loader.rs:796-801`. `EnableParent::
default_disabled()` returns true for any non-zero non-inverted parent.
XESP semantics need the **parent's** initial-disabled flag (REFR bit
0x0800). Most XESP chains link to persistent enabled actors — child
should render. #349 closed on a wrong assumption; vanilla FNV disproves
the "default disabled" common case.

**FNV-3-L1** `crates/plugin/src/esm/records/climate.rs:64-83`. WLST chance
parsed as `u32`; doc says `i32`. Negative chance wraps to huge positive,
wins `max_by_key`. Comment at line 65 says "i32"; code reads u32.

**FNV-3-L2** `byroredux/src/cell_loader.rs:1003-1009`. `mesh_count` =
`reg.len().saturating_sub(cache_size_at_entry)` → cache misses, not
spawned meshes. 100% cache-hit repeat loads report `mesh_count = 0`
despite spawning hundreds of entities.

**FNV-3-L3** `crates/plugin/src/esm/records/climate.rs:27,87`.
`climate.sun_texture` stored but no renderer consumer. Usable once
FNV-3-H1 is fixed.

## Dimension 4 — RT Lighting Pipeline

**Verified**: RIS M=8 unbiased W at `triangle.frag:851-967`, 64× clamp
engaged. TAA dispatched after SVGF, before composite
(`context/draw.rs:1032-1040`). Composite rebinds HDR to TAA output at
init (`context/mod.rs:792`) and resize (`context/resize.rs:347`).
ACES tone-maps once (`composite.frag:155`/`:202`). SVGF uses mesh_id
disocclusion + bilinear × match weight reprojection
(`svgf_temporal.comp:12-13,81`). TLAS cull conservative. BLAS compaction
query pool + readback at `acceleration.rs:684-820`. BLAS LRU budget =
VRAM/3 with 256 MB floor.

**FNV-REN-M1** `crates/renderer/src/vulkan/composite.rs:967-973`. Stale
doc comment: "Used to switch composite's input from raw HDR to TAA
output **once TAA is wired up**." TAA is already wired — two live callers
pass `taa_views`. A reader debugging tone-map or temporal lag will chase
a nonexistent bug.

**FNV-REN-M2** `crates/renderer/src/vulkan/context/draw.rs:1013-1030`.
Caustic scatter (#321) runs AFTER SVGF temporal but BEFORE TAA. Caustic
splats enter TAA history; YCoCg variance clamp rejects high-energy
sparse samples → caustic flickers on high-roughness surfaces. Fix: move
caustic AFTER TAA or tag pixels so TAA bypasses clamp.

**FNV-REN-L1** `crates/renderer/src/vulkan/context/draw.rs:1032-1040`.
TAA dispatch failure only logged as `warn!`; composite still samples
whatever was in the descriptor from last successful dispatch. Permanent
failure (lost device, descriptor-pool exhaustion) → stale frozen HDR
without user indication. Same pattern in SVGF + caustic.

**FNV-REN-L2** `crates/renderer/shaders/triangle.frag:411`. Comment
fragment ends mid-sentence. Low priority but indicates a merge hiccup.

Related open issues referenced: #280-284 (Vulkan sync), #270 (shadow
budget), #272 (draw batch), #321 (caustic), #451/#452/#453 (FO3 env map
/ parallax — renderer gaps).

## Dimension 6 — Animation & Skinning

**Verified**: full samplers reachable, FixedString interning at clip-load
(#340), zero-alloc per-frame key lookup (#240), SLERP + log-space TBC
Hermite for rotation, text-key event visitor (#339), ping-pong
`CycleType::Reverse` correctly reflects at both endpoints, skinning
end-to-end (#178 partial scope). KFM binary parser present at
`crates/nif/src/kfm.rs` for v1.2–2.2.

**FNV-AN-H1** `crates/core/src/animation/stack.rs`,
`crates/core/src/animation/types.rs:140`,
`byroredux/src/anim_convert.rs:214`. `AnimationClip.weight` loaded from
`NiControllerSequence.weight` but `grep clip.weight` returns zero hits
workspace-wide. `stack.rs::sample_blended_transform` uses only
`layer.effective_weight()`, never combines with `clip.weight`. Per-sequence
default weights from KF data silently dropped — all clips behave as if
weight=1.0. Fix: multiply `ew *= clip.weight` at the two weight-gathering
sites (6 lines). Same gap in single-player path.

**FNV-AN-M1** Duplicate of #338 — NiControllerManager state machine.
KFM parser present but `parse_kfm` has zero call sites. By design per
module doc; noted for roadmap alignment.

**FNV-AN-M2** `ControlledBlock` records start/stop times per channel but
`TransformChannel` stores only keys. `advance_stack` assumes layer fills
its clip's full `[0, duration]`. Partial-range sub-clips (shared NIF with
multiple sequences) not expressible. Not triggered by bundled FNV
`_male.kf` but blocks accurate actor idle chains.

**FNV-AN-L1** `crates/core/src/animation/interpolation.rs:239`.
`KeyType::Quadratic` rotation falls back to SLERP — comment acknowledges
"RotationKey doesn't store forward/backward quats today." nif.xml's
`QuaternionKey` carries them. Small impact — Bethesda KFs mostly Linear
+ TBC.

**FNV-AN-L2** `AnimationPlayer::reverse_direction` not serialized for
debug snapshots. Ping-pong replay from save would reset to forward. Not
user-visible today.
