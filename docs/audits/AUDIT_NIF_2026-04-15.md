## Findings

### CRITICAL

_None._

### HIGH

#### H1 ‚Äî `NiPSysData` over-reads corrupt particle system blocks (D3-02)
- **Dimension**: Stream Position
- **Game Affected**: FNV (primary), FO3, likely Skyrim LE/SE
- **Location**: `crates/nif/src/blocks/` (`NiPSysData` parser)
- **Status**: NEW
- **Evidence**: 559 of the 572 total over-reads in FNV corpus (`/tmp/audit/nif/fnv_stderr.log`). Observed `expected 109 B, consumed 267`, `expected 301 B, consumed 14133`. Plus 692 parse failures. 1,252 broken NiPSysData parses ‚âà 7.3% of NiPSysData population.
- **Impact**: Particle emitter vertex/key data is junk for affected blocks ‚Üí wrong geometry/colors/lifetimes on FNV/FO3 muzzle flashes, explosions, ambient dust. Guardrail prevents downstream corruption but does not save the NiPSysData block itself.
- **Suggested Fix**: Re-audit vs Gamebryo 2.3 `CoreLibs/NiParticle/NiPSysData.cpp` + nif.xml (watch for nif.xml divergence; cf. NiTexturingProperty / #149). Check subtexture offsets, rotation-angle/axis arrays, and `data_size`/`data_size_2` split. Promote `consumed > size` from WARN to ERROR on FO3+ files in `nif_stats`.

### MEDIUM

#### M1 ‚Äî `NiMaterialProperty` consistent `-20 B` under-read (D3-04)
- **Dimension**: Stream Position
- **Game Affected**: Fallout NV (likely FO3, possibly FO4 compact-material files)
- **Location**: [properties.rs:36-86](crates/nif/src/blocks/properties.rs#L36-L86)
- **Status**: NEW
- **Evidence**: 24 blocks stable `expected 68, consumed 48`.
- **Impact**: `compact_material()` gate at `properties.rs:44` returns true when it shouldn't ‚Äî ambient/diffuse default to `(0.5, 0.5, 0.5)` instead of file values. Affected meshes appear washed-out.
- **Suggested Fix**: Version-gate `compact_material()` more tightly (FO4+ BGSM-driven only, not Gamebryo-native FO3/FNV NiMaterialProperty). Dump `bs_version`/`user_version`/variant for failing blocks to confirm.

#### M2 ‚Äî Oblivion cascading-failure skip-table architecture (D5)
- **Dimension**: Coverage
- **Game Affected**: Oblivion (v20.0.0.5, no `block_sizes` table)
- **Location**: `crates/nif/src/lib.rs` parse loop + per-block parsers
- **Status**: NEW (architectural; related closed #104, #224)
- **Evidence**: `oblivion_stats.txt:20-21` shows `NiFloatData` KeyType error truncating 7 subsequent blocks on a single file. Any mis-read poisons downstream.
- **Impact**: Dispatch coverage metrics deceive ‚Äî 100% dispatch on a tiny Oblivion sample still produces cascading data loss. Every block-parser addition is a corpus regression risk.
- **Suggested Fix**: Pre-scan pass over raw NIF to build synthetic skip-table (name-indexed lookup of next valid block header), used as fallback when a parser errors on v20.0.0.5.

#### M3 ‚Äî SLSF2 `Decal` bit (`0x200`) never consulted on `BSLightingShaderProperty` (D4-04)
- **Dimension**: Import Pipeline
- **Game Affected**: Skyrim LE/SE, FO4, FO76, Starfield
- **Location**: [material.rs:273](crates/nif/src/import/material.rs#L273)
- **Status**: **Existing: #176** (OPEN)
- **Suggested Fix**: Add `shader.shader_flags_2 & 0x200 != 0` to `is_decal` in both NiTriShape and BsTriShape paths.

#### M4 ‚Äî `NiGeometryData` `Has UV` window too wide (N1-03)
- **Dimension**: Block Parsing
- **Game Affected**: Pre-Oblivion Gamebryo (Morrowind adjacent, v 4.0.0.3‚Äì20.0.0.3)
- **Location**: [tri_shape.rs:671-676](crates/nif/src/blocks/tri_shape.rs#L671-L676)
- **Status**: NEW
- **Evidence**: nif.xml gates `Has UV` on `until="4.0.0.2"`; parser reads it for `version < 20.0.0.4`.
- **Impact**: Breaks stated Morrowind aspirational compatibility; misaligns every UV-bearing block in the gap window.
- **Suggested Fix**: Gate `has_uv` read on `version <= NifVersion(0x04000002)` only.

### LOW (summarized)

Block parsing (Dim 1) ‚Äî version-gate drift vs nif.xml, all non-Bethesda:
- **N1-01** `NiGeometryData.Group ID` threshold `10.0.1.0` ‚Üí should be `10.1.0.114`. [tri_shape.rs:607-609](crates/nif/src/blocks/tri_shape.rs#L607-L609)
- **N1-02** `NiGeometryData.Keep/Compress Flags` threshold `10.0.1.0` ‚Üí should be `10.1.0.0`. [tri_shape.rs:611-614](crates/nif/src/blocks/tri_shape.rs#L611-L614)
- **N1-04** `NiAVObject` bounding-volume skip fires in `[4.2.2.1, 10.0.0.x]` where field doesn't exist. [base.rs:108-113](crates/nif/src/blocks/base.rs#L108-L113)
- **N1-05** `NiExtraData` subclass parsers call `read_string()` unconditionally (latent). [extra_data.rs:167 and 7 peers](crates/nif/src/blocks/extra_data.rs#L167)
- **N1-06** `NiExtraData` legacy `Next Extra Data` threshold uses `10.0.1.0`, nif.xml says `until="4.2.2.0"`. [extra_data.rs:44-46](crates/nif/src/blocks/extra_data.rs#L44-L46)

Stream position (Dim 3):
- **NIF-D3-05** Havok constraint parsers (`bhkLimitedHingeConstraint`, `bhkRagdollConstraint`, `bhkMalleableConstraint`) under-read by ~141 B (read header only, skip descriptor body). 1,116 FNV blocks. Safe until physics lands.

Import pipeline (Dim 4):
- **D4-01** `NiAVObject` flag mask `0x21` conflates `APP_CULLED (0x01)` with `DISPLAY_OBJECT (0x20)`. [walk.rs:126,158,253,290,373](crates/nif/src/import/walk.rs#L126). Use `& 0x01` only.
- **D4-02** `BSEffectShaderProperty` decal / two-sided / normal-map extraction missing. [material.rs:290-306](crates/nif/src/import/material.rs#L290-L306), [mesh.rs:277-335](crates/nif/src/import/mesh.rs#L277-L335).
- **D4-03** `BsTriShape` material path ignores ancestor `NiZBufferProperty`/`NiAlphaProperty`/`NiStencilProperty`. [mesh.rs:216-397](crates/nif/src/import/mesh.rs#L216-L397).
- **D4-05** `zup_matrix_to_yup_quat` fast-path gate too loose (`(det-1).abs() < 0.1`) ‚Üí non-unit quaternion drift. [coord.rs:33-42](crates/nif/src/import/coord.rs#L33-L42). Normalize or tighten gate.
- **D4-06** Shape hidden-bit check (`& 0x01`) inconsistent with node (`& 0x21`). Paired with D4-01.

### INFO / Verified Clean

- **D3-01** Parse-loop guardrail (`stream.set_position(start_pos + block_size)` on Ok/Err) works for FO3+; bugs don't cascade. [lib.rs:179-230](crates/nif/src/lib.rs#L179-L230).
- **D3-03** `NiTexturingProperty` 1-byte shortfall is NOT present; #149 revert is correct per Gamebryo 2.3 source. [properties.rs:277-319](crates/nif/src/blocks/properties.rs#L277-L319).
- **Dim 2** all nif.xml thresholds verified with regression tests: `20.1.0.1`, `20.2.0.5`, `5.0.0.1`, `10.0.1.8`, `4.1.0.1`, `4.2.2.0`; Oblivion pre-match guard; non-Bethesda bsver fallthrough (#160); Starfield detection (#173 open for `168-169` gap).
