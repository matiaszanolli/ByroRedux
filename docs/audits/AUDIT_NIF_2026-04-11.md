# NIF Parser Audit — 2026-04-11

**Scope**: 5-dimension deep audit of the NIF binary format parser across all supported
game variants (Oblivion → Starfield), covering block parsing, version handling,
stream position, import pipeline, and coverage gaps.

**Baseline**: [AUDIT_NIF_2026-04-07.md](AUDIT_NIF_2026-04-07.md) (39 prior findings)

**Dimensions executed**:
1. Block Parsing Correctness — field reads vs nif.xml / Gamebryo 2.3 source
2. Version Handling — variant detection + feature flags
3. Stream Position Integrity — drift, skip safety, block_size reconciliation
4. Import Pipeline Correctness — parsed blocks → ECS entities
5. Coverage Gaps — block types missing from dispatch

---

## Executive Summary

The NIF parser is in **strong shape overall**. The stream-position dimension returned
**zero new findings**; three prior issues from the 2026-04-07 audit are confirmed fixed
(`skip()` overflow check, particle emitter unknown-mode error, Oblivion `truncated` flag).
Base-class parsing, boolean type distinction (`read_bool` vs `read_byte_bool`), skinning
block byte-correctness, and the `.av.*` field refactor are all clean.

**Remaining weaknesses cluster in four areas**:

1. **FO76/Starfield material flags** — two feature-flag predicates
   ([version.rs:132–145](crates/nif/src/version.rs#L132-L145)) exclude FO76 and
   Starfield, silently corrupting NiMaterialProperty reads for any shipped asset in
   those games that uses the block.
2. **Property inheritance** — the importer only looks at a shape's own properties;
   alpha/material/two-sided flags attached to parent `NiNode`s are silently dropped,
   which is a classic Oblivion/FO3 pattern.
3. **Havok cascade on Oblivion** — `NiCollisionObject` and 10 Havok constraint types
   are skip-only. Oblivion has no `block_size`, so any mesh referencing them hard-fails.
4. **Unknown-block allocation** — the fallback path calls
   `stream.read_bytes(size as usize)` with no upper bound; a malformed file can
   trigger a 4 GB allocation.

**Coverage has improved dramatically** since 2026-04-07: per-game block-type coverage
is now ~95–98% across Oblivion/FO3/FNV/Skyrim/FO4/FO76/Starfield (up from ~70–85%).
Parse rate remains 100% across 177,286 NIFs.

## Findings — grouped by severity

### CRITICAL (3)

#### NIF-04-11-C1: `compact_material()` & `has_emissive_mult()` exclude FO76/Starfield
- **Severity**: CRITICAL
- **Dimension**: Version Handling
- **Game Affected**: Fallout 76, Starfield
- **Location**: [crates/nif/src/version.rs:132–145](crates/nif/src/version.rs#L132-L145)
- **Status**: Existing: #169
- **Description**: Both feature predicates explicitly omit `Fallout76` and `Starfield`
  from the `matches!` arm. Per nif.xml, NiMaterialProperty uses the compact layout
  (no ambient/diffuse) and carries an emissive multiplier for every BSVER ≥ 27, which
  covers both games.
- **Evidence**:
  ```rust
  pub fn compact_material(self) -> bool {
      matches!(self, Self::Fallout3 | Self::FalloutNV | Self::SkyrimLE
               | Self::SkyrimSE | Self::Fallout4)
      // ❌ missing Fallout76, Starfield
  }
  ```
- **Impact**: Any FO76/Starfield NIF with NiMaterialProperty reads wrong byte layout,
  producing washed-out or over-bright material colors. Stream may still recover via
  block_size, but color values are wrong.
- **Suggested Fix**: Add `Self::Fallout76 | Self::Starfield` to both arms.

#### NIF-04-11-C2: Starfield BSVER 168–169 silently classified as Fallout76
- **Severity**: CRITICAL (edge case)
- **Dimension**: Version Handling
- **Game Affected**: Starfield
- **Location**: [crates/nif/src/version.rs:99–108](crates/nif/src/version.rs#L99-L108)
- **Status**: Existing: #173
- **Description**: The variant detection uses `uv2 < 170 => Fallout76`, so BSVER 168
  and 169 (transitional/test builds, DLC early-access, mod tools) are silently routed
  to FO76. Masked today because both games use FO76-style shader flags.
- **Suggested Fix**: Change the final arm to an explicit `uv2 >= 170` guard.

#### NIF-04-11-C3: Havok skip-only blocks cascade-fail on Oblivion
- **Severity**: CRITICAL (Oblivion), HIGH (FO3+ data loss)
- **Dimension**: Coverage
- **Game Affected**: All (Oblivion fatal, FO3+ silent)
- **Location**: [crates/nif/src/blocks/mod.rs:520–543](crates/nif/src/blocks/mod.rs#L520-L543)
- **Status**: Existing: #117, #124, #125
- **Description**: 11 Havok block types are skip-only: `NiCollisionObject` (base),
  `bhkNPCollisionObject` (FO4), and 7 constraint types (Malleable, Ragdoll, LimitedHinge,
  Hinge, BallAndSocket, StiffSpring, Prismatic) plus `bhkPhysicsSystem` / `bhkRagdollSystem`.
  On Oblivion (no block_size), any mesh referencing these types aborts the entire parse.
  On FO4+, the bytes are preserved in NiUnknown but physics parameters are lost.
- **Impact**:
  - Oblivion: collision-enabled meshes that use `NiCollisionObject` as a direct block fail to load.
  - FO4: `bhkNPCollisionObject` is the FO4 collision root for many items; all its physics data is silently dropped (#124).
- **Suggested Fix**: Implement stub parsers that read the NiAVObject base + trailing
  bytes and return NiUnknown. Full physics parsing deferred to M28.

### HIGH (3)

#### NIF-04-11-H1: Property inheritance from parent `NiNode`s is not applied
- **Severity**: HIGH
- **Dimension**: Import Pipeline
- **Game Affected**: All (most impactful on Oblivion/FO3/FNV)
- **Location**: [crates/nif/src/import/walk.rs](crates/nif/src/import/walk.rs), [crates/nif/src/import/material.rs](crates/nif/src/import/material.rs)
- **Status**: Existing: #208
- **Description**: The importer reads material/alpha/stencil properties only from
  `shape.av.properties`. In the Gamebryo scene-graph model, properties attached to a
  parent `NiNode` are inherited by descendant geometry via a first-match walk. Any
  NiAlphaProperty or NiTexturingProperty placed on a NiNode is currently lost.
- **Impact**: Architectures, furniture, and creature meshes that factor out properties
  onto parent nodes render with wrong blend mode, wrong two-sided state, or no alpha
  at all. Classic Oblivion pattern.
- **Suggested Fix**: When extracting properties for a shape, fall back to walking
  `parent_node` up the hierarchy on the first miss per property type.

#### NIF-04-11-H2: `has_bs_stream_header` gating condition broader than nif.xml spec
- **Severity**: HIGH
- **Dimension**: Version Handling
- **Game Affected**: NetImmerse 10.0.1.3–10.0.1.7
- **Location**: [crates/nif/src/header.rs:99–117](crates/nif/src/header.rs#L99-L117)
- **Status**: Existing: #170
- **Description**: Current code:
  `has_bs_stream_header = version == 0x0A000102 || user_version >= 3`. Per nif.xml, the
  correct rule is `v == 10.0.1.2 OR (v >= 10.0.1.3 AND user_version >= 3)`. The current
  predicate triggers on any v >= 10.0.1.3 with uv >= 3, but also misses v10.0.1.3–10.0.1.7
  with uv < 3. Low practical impact because Oblivion+ is all v10.0.1.8+.
- **Suggested Fix**:
  ```rust
  let has_bs_stream_header = version == NifVersion(0x0A000102)
      || (version >= NifVersion(0x0A000103) && user_version >= 3);
  ```

#### NIF-04-11-H3: Unknown-block fallback allocates without an upper bound
- **Severity**: HIGH (safety / DoS surface)
- **Dimension**: Coverage / Safety
- **Game Affected**: All
- **Location**: [crates/nif/src/blocks/mod.rs:548](crates/nif/src/blocks/mod.rs#L548)
- **Status**: Existing: #113
- **Description**: The unknown-block fallback calls `stream.read_bytes(size as usize)?`
  with `size` taken directly from the header. A corrupted or malicious NIF with
  `block_size = 0xFFFFFFFF` triggers a 4 GB heap allocation. No upper bound is
  enforced.
- **Impact**: OOM crash on untrusted input. DoS vector if NIF loading ever accepts
  network-delivered files.
- **Suggested Fix**: Cap at e.g. 16 MB per unknown block and return
  `InvalidData` if exceeded.

### MEDIUM (7)

#### NIF-04-11-M1: `BsTriShape` two-sided flag ignores `BSEffectShaderProperty`
- **Severity**: MEDIUM
- **Dimension**: Import Pipeline
- **Game Affected**: Skyrim SE+, FO4, FO76, Starfield
- **Location**: [crates/nif/src/import/mesh.rs:197–204](crates/nif/src/import/mesh.rs#L197-L204)
- **Status**: NEW (related to #128)
- **Description**: The two-sided extraction path only downcasts to
  `BSLightingShaderProperty`. `BsTriShape.shader_property_ref` can legitimately point
  at a `BSEffectShaderProperty` (particles, fog, visual effects), and those shaders
  also carry the two-sided bit. When the target is an effect shader, the importer
  returns `false` unconditionally.
- **Impact**: Effect meshes render one-sided when they should be two-sided, producing
  visible backface-culling artifacts in particle/fog materials.
- **Suggested Fix**: Add a second arm that downcasts to `BSEffectShaderProperty` and
  checks the same `shader_flags_2 & 0x10` bit (verify bit position against nif.xml).

#### NIF-04-11-M2: `NiAlphaProperty` blend factors are discarded in the importer
- **Severity**: MEDIUM
- **Dimension**: Import Pipeline
- **Game Affected**: Oblivion, FO3, FNV
- **Location**: [crates/nif/src/import/material.rs:212–221](crates/nif/src/import/material.rs#L212-L221)
- **Status**: Existing: #213
- **Description**: The importer reads `NiAlphaProperty` flags (threshold + alpha-test /
  alpha-blend toggles) but never captures `src_blend_mode` / `dest_blend_mode`. The
  renderer hard-codes `(SrcAlpha, OneMinusSrcAlpha)` for every blended material.
- **Impact**: Legacy additive materials (magic effects, glow, glass) render incorrectly.
  Non-SrcAlpha blend modes are invisible to the pipeline.
- **Suggested Fix**: Add `src_blend` / `dest_blend` fields to `ImportedMesh` and thread
  through to the renderer's blend state.

#### NIF-04-11-M3: `NiTexturingProperty` per-slot UV transforms are discarded
- **Severity**: MEDIUM
- **Dimension**: Import Pipeline
- **Game Affected**: Oblivion, FO3, FNV
- **Location**: [crates/nif/src/import/material.rs:223–242](crates/nif/src/import/material.rs#L223-L242)
- **Status**: Existing: #219
- **Description**: Each `TexDesc` has per-slot `uv_offset` and `uv_scale`. The importer
  extracts only the filename, so scaled/offset detail maps, parallax-offset glow, etc.
  render at the wrong UV.
- **Impact**: Visual accuracy regression on complex materials in pre-Skyrim games.
- **Suggested Fix**: Extend `ImportedMesh` with per-texture-slot UV transforms and plumb
  through to the sampler / vertex shader.

#### NIF-04-11-M4: `NiPathController` / `NiLookAtController` families missing
- **Severity**: MEDIUM
- **Dimension**: Coverage
- **Game Affected**: Oblivion (other games use block_size fallback)
- **Location**: [crates/nif/src/blocks/mod.rs](crates/nif/src/blocks/mod.rs)
- **Status**: Existing: #228
- **Description**: `NiPathController`, `NiPathInterpolator`, `NiPathEvaluator`,
  `NiLookAtController`, `NiLookAtInterpolator`, `NiLookAtEvaluator` are not in the
  dispatch table. On Oblivion this is fatal; on FO3+ they land in NiUnknown and
  semantic data is lost.
- **Impact**: Legacy spline camera/object motion (cinematics) and bone-aiming (eye
  tracking) don't work. Prevalence estimated <1% of shipped content.
- **Suggested Fix**: Verify prevalence via `nif_stats` against Oblivion corpus; if
  non-zero, implement stubs (layout documented in Gamebryo 2.3 source).

#### NIF-04-11-M5: FO4 morph controller variants missing
- **Severity**: MEDIUM
- **Dimension**: Coverage
- **Game Affected**: Fallout 4
- **Location**: [crates/nif/src/blocks/mod.rs](crates/nif/src/blocks/mod.rs)
- **Status**: NEW (untracked)
- **Description**: `NiMorphWeightsController`, `NiMorphController`, and `NiMorpherController`
  are face-morph drivers distinct from the already-parsed `NiGeomMorpherController`.
  Not in the dispatch table; fall through to NiUnknown on FO4+.
- **Impact**: Estimated 2–5% of FO4 NPC meshes. Geometry renders correctly but face
  morph animation is lost.
- **Suggested Fix**: Check prevalence in FO4 KF corpus; implement or alias to the
  geometry morpher controller.

#### NIF-04-11-M6: FO3+ specialized controllers partially missing
- **Severity**: MEDIUM
- **Dimension**: Coverage
- **Game Affected**: FO3, FO4, FO76, Starfield
- **Location**: [crates/nif/src/blocks/mod.rs](crates/nif/src/blocks/mod.rs)
- **Status**: NEW (untracked)
- **Description**: `BSLagBoneController` (physics lag on hair/cloth bones),
  `BSKeyframeController`, `BSFrustumFOVController`, `BSMaterialEmittanceMultController`,
  `BSRefractionStrengthController`, `BSProceduralLightningController` are not in the
  dispatch. The effect-shader float/color variants *are* handled.
- **Impact**: ~1–3% of FO3+ character meshes. Rendering unaffected, but hair/cloth lag
  animation, emissive pulsing, and FOV sweeps are lost.
- **Suggested Fix**: Implement or alias to `NiSingleInterpController` — most are thin
  subtypes.

#### NIF-04-11-M7: Oblivion edge-case cascade risk on modded content
- **Severity**: MEDIUM (CRITICAL mitigated)
- **Dimension**: Coverage
- **Game Affected**: Oblivion
- **Location**: [crates/nif/src/blocks/mod.rs:559–569](crates/nif/src/blocks/mod.rs#L559-L569)
- **Status**: Existing: #224 (partially mitigated)
- **Description**: Oblivion NIFs have no `block_sizes`, so any unhandled block type
  terminates the parse. The 38+ critical gaps from 2026-04-07 are now closed, but
  modded content and rare vanilla variants may still reach the fallback arm.
- **Suggested Fix**: Run `nif_stats --bsa` against the full Oblivion vanilla corpus
  and confirm zero unknown types; flag any residual types for dedicated parsers.

### LOW (5)

#### NIF-04-11-L1: `NiParticleSystem` uses `variant().has_shader_alpha_refs()` instead of raw `bsver()`
- **Severity**: LOW
- **Dimension**: Block Parsing
- **Game Affected**: Non-Bethesda Gamebryo (theoretical)
- **Location**: [crates/nif/src/blocks/particle.rs:554](crates/nif/src/blocks/particle.rs#L554)
- **Status**: NEW
- **Description**: `base.rs:73` and `node.rs:107` use direct `stream.bsver()` comparisons
  to gate BSVER > 34 fields. `particle.rs:554` instead calls
  `stream.variant().has_shader_alpha_refs()`, which returns false for non-Bethesda
  variants even when BSVER > 34. This is an 8-byte stream misalignment for any
  hypothetical non-Bethesda Gamebryo content above BSVER 34.
- **Impact**: Unlikely to trigger in practice, but violates the consistency rule used
  elsewhere in the parser.
- **Suggested Fix**: `if stream.bsver() > 34 { ... }` to match the documented condition.

#### NIF-04-11-L2: Skyrim SLSF2 decal bits unchecked in material importer
- **Severity**: LOW
- **Dimension**: Import Pipeline
- **Game Affected**: Skyrim LE, Skyrim SE
- **Location**: [crates/nif/src/import/material.rs:149–153](crates/nif/src/import/material.rs#L149-L153)
- **Status**: Existing: #176
- **Description**: Decal detection checks `shader_flags_1 & (DECAL_SINGLE_PASS |
  DYNAMIC_DECAL)` but not any SLSF2 decal bits. `find_decal_bs` has the same gap.
- **Impact**: Ground decals (blood, scorch, footprints) marked via SLSF2 may z-fight.
- **Suggested Fix**: Look up SLSF2 decal bit positions in nif.xml and OR them in.

#### NIF-04-11-L3: `NiMaterialProperty` diffuse/ambient discarded
- **Severity**: LOW
- **Dimension**: Import Pipeline
- **Game Affected**: Oblivion, FO3, FNV
- **Location**: [crates/nif/src/import/material.rs:212–221](crates/nif/src/import/material.rs#L212-L221)
- **Status**: Existing: #221
- **Description**: Specular, emissive, shininess, alpha are captured; diffuse and
  ambient are not. In the no-per-vertex-color case, the base tint falls back to white.
- **Impact**: Unlit and simple-shaded legacy surfaces lose their material tint.
- **Suggested Fix**: Capture `mat.diffuse` / `mat.ambient` and route to the material
  pipeline.

#### NIF-04-11-L4: `NiSkinPartition` SSE gating uses `bsver == 100`
- **Severity**: LOW
- **Dimension**: Version Handling
- **Game Affected**: FO4, FO76, Starfield (hypothetical)
- **Location**: [crates/nif/src/blocks/skin.rs:185,302](crates/nif/src/blocks/skin.rs#L185)
- **Status**: Existing: #126
- **Description**: SSE-specific vertex data and the trailing triangle copy are gated
  on exact `bsver == 100`. If any FO4+ variant reuses the SSE layout it will not be
  recognized. No current runtime impact.
- **Suggested Fix**: Use `>= 100` (or an explicit `matches!(variant, SkyrimSE | ...)`)
  if future FO4+ formats are expected.

#### NIF-04-11-L5: `has_shader_emissive_color()` comment is misleading
- **Severity**: LOW (documentation)
- **Dimension**: Version Handling
- **Game Affected**: n/a
- **Location**: [crates/nif/src/version.rs:150–155](crates/nif/src/version.rs#L150-L155)
- **Status**: NEW
- **Description**: The predicate excludes FO76/Starfield with a comment "Present in
  FNV+ (user_version_2 >= 34)". The exclusion is semantically correct (the predicate
  is only queried on `BSShaderPPLightingProperty`, which those games don't use), but
  the comment invites the same misreading that produced C1.
- **Suggested Fix**: Either rename to `has_pp_shader_emissive_color` or append a
  sentence clarifying that FO76/Starfield use `BSLightingShaderProperty` instead.

## Verified Correct (regression check)

Items re-verified against the 2026-04-07 baseline and found still sound:

- **`NiTexturingProperty` shader textures** — unconditional `u32` count read, matching
  Gamebryo 2.3 source. nif.xml is wrong on this field. (Attempt to "fix" was reverted.)
- **`stream.skip()`** — `checked_add` overflow guard + EOF bound check (#N26-3-03 FIXED).
- **Particle emitter unknown interpolation** — returns `InvalidData` with block context
  instead of silent 8-byte default (#N26-3-02 FIXED).
- **Oblivion truncation visibility** — `NifScene.truncated` flag set on mid-parse
  failure; pinned by `test_parse_nif_truncated_flag` (#N26-3-05 FIXED).
- **Main loop block-size reconciliation** — per-block `position` check + seek to
  `start + block_size` on mismatch with WARN logging.
- **Base class delegation** — `NiObjectNETData::parse` / `NiAVObjectData::parse` /
  `parse_no_properties` routed correctly by every derived block.
- **Boolean type distinction** — `read_bool` (4-byte pre-4.1.0.1, 1-byte post) vs
  `read_byte_bool` (always 1-byte) applied consistently across geometry/shader/particle.
- **`NiSkinData`, `NiSkinPartition`, `BsDismemberSkinInstance`** — byte-correct vs
  nif.xml; full SSE path validated (asymmetric `has_bone_indices` gate logged as
  LOW design issue, not a functional bug).
- **`BSTriShape` variants** — `parse_segmented` / `parse_lod` / `parse_dynamic`
  dispatch + layout verified (#146).
- **Z-up → Y-up rotation** — SVD repair in `transform.rs` and `coord.rs` validates
  determinant, fixes singular values < 0.01, and corrects negative det via U-column
  negation.
- **`.av.*` field refactor** — zero stale direct accesses (`node.name`, `shape.flags`,
  `shape.transform`, `node.properties`) found via grep.
- **Alpha precedence** — `apply_alpha_flags` correctly prefers alpha-test over
  alpha-blend when both bits are set (unit tests pass for all 5 cases).
- **Variant detection** — all 9 variants (Oblivion → Starfield) sequenced correctly
  on `(version, uv, uv2)`; BSVER canonical values verified.
- **Feature flags (sound ones)** — `has_properties_list`, `has_material_crc`,
  `avobject_flags_u32`, `has_shader_alpha_refs`, `has_effects_list`,
  `uses_bs_tri_shape`, `uses_fo76_shader_flags` all match nif.xml gating.

## Prioritized Fix Order

Rendering first, then animation, then safety/physics, then polish.

1. **NIF-04-11-C1** (FO76/Starfield compact material) — one-line fix, immediately
   unblocks correct material rendering for two shipped games.
2. **NIF-04-11-H3** (unknown-block allocation cap) — safety hardening, ~30 min.
3. **NIF-04-11-C3** (Havok cascade stubs) — Oblivion collision-enabled meshes, plus
   FO4 `bhkNPCollisionObject` data recovery. 2–4 h.
4. **NIF-04-11-H1** (property inheritance) — unlocks correct rendering for a broad
   class of legacy meshes. Moderate effort (ancestor walk from import).
5. **NIF-04-11-M1** (BsTriShape two-sided effect shader) — low effort, visible
   particle/fog fix.
6. **NIF-04-11-M3** (NiTexturingProperty UV transforms) — moderate effort, material
   accuracy for legacy.
7. **NIF-04-11-M2** (NiAlphaProperty blend factors) — moderate effort.
8. **NIF-04-11-M7** (Oblivion vanilla corpus validation) — run `nif_stats`; cheap.
9. **NIF-04-11-C2** (Starfield BSVER 168–169 threshold) — one-line fix.
10. **NIF-04-11-H2** (BSStreamHeader gate) — one-line fix, spec compliance.
11. **NIF-04-11-M4** (Path/LookAt controllers) — prevalence-gated; validate first.
12. **NIF-04-11-M5 / M6** (morph + BS specialized controllers) — prevalence-gated.
13. **NIF-04-11-L1** (particle.rs variant check → bsver) — one-line consistency fix.
14. **NIF-04-11-L2** (SLSF2 decal bits) — lookup + OR mask.
15. **NIF-04-11-L3** (diffuse/ambient capture) — co-implement with H1.
16. **NIF-04-11-L4 / L5** (doc/gate cleanups) — batch with a version.rs refactor.
17. **NIF-04-11-E1** (Z-up translation helper) — pure refactor, last.
