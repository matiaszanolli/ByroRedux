- **Findings**: 0 CRITICAL, 0 HIGH, 1 MEDIUM, 4 LOW.
- **Headline**: FO4 ESM architecture-record parsing is healthy. ## Findings

### MEDIUM — FO4-D4-NEW-08: XMSP REFR sub-record never parsed; ~2,500 MSWP tables sit unused

- **Premise**: `parse_mswp_group` at [crates/plugin/src/esm/cell/support.rs:486-509](../../crates/plugin/src/esm/cell/support.rs#L486-L509) populates `EsmCellIndex::material_swaps` with all ~2,500 vanilla FO4 MSWP records. The REFR walker `parse_refr_group` at [crates/plugin/src/esm/cell/walkers.rs:336-487](../../crates/plugin/src/esm/cell/walkers.rs#L336-L487) matches XATO/XTNM/XTXR/XEMI/XESP/XTEL/XPRM/XLKR/XRMR/XPOD/XRDS/XOWN/XRNK/XGLB but has no `b"XMSP"` arm. Doc comment at [crates/plugin/src/esm/cell/mod.rs:658](../../crates/plugin/src/esm/cell/mod.rs#L658) and at `records/mswp.rs:5,31` both promise XMSP routing.
- **Gap**: Every REFR's `XMSP` (4-byte MSWP FormID) falls on the `_ => {}` catch-all at `walkers.rs:486`. `PlacedRef` has no `material_swap_ref: Option<u32>` field; `build_refr_texture_overlay` at `cell_loader_refr.rs:200-235` never looks up `index.material_swaps`.
- **Impact**: Every vanilla Raider armour colour-variant, settlement clutter variation, station-wagon rust pattern, and Vault decay overlay renders with the base mesh's textures.
- **Suggested fix**: Add `material_swap_ref: Option<u32>` to `PlacedRef`, add a `b"XMSP" => material_swap_ref = r.u32().ok()` arm to `parse_refr_group`, and extend `build_refr_texture_overlay` to look up `index.material_swaps.get(&swap_ref)` and apply each `MaterialSwapEntry` to the overlay (analogous to existing XATO/XTNM/XTXR fill paths).
- **Dedup**: New.

### LOW — FO4-D4-NEW-09: `TextureSet::flags` HasModelSpaceNormals bit parsed but renderer never consults it

- **Premise**: [crates/plugin/src/esm/cell/support.rs:251-257](../../crates/plugin/src/esm/cell/support.rs#L251-L257) parses `DNAM` into `TextureSet::flags: u16` (100% vanilla FO4 TXST coverage); bit 2 is `HasModelSpaceNormals`.
- **Gap**: `grep -rn "ts.flags|HasModelSpaceNormals|TextureSetFlags"` finds zero consumers outside tests. The renderer's normal-map decode branches off `BGSM.model_space_normals` (`crates/bgsm/src/bgsm.rs:94`) but never reads the TXST DNAM flag for direct-TXST (non-MNAM) REFRs.
- **Impact**: FO4 TXST records that override a host mesh's normal map AND use a model-space (not tangent-space) normal will decode as tangent-space. Visual symptom: discoloured / wrongly-shaded normal mapping on TXST-overridden re-skinned props. Vanilla count small (most route through BGSM); mod content authoring direct TXST overrides will hit it.
- **Suggested fix**: Plumb `TextureSet::flags & 0x04` through `RefrTextureOverlay` (new `model_space_normals: bool`), branch the renderer's normal decode at the existing BGSM tap. Defer until the renderer has a stable model-space normal path.
- **Dedup**: New.

### LOW — FO4-D4-NEW-10: `DecalData` parsed but no decal pipeline consumes it (M28 dependency)

- **Premise**: 207/382 vanilla FO4 TXSTs ship a `DODT` payload populating `TextureSet::decal_data: Option<DecalData>` ([crates/plugin/src/esm/cell/mod.rs:470-486](../../crates/plugin/src/esm/cell/mod.rs#L470-L486), `support.rs:262-280`).
- **Gap**: `grep -rn "decal_data\b"` outside tests/issue-docs returns zero consumers. The doc comment at `cell/mod.rs:466-468` already calls this out — consumed by `RenderLayer::Decal` once the M28 decal pipeline lands.
- **Impact**: FO4 blood splatters, scorch marks, posters, graffiti — rendered as static alpha-tested overlay with no per-decal projection/parallax. Parser-side gap is closed; renderer pass that consumes it doesn't exist yet.
- **Suggested fix**: Track as part of M28 decal-pipeline milestone, not a bug.
- **Dedup**: New (or fold into M28 tracker).

### LOW — FO4-D4-NEW-11: MOVS sound/destruction fields parsed but no consumer (DLC-only impact)

- **Premise**: `parse_movs` at [crates/plugin/src/esm/records/movs.rs:85-131](../../crates/plugin/src/esm/records/movs.rs#L85-L131) captures `LNAM` looping sound, `ZNAM` activate sound, `DEST` destruction-presence flag.
- **Gap**: Vanilla FO4 ships zero MOVS (harness pins `movables.len() == 0`). DLC + mod content authors MOVS for breakable furniture / deployable workshop objects. No audio system reads LNAM, no destruction subsystem reads DEST.
- **Impact**: DLC- or mod-shipped MOVS render visually but stay silent and unbreakable. No vanilla-FO4 impact.
- **Suggested fix**: Defer until either an audio-emitter component reads LNAM or a destruction subsystem reads DEST.
- **Dedup**: New (LOW; consumer deferred).

### LOW — FO4-D4-NEW-12: PKIN `VNAM` / `FNAM` parsed but no consumer (workshop UI dependency)

- **Premise**: `parse_pkin` at [crates/plugin/src/esm/records/pkin.rs:76-142](../../crates/plugin/src/esm/records/pkin.rs#L76-L142) captures `VNAM` (workshop/preview marker FormID) and `FNAM` ("Location Reference Type", "Perk" flag bits).
- **Gap**: PKIN expansion at `cell_loader_refr.rs:267-311` only reads `pkin.contents`. `vnam_form_id` and `flags` are dead fields.
- **Impact**: Workshop build-mode preview / "is-perk-required" gating not honoured. Vanilla cell rendering unaffected.
- **Suggested fix**: No action until workshop-mode build-bar work lands.
- **Dedup**: New (LOW; consumer deferred).

## Minimum FO4 Cell-Render Path

For a basic FO4 interior cell (Sanctuary Hills basement, Vault 111 entry) to render with placements:

1. ✓ **TXST 8-slot + MNAM + DNAM + DODT parsing** — every variant captured.
2. ✓ **SCOL ONAM/DATA placement decode + cell-loader expansion** — `expand_scol_placements` with composed Z-up→Y-up transform at `byroredux/src/cell_loader_refr.rs:336-378` (#585).
3. ✓ **PKIN CNAM bundle decode + expansion with cycle cap** — `expand_pkin_placements_with_depth`, `MAX_PKIN_DEPTH = 4` (#589, #635).
4. ✓ **MOVS visual placement** — registered as STAT-shape `StaticObject` with `RecordType::MOVS`; physics activation is a separate milestone.
5. ✓ **CELL XCLL ambient/directional/fog + XCLW water + LTMP fallback** — wired in `walkers.rs`.
6. ✓ **REFR XATO/XTNM/XTXR texture overlay** — `build_refr_texture_overlay`.
7. ✓ **REFR XESP enable-parent, XTEL teleport, XPRM primitive bounds, XLKR linked refs, XRMR rooms, XPOD portals, XRDS light radius, XEMI emissive light, XOWN/XRNK/XGLB ownership** — all on the walker.
8. ✗ **REFR XMSP → MSWP material-swap application** — FO4-D4-NEW-08. Most vanilla colour variants stay generic.
9. ✗ **TXST DNAM `HasModelSpaceNormals` → renderer branch** — FO4-D4-NEW-09. Visual fidelity gap for TXST-overridden meshes.
10. ✗ **CELL XCWT/XCAS/XCMO consumer wiring** — parsed at `walkers.rs:109-120` but audio/water subsystem TBD (cross-dimension).
11. ✗ **DLC-master remap path through CLI** — `FormIdRemap` exists (#445) but the CLI is single-plugin today; not required for vanilla `Fallout4.esm` interior cells.

For a vanilla `Fallout4.esm` interior cell with no mod content, items 1-7 are sufficient. Items 8-9 raise visual fidelity but won't block initial render.
