## Executive Summary

**Zero findings. FNV ↔ FO3 BSA-v104 parity is confirmed end-to-end.**

FO3 and FNV share an identical v104 codepath (single `version >= BSA_V_FO3_SKYRIM` branch at [`crates/bsa/src/archive/open.rs:75`](../../crates/bsa/src/archive/open.rs#L75) for the `embed_file_names` flag, same 16-byte folder records, same zlib codec at [`extract.rs:128-138`](../../crates/bsa/src/archive/extract.rs#L128-L138)). Since FNV reports 100% on `Fallout - Meshes.bsa` per `ROADMAP.md` and the codepath is shared, FO3 inherits that guarantee — and the real-data sweep below confirms it.

**DDS validity**: all 12 261 entries from `Fallout - Textures.bsa` begin with the `DDS ` magic (`0x20534444`). v104 stores complete DDS bytes — no header-reconstruction logic exists in the BSA reader; reconstruction is exclusive to BA2 DX10 archives at [`crates/bsa/src/ba2.rs:749-759`](../../crates/bsa/src/ba2.rs#L749-L759). The pass-through path is confirmed via [`extract.rs:164-172`](../../crates/bsa/src/archive/extract.rs#L164-L172) (uncompressed branch) and `extract.rs:128-138` (zlib branch).

**Hash collisions**: 0 collision groups across `(folder_hash, file_hash)` pairs in the Meshes (14 167 entries), Textures (12 280), and Sound (2 709) archives. The hash function at [`hash.rs:27`](../../crates/bsa/src/archive/hash.rs#L27) matches BSArch / libbsarch (verified per #449). Sanity-only check — the live reader at [`mod.rs:55`](../../crates/bsa/src/archive/mod.rs#L55) is path-keyed, not hash-keyed, so collisions wouldn't even reach the lookup path.

**DLC sibling auto-loader sanity**: `Fallout - Textures2.bsa`, `Anchorage - Main2.bsa`, etc. don't exist on disk. The `open_with_numeric_siblings` loop at [`byroredux/src/asset_provider.rs:337`](../../byroredux/src/asset_provider.rs#L337) hits `Path::is_file()` → false → `continue` — no spurious DLC-name probes, no error logs.
