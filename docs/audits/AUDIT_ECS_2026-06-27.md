## Notable observations (not findings)

1. **Boot guard strengthened beyond the documented invariant.** The skill
   documents `debug_assert_eq!(undeclared_parallel_count(), 0)` (#1394). The live
   guard at `byroredux/src/main.rs:973-996` now **triple-gates** —
   `undeclared_parallel_count() == 0` **and** `known_conflict_count() == 0`
   **and** `unknown_pair_count() == 0` (#1602). All three evaluate to 0 (the
   binary compiles clean in debug; the only post-guard registration is
   `debug-server`'s `add_exclusive(Late, …)`, which cannot perturb a parallel
   count). The access-declaration invariant is healthier than the skill text.

2. **`TRACK_CHANGES` enabled on more than Transform/GlobalTransform.** It is also
   on `Parent`/`Children` (`hierarchy.rs:18,30`) and `LocalBound`
   (`local_bound.rs:91`) — all `SparseSetStorage`-backed, so they drive the
   `structural_generation` reparent-detection path that feeds the dim-8c fast
   path, not the dirty-set path. Consistent with the propagation design, not a
   regression. (The skill text names only Transform/GlobalTransform; the live set
   is a superset by design.)
