# Fallout New Vegas Compatibility Audit — 2026-04-21

Audit of ByroRedux readiness for Fallout: New Vegas content across six
dimensions: NIF parser regression guard, ESM record coverage, cell loading
end-to-end, RT lighting pipeline, real-data validation, and animation +
skinning.

FNV is the **reference title** — the most validated end-to-end path in the
engine. Regressions against roadmap baselines are Critical by default.

Git HEAD at audit time: `3aa5ea0` (`chore(bsa): add bsa_extract_one example`).

## Executive Summary

**Compatibility level**: Tier-1 reference — all previously-validated baselines
hold. 14,881 / 14,881 vanilla NIFs parse at 100%. `FalloutNV.esm` produces
13,684 structured records matching the M24 Phase 1 baseline to the record.
Prospector Saloon (GSProspectorSaloonInterior) loads live at 799 entities /
773 draws / 25 point lights / ~272 FPS avg on RTX 4070 Ti. WastelandNV 7×7
grid loads live at 3,927 cell entities / ~4,120 draws / ~208 FPS avg with
`NVWastelandClear` weather picked correctly. Every named regression-guard
commit (#340, #350, #381, #408, #440, #441, #454, #455, #463, c46dc78,
3ece152) verified intact.

**One CRITICAL RT correctness bug**:
[FNV-RT-1](#fnv-rt-1-frustum-culling-drops-off-screen-blas-from-the-tlas) —
frustum culling at [byroredux/src/render.rs:302-306](byroredux/src/render.rs#L302-L306)
drops off-screen geometry from the `DrawCommand` stream **before** BLAS/TLAS
registration and **before** the LRU stamp site. On-screen fragments lose
shadow / reflection / GI contributions from off-screen occluders (ray misses
→ fragment stays fully lit), and occluder BLAS age monotonically starting
the frame they leave the frustum — generating hitches when the camera pans
back. This is the single highest-impact finding and predates all prior FNV
audits.

**Five HIGH findings**: two are pre-existing OPEN ESM layout bugs
(#481 FACT DATA u32-vs-u8, #377 NPC disposition_base sign-truncation);
one is new ECS-layer lossy color-target
([FNV-ANIM-1](#fnv-anim-1-animatedcolor-loses-target-slot-discriminator));
one is doc-drift on the Shader Struct Sync memory
([FNV-RT-2](#fnv-rt-2-shader-struct-sync-doc-claims-3-shaders-actual-surface-is-4));
one is tooling ([FNV-RUN-1](#fnv-run-1-texmissing-texloaded-meshcache-meshinfo-unreachable-via-byro-dbg))
where the Session-10 `tex.missing` / `tex.loaded` / `mesh.cache` console
commands are registered in-engine but unreachable from the out-of-process
`byro-dbg` CLI.

**Two confirmed regressions**:
- [FNV-RUN-2](#fnv-run-2-exterior-unresolved-statics-count-regressed-from-83-to-129) —
  exterior unresolved-statics rose from 83 → 129 since session 12, likely an
  artifact of #445 `FormIdRemap` exposing DLC master references without
  dispatchers. Already covers ground under #386 but the count grew.
- [FNV-RUN-3](#fnv-run-3-exterior-tlas-coverage-drops-during-first-3-frames) —
  TLAS coverage drops 10–20% during the first 3 exterior frames of the
  cell-load burst; ~195 meshes in steady state never get BLAS. Related to
  #510.

**Parse / record / RT infrastructure is clean.** Streaming RIS (K=8,
64× W clamp, unbiased Σw/(K·w_sel)), distance fade (4000–6000), BLAS
compaction (`ALLOW_COMPACTION` + query pool + `hostQueryReset`), SVGF
(motion + mesh_id disocclusion + 2×2 bilinear), TAA (Halton(2,3), α=0.1,
YCoCg variance clip γ=1.25), sky+cloud compositing, and GpuInstance 192-byte
lockstep across **all 4** shader declaration sites + Rust side are all
verified correct. The only mechanism-level defect is the frustum-cull path

**Stale doc (recurring)**: CLAUDE.md + ROADMAP still cite a 48 FPS
Prospector target and a "3 shaders" GpuInstance sync note; live measurement
is ~272 FPS on RTX 4070 Ti and the actual sync surface is 4 shaders
(`caustic_splat.comp` added in #321). Session-12 already date-stamped FO3
rows under #456/#457 — apply the same treatment here.

## Findings (Severity-Sorted)

### FNV-RT-1: Frustum culling drops off-screen BLAS from the TLAS
- **Severity**: CRITICAL
- **Dimension**: 4 (RT lighting)
- **Location**: [byroredux/src/render.rs:296-306](byroredux/src/render.rs#L296-L306) (cull → `continue`) and [crates/renderer/src/vulkan/acceleration.rs:1070-1074](crates/renderer/src/vulkan/acceleration.rs#L1070-L1074), [:1364-1373](crates/renderer/src/vulkan/acceleration.rs#L1364-L1373)
- **Status**: NEW
- **Description**: `build_render_data` applies a view-frustum sphere test
  against each entity's `WorldBound` and `continue`s before pushing a
  `DrawCommand`. Consequently, out-of-frustum geometry:
  1. Is **absent from the TLAS** — shadow / reflection / GI rays cast
     from on-screen fragments miss it and return "no hit" (fragment stays
     fully lit).
  2. Is not visited by the `last_used_frame` update loop at
     `acceleration.rs:1364-1373` (which only iterates `draw_commands` and
     additionally gates on `if !draw_cmd.in_tlas { continue; }`), so its
     BLAS ages monotonically starting from when it was last on-screen even
     when it is an active occluder / reflector of an on-screen pixel.
  3. Becomes an eviction candidate (`evict_unused_blas`) as soon as
     `idle >= MAX_FRAMES_IN_FLIGHT + 1`, after which re-entering the
     frustum requires a full BLAS rebuild → hitch.
- **Evidence**:
  - `render.rs:302`: `if wb.radius > 0.0 && !frustum.contains_sphere(wb.center, wb.radius) { continue; }` — no DrawCommand pushed, so no TLAS entry, no `last_used_frame` touch.
  - `acceleration.rs:1072`: `if !draw_cmd.in_tlas { continue; }` inside the instance-build loop.
  - `acceleration.rs:1366-1372`: the **only** LRU-stamp site, identical filter — `!draw_cmd.in_tlas` skips before `blas.last_used_frame = self.frame_counter`.
  - `acceleration.rs:1054`: `self.frame_counter += 1` happens unconditionally at the top of `build_tlas` every frame, so any BLAS not re-stamped this frame drifts one frame older per TLAS build.
- **Impact**: Shadows from any occluder larger than the view frustum
  (walls of the room the camera is inside, cliffs the camera is standing
  behind, buildings the player just rounded the corner of) are silently
  incorrect — the ray misses, the fragment is treated as unshadowed. The
  streaming RIS subtraction only ever subtracts for rays that hit, so
  missing occluders → stale light addition, not merely an under-shaded
  pixel. Visible as lightshaft / self-illum pop-in as soon as the
  occluder's bounding sphere crosses the far plane, and as "flashlight
  through a wall" when the player rotates to face away from a backlight
  whose occluder is now behind the camera. On cell transition the BLAS
  LRU evicts occluders faster than their reuse cadence warrants,
  guaranteeing hitches on return paths.
- **Suggested Fix**:
  1. Split the DrawCommand predicate: rasterizer-cull continues to use
     `frustum.contains_sphere`, but ALL opaque draws are pushed with
     `in_tlas = true`. Introduce a separate `in_raster` bit (or move the
     frustum-cull out of `build_render_data` entirely and into the
     DrawCommand-iteration in `draw_frame`). The rasterizer already
     honors `in_tlas` at `acceleration.rs:1072` for TLAS inclusion.
  2. Alternative narrow fix: introduce a second, looser "TLAS frustum"
     (e.g., 1.5× the view frustum, or a light-source-union frustum) so
     the TLAS carries all geometry that could shadow any visible pixel.
     Preserves most of the M31 culling win while closing the correctness
     hole.
  3. Independently, move the `last_used_frame` stamp from `build_tlas`
     out to the DrawCommand-assembly pass in `render.rs` (pre-cull) so an
     occluder's BLAS stays warm even when the current frame does not need
     its TLAS slot. That alone restores the LRU residency half of this
     bug.

### FNV-RT-2: Shader Struct Sync doc claims 3 shaders, actual surface is 4
- **Severity**: HIGH (doc-drift → silent binary breakage on the next edit)
- **Dimension**: 4
- **Location**: [crates/renderer/src/vulkan/scene_buffer.rs:104](crates/renderer/src/vulkan/scene_buffer.rs#L104) + CLAUDE.md memory key `Shader Struct Sync`
- **Status**: NEW
- **Description**: The `gpu_instance_defined_in_all_four_shaders` test (at
  `scene_buffer.rs:1316`) DOES enumerate four files (`triangle.vert`,
  `triangle.frag`, `ui.vert`, `caustic_splat.comp`). But the inline doc
  comment on `GpuInstance` (`scene_buffer.rs:104` — "3 shaders") and the
