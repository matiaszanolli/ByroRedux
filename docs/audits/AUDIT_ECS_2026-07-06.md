## Per-Dimension Results

### 1. Lock Ordering & Deadlock — CLEAN
- Every `query`/`query_mut`/`resource`/`resource_mut`/`get`/`try_resource*` site
  in `world.rs` arms a `TrackedRead`/`TrackedWrite` scope, `defuse()`s it only
  **after** the real lock is acquired, and the wrapper's `Drop`
  (`query.rs`/`resource.rs`) untracks. `World::get` correctly keeps the scope
  armed on the `None` branch (no double-untrack).
- `query_2_mut` / `query_2_mut_mut` / `resource_2_mut` acquire in `id_a < id_b`
  TypeId order **and** arm the tracker scopes in the same TypeId order (the #313
  fix) — the `if id_a < id_b { … } else { … }` branches confirm scopes are not
  armed in generic-parameter order. `try_resource_2_mut` does both existence
  checks before acquiring either lock (#465).
- Same-type double-lock: `assert_ne!(A, B)` present on all three paired APIs.
- Cross-thread ABBA graph (`lock_tracker::global_order`) is `cfg(debug_assertions)`,
  env-gated, with fast-path read + write-upgrade-on-novel-edge. Regression guard
  `global_graph_detector_end_to_end` present.
- Poison resolution: every acquisition resolves `PoisonError` through
  `storage_lock_poisoned::<T>()` / `storage_lock_poisoned_erased()` /
  `resource_lock_poisoned::<R>()`. `despawn` and `clear_entities` use the
  type-erased variant fed by the `type_names` side-table (#466/#1836).
  `insert_resource`/`remove_resource` re-panic on a poisoned prior value (#1837).
  Guards: `poisoned_storage_lock_panics_with_type_name`,
  `poisoned_resource_lock_panics_with_type_name`,
  `insert_resource_over_poisoned_lock_panics_with_type_name`.

### 2. Storage Correctness — CLEAN
- `SparseSetStorage` (`sparse_set.rs`): swap-remove fixes the moved entity's
  sparse pointer (`self.sparse[moved_entity] = Some(dense_idx)`); last-element
  removal takes the no-swap `pop` path; insert overwrites in place. `structural_gen`
  bumps on insert (incl. overwrite/reparent) and on remove (only after the `?`
  present-check) and on `clear_erased`. Guards `swap_remove`, `remove_last`,
  `overwrite` present.
- `PackedStorage` (`packed.rs`): `binary_search` maintains sort order on
  insert/remove; `insert_bulk` uses append + single stable-sort-by-index +
  last-writer-wins dedup (#467). Guards
  `insert_bulk_matches_serial_insert_for_unique_ids`,
  `insert_bulk_duplicate_ids_last_writer_wins`,
  `insert_bulk_extends_non_empty_storage`, `insert_bulk_empty_is_noop` present.
- Change tracking: `mark_dirty` fires on insert/get_mut/remove/iter_mut/insert_bulk;
  `drain_dirty_into` clears `out` then preserves `self.dirty` capacity (#1371);
  `take_dirty` hands capacity away as documented. `TRACK_CHANGES` const-false by
  default (branch folds). Guards `drain_dirty_into_preserves_storage_capacity`,
  `untracked_component_never_accumulates_dirty`,
  `tracked_get_mut_and_insert_record_dirty` present.
- `World::insert_batch` wraps the iterator with a per-item `debug_assert(entity <
  next_entity)` adapter so the bulk path can't admit unspawned IDs.

### 3. Query Borrow Safety — CLEAN
- `QueryRead`/`QueryWrite`/`ComponentRef` (`query.rs`) each hold the guard for
  the wrapper's lifetime and cache the downcast pointer once in `new()` (#1367).
  The `#[allow(dead_code)] guard` field is declared before the pointer, so it is
  not dropped early; every `unsafe { &*… }` / `&mut *…` has a SAFETY comment tying
  validity to the live guard. `ComponentRef` retains the guard (sound replacement
  for the #35 UAF pattern).
- `QueryWrite::storage_mut` requires `&mut self`; `Deref`/`DerefMut` route through
  `storage()`/`storage_mut()`, so the borrow checker forbids simultaneous `&`/`&mut`.
- `query`/`query_mut` return `None` for never-created storage (no lazy creation on
  the read path).

### 4. Resource Lifetimes — CLEAN
- `resource()`/`resource_mut()` panic with the type name when absent;
  `try_resource*` return `None`. `ResourceRead`/`ResourceWrite` downcast through
  the guard on each `Deref` (not cached — correctly excluded from the #1367 hot
  path; keyed by TypeId so the `expect` cannot fire).
- `insert_resource` returns the prior value via downcast; `remove_resource` and
  `insert_resource` resolve poison via `resource_lock_poisoned`.
- `try_resource_2_mut` performs both `contains_key` checks before delegating to
  `resource_2_mut` (#465 — no partial-acquire window). Guard
  `resource_2_mut_reverse_order_same_result` present.

### 5 / 5b. System & Scheduler Wiring + Access Declarations — CLEAN
- Blanket `System` impl for `FnMut(&World, f32)` (`system.rs`); default
  `access()` is `None`. Mutations visible across stages
  (`mutation_visible_across_stages`). Empty scheduler/stages run cleanly
  (`empty_scheduler_runs_cleanly`, `empty_stages_skipped`).
- `system_names()` yields stage-order then parallel-before-exclusive;
  `try_add_*` reject duplicates with `Err(name)` across the flat name space (#312).
- Panic policy is fail-fast by design (TS-08/#1412) — documented in the `run`
  doc comment; `run` takes `&mut self` and `Scheduler` is not a `Resource` (#868).
  Not reported as a gap, per skill.
- Stages `Early(0) → Update(1) → PostUpdate(2) → Physics(3) → Late(4)` iterated
  via `BTreeMap<Stage,_>` Ord; no `ParallelUpdate`/`LateExclusive` stage —
  "exclusive" is a per-stage phase (`StageData.exclusive`). Guard
  `all_five_stages_run_in_order` present.
- `Access` is the declaration type (`access.rs`); `AccessConflict` has exactly
  `None` / `Unknown { left_undeclared, right_undeclared }` / `Conflict { pairs }`
  (no `Parallel` variant). `analyze_pair` returns `Unknown` when either side is
  undeclared. Accessors `undeclared_parallel_count`, `undeclared_exclusive_count`,
  `known_conflict_count`, `unknown_pair_count` present (#1394/#1237). Guards
  `undeclared_closure_pairs_show_as_unknown`,
  `exclusive_systems_are_listed_but_not_paired` present.
- Binary boot guard live: `byroredux/src/main.rs:1049`
  `debug_assert_eq!(report_snapshot.undeclared_parallel_count(), 0, …)`. Every
  parallel-stage registration uses `add_to_with_access` (12 sites confirmed);
  `timer_tick_system` declared via `add_to_with_access` (main.rs:694). Exclusives
  (`event_cleanup_system`, `audio_system`, `ragdoll_writeback_system`) registered
  via plain `add_exclusive(Stage::Late, …)` — undeclared-exclusive is by design,
  not a finding.

## Notes for maintainers

The ECS core has reached a mature, well-guarded state: every invariant this audit
checked is backed by a named regression test that is currently green. The audit
adds no issues. Future regressions most likely to matter (watch these guards):

- `drain_dirty_into_preserves_storage_capacity` — the #1371 capacity-preservation
  contract that the transform-propagation fast path depends on.
- `change_detection_reacts_to_reparent_overwrite` — the structural-generation
  path that a switch of `Parent`/`Children` off `SparseSetStorage` (or dropping
  their `TRACK_CHANGES`) would silently break.
- The `undeclared_parallel_count() == 0` boot guard in `main.rs` — the only thing
  keeping the M27 access-declaration migration from silently regressing when a new
  parallel system is added via `add_to` instead of `add_to_with_access`.
