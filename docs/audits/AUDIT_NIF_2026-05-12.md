## Findings ‚Äî Grouped by Severity

### HIGH (1)

#### NIF-D5-NEW-01: ‚â•36 dispatched extra-data / controller / modifier types are orphan-parsed ‚Äî never `downcast_ref`'d on the consumer side
- **Severity**: HIGH
- **Dimension**: Coverage Gaps
- **Game Affected**: FO3, FNV, Skyrim SE, FO4, FO76 (FO4 / FO76 are the heaviest hit because they ship the most Bs*ExtraData / connect-point content)
- **Location**: dispatch arms in `crates/nif/src/blocks/mod.rs` vs consumer surface in `crates/nif/src/import/walk.rs` and `crates/nif/src/anim.rs` (24 distinct `downcast_ref::<T>` / `get_as::<T>` targets confirmed; ‚â•36 dispatched types absent from the set)
- **Status**: NEW (expands open #869)
- **Description**: Every dispatched block type carries a parser; not every dispatched type has a consumer. The orphan set includes `BsClothExtraData`, `BsDecalPlacementVectorExtraData`, `BsBehaviorGraphExtraData`, `BsInvMarker`, `BsBound`, `BsWArray`, `BsCollisionQueryProxyExtraData`, `BsDistantObjectLargeRefExtraData`, `BsConnectPointParents`, `BsConnectPointChildren`, `BsEyeCenterExtraData`, `BsTreadTransfInterpolator`, `NiBsBoneLodController`, all four `NiLight*Controller` variants, `NiFloatExtraDataController`, all six `NiPSys*FieldModifier` (Vortex/Gravity/Drag/Turbulence/Air/Radial), and the `BsRangeNode` kind discriminator (Blast/DamageStage/Debris). All parse cleanly; all flow into `NifScene` blocks; none of them are read by the import pipeline.
- **Impact**: behavioural drift on every NIF carrying these blocks. Concrete examples:
  - Vertibird / Liberty Prime tread animation is static (`BsTreadTransfInterpolator` parser landed #941, no consumer wired)
  - FaceGen dialogue camera frames the NIF origin instead of the eye centroid (`BsEyeCenterExtraData` parser landed #720, no consumer)
  - Weapon-mod attachment points (`BsConnectPointParents` ‚Äî ubiquitous in FO4) parsed but never threaded into the equip / mod system
  - Particle force fields parse but particles ignore gravity / vortex / turbulence
  - Light flicker / dim / pulse controllers parse but every lantern, campfire, plasma weapon emits constant light
- **Suggested Fix**: too broad for a single issue. **File a tracking meta-issue** expanding #869: enumerate every dispatched type, assert at least one `downcast_ref::<T>` consumer exists, document the rationale for each "no consumer yet (deferred to M-XX)" case. Then triage by impact band (visible-on-shipping-content first).

### MEDIUM (7)

#### NIF-D1-NEW-01: `hkPackedNiTriStripsData` ignores `Compressed` flag ‚Äî reads f32 vertices when authored content sets `Compressed != 0`
- **Severity**: MEDIUM (dormant on vanilla; live on modded/future content)
- **Dimension**: Block Parsing
- **Game Affected**: All Bethesda (Oblivion onward)
- **Location**: `crates/nif/src/blocks/collision.rs:942-953`
- **Status**: NEW
- **Description**: Per `nif.xml:3962-3967`, `hkPackedNiTriStripsData` flips between `Vector3[]` and `HalfVector3[]` based on a `Compressed: bool` byte. Our parser reads the byte into `_compressed`, discards it, then always reads `f32` triplets. When compressed=1, the parser over-reads 6 B/vertex and corrupts subsequent fields (`Num Sub Shapes` lands on garbage; gigabytes of HkSubPartData entries allocated or block aborts). FO3+ block_size recovery realigns silently ‚Äî parse-rate metrics stay green, but the collision mesh's vertex buffer is fully scrambled. Oblivion has no block_sizes ‚Üí cascade.
- **Suggested Fix**: capture `compressed: bool` and branch the vertex read between `f32` triplets and `half_to_f32(u16)` triplets. `half_to_f32` already exists at `crate::blocks::tri_shape::half_to_f32`. Add a fixture-based regression test under `nif/tests/`.

#### NIF-D4-NEW-02: BSLightingShaderProperty drops Starfield `.mat` material references ‚Äî asymmetric suffix logic with the sibling BSEffectShader branch
- **Severity**: MEDIUM
- **Dimension**: Import Pipeline
- **Game Affected**: Starfield primarily; FO76 marginally
- **Location**: `crates/nif/src/import/material/walker.rs:116-121`
- **Status**: NEW (upstream of open #762 SF-D6-03)
- **Description**: The BSLightingShaderProperty branch inlines an ad-hoc suffix test that accepts only `.bgsm` / `.bgem`. The shared `is_material_reference` helper (`blocks/shader.rs:24`) accepts `.mat` (Starfield JSON materials) AND trims trailing `\0`/whitespace per #749. The BSEffectShaderProperty branch four lines below correctly delegates through `mesh::material_path_from_name` ‚Üí `is_material_reference`. The asymmetry means a Starfield BSLightingShaderProperty whose `name` carries a `.mat` reference imports with `material_path = None`, and the BgsmProvider lookup never fires.
- **Suggested Fix**: route through `mesh::material_path_from_name(shader.net.name.as_deref(), pool)` in lockstep with the sibling branch.

#### NIF-D4-NEW-03: BSSkyShaderProperty / BSWaterShaderProperty parsed but never consumed by importer ‚Äî Skyrim+ sky / sun-glare / moon / star NIFs render as magenta
- **Severity**: MEDIUM
- **Dimension**: Import Pipeline
- **Game Affected**: Skyrim SE, FO4, FO76
- **Location**: parsed at `crates/nif/src/blocks/shader.rs:493-599`; no `scene.get_as::<BSSkyShaderProperty>` / `<BSWaterShaderProperty>` site in `crates/nif/src/import/material/walker.rs`
- **Status**: NEW
- **Description**: Both Skyrim-era subclasses fully parse (`source_texture`, `sky_object_type`, `uv_offset/uv_scale`, `water_shader_flags`). The FO3/FNV-era counterparts (`SkyShaderProperty`, `TallGrassShaderProperty`) were plumbed in #940; the Skyrim+ siblings were missed. Every `meshes/sky/*.nif` (clouds, sunglare, moon, stars) imports with `texture_path = None` and renders as the magenta placeholder.
- **Suggested Fix**: add two `scene.get_as::<BSSkyShaderProperty>(idx)` / `<BSWaterShaderProperty>` branches inside the `shader_property_ref` block alongside the BSLighting / BSEffect branches.

#### NIF-D5-NEW-02: uncompressed B-spline interpolators (`NiBSplineFloatInterpolator`, `‚Ä¶Point3Interpolator`, `‚Ä¶TransformInterpolator`) have no dispatch arms

- **Game Affected**: FO3, FNV, Skyrim SE (older idle KFs authored without compression)

#### NIF-D5-NEW-03: `bhkBallSocketConstraintChain` missing ‚Äî Oblivion ragdoll spine cascade
