# FO4 Compatibility Audit — 2026-05-15

**Engine**: ByroRedux · **Branch**: main · **HEAD**: 94675f12
**Auditor**: `/audit-fo4` multi-agent orchestrator (6 dimension agents)
**Data**: `/mnt/data/SteamLibrary/steamapps/common/Fallout 4/Data/`
**Scope**: NIF parser (BSVER 130), BA2 reader, shader flags, ESM architecture records, real-data validation, forward blockers

---

## Executive Summary

FO4 is in excellent parse-and-render shape. The hard metrics are:

- **NIF parse rate**: 99.71% clean / 100% recoverable (34 895 clean / 100 truncated / 0 hard failures across 34 995 NIFs). The 100 truncated files are all FaceGen morphs with a missing `"NiExtraData"` dispatch arm — one targeted fix closes the gap.
- **BA2 reader**: Exhaustive version `match` over `{1, 2, 3, 7, 8}` with named constants; two-axis compression model correct; DDS reconstruction spec-compliant. No BA2-layer correctness issues.
- **ESM architecture records**: SCOL, MOVS, PKIN, TXST, MSWP all parsed and registered in `EsmCellIndex`. SCOL expansion, PKIN expansion, and MSWP resolution are wired in the cell loader.
- **Interior render**: MedTekResearch01 10 809 entities @ 68.5 FPS / 14.61 ms — the "FO4 interior renders" milestone is cleared.
- **BGSM parser**: `crates/bgsm` implements BGSM v1–v22 / BGEM v1–v22 with template chain resolution, wired through `MaterialProvider`.

The three **new actionable gaps** surfaced by this audit:

1. **`"NiExtraData"` dispatch missing** (HIGH) — 100 FO4 FaceGen morph NIFs each drop one block. One-line fix in `blocks/mod.rs`.
2. **Renderer rejects 7 DXGI formats BA2 correctly extracts** (MEDIUM) — B8G8R8A8 (FO4 normal maps), R16/R8, BC4_SNORM, BC6H. Affects texture upload for those asset types.
3. **BGSM → `ImportedMesh` forwarding incomplete** (MEDIUM) — 4 texture slots and all PBR boolean flags (translucency, `pbr`, model-space normals) are parsed but never sent to the GPU.

The **one HIGH forward blocker** for future milestones: FO4 ships no `skeleton.nif`; humanoid NPC skeletal animation and equipped-armor rendering require a Havok `.hkx` parser or skeleton reconstruction — deferred to M41.x.

## Severity-Grouped Findings

### CRITICAL
_None._

### HIGH

**FO4-D5-001**: Stale per-block baseline fires false regression on 3 types now fixed — `crates/nif/tests/data/per_block_baselines/fallout_4.tsv` was captured at commit `a9c7bc9` (2026-04-24), before fixes #710, #720, #722 for `BSClothExtraData`, `BSEyeCenterExtraData`, `BSPositionData`. Running `per_block_baseline_fallout_4` panics as if those types regressed. Baseline must be regenerated and committed (`BYROREDUX_REGEN_BASELINES=1`). The underlying parsers are correct; only the golden file is stale.

**FO4-D5-002**: `"NiExtraData"` bare-base dispatch missing → 100 FO4 FaceGen morph NIFs each drop one block — `crates/nif/src/blocks/mod.rs` has no arm for the literal string `"NiExtraData"` (the abstract base class used directly as an RTTI class name in 100 FO4 facial morph NIFs). All 100 fall to `NiUnknown` and are classified as `Truncated`, pulling the clean-parse rate from 100% to 99.71%. nif.xml defines `NiExtraData` as a concrete block with only the inherited `Name` field; parsing it as a name-only block is correct. **Fix**: add `"NiExtraData" => Ok(Box::new(NiExtraData::parse(stream, "NiExtraData")?))` to the dispatch table.

**FO4-D6-008**: FO4 humanoid skeleton is `.hkx`-only; no `skeleton.nif` in any FO4 BA2 — `spawn_prebaked_npc_entity` cannot bind a skeleton and produces mesh-less NPC husks for all 23 FO4 humanoid NPCs in MedTekResearch01. A Havok `.hkx` parser or `_1stperson` skeleton reconstruction is required. Deferred to M41.x (Tier 5). Blocks "FO4 humanoid NPCs with equipped armor" milestone only; does not block current "FO4 interior renders" bar.

### MEDIUM

**FO4-D2-008**: Renderer `dds.rs:map_dxgi_format` rejects 7 DXGI formats that BA2 extracts correctly — `crates/renderer/src/vulkan/dds.rs` returns `"Unsupported DXGI format"` for: BC4_SNORM (81), BC6H_UF16 (95), BC6H_SF16 (96), B8G8R8A8_UNORM (87), B8G8R8A8_SRGB (91), R16_UNORM (56), R8_UNORM (61). B8G8R8A8 is explicitly noted in `ba2.rs:808` as the FO4 normal-map format. Missing mappings: `87 → vk::Format::B8G8R8A8_UNORM`, `91 → vk::Format::B8G8R8A8_SRGB`, `56 → vk::Format::R16_UNORM`, `61 → vk::Format::R8_UNORM`, `95 → vk::Format::BC6H_UFLOAT_BLOCK`, `96 → vk::Format::BC6H_SFLOAT_BLOCK`, `81 → vk::Format::BC4_SNORM_BLOCK`. Fix is in `crates/renderer/src/vulkan/dds.rs`, not `ba2.rs`.

**FO4-D5-003**: Per-block baseline: `NiPSysBlock` shrank 26 420→25 075; `NiTimeController` 124→0 — Both are improvements (new dedicated parsers for NiPSys force-field modifiers and BSLagBoneController/#837) masquerading as regressions against the stale baseline. No functional issue; regen resolves.

**FO4-D5-005**: `Fallout4 - MeshesExtra.ba2` not covered by any automated parse-rate test — DLC mesh overrides, settlement pieces, weapon mods, and power armor variants are in this second archive and are never tested. Model: add `parse_rate_fo4_all_meshes` parallel to the Starfield multi-archive gate.

**FO4-D6-002**: BGSM `specular_texture` / `lighting_texture` / `flow_texture` / `wrinkles_texture` parsed but not forwarded to `ImportedMesh` — `merge_bgsm_into_mesh` (in `byroredux/src/asset_provider.rs:640–723`) forwards 6 slots but omits 4. FO4 animated water surfaces (`flow_texture`), NPC wrinkle blending, and standalone specular maps render as absent.

**FO4-D6-003**: BGSM PBR boolean flags (`pbr`, `translucency`, model-space normals, glowmap, tessellate) parsed but not forwarded to renderer — The renderer's `GpuMaterial` has no `pbr` flag switch. FO4 `pbr=true` surfaces render through the legacy Gamebryo specular path. `translucency` suite (subsurface scattering on skin/vegetation/glass) also not wired.

**FO4-D6-004**: BGSM `smooth_spec_texture` per-texel roughness path incomplete — `merge_bgsm_into_mesh` writes `bgsm.smoothness` scalar into `mesh.glossiness`, but `specular_texture` (separate from `smooth_spec_texture`) is not mapped. FO4 PBR materials lack correct per-texel roughness/specular from BGSM texture slots.

**FO4-D6-012**: REFR `XMSP` sub-record not parsed onto `PlacedRef` → ~2 500 MSWP material-swap tables indexed but unused — `parse_refr_group` in `crates/plugin/src/esm/cell/walkers.rs` has no `b"XMSP"` arm. All vanilla Raider armor color variants, settlement clutter variations, and Vault decay overlays render with base-mesh textures.

**FO4-D4-014**: `parse_rate_fo4_esm` is `#[ignore]`-gated — CI doesn't run FO4 ESM per-category floor asserts on every push. Should be in the pre-release smoke-test checklist.

**FO4-D5-008**: `BSPositionData` + `BSEyeCenterExtraData` parsed but not imported — Dismemberment morphing and dialogue eye tracking absent. Parser correct; ECS bridge missing.

**FO4-D6-005**: BGSM `lum_emittance` / adaptive emissive suite (v>=12/13) parsed but not forwarded — Interior neon/fluorescent adaptive brightness absent.

**FO4-D6-006**: BGSM `distance_field_alpha_texture` (v>=17) not forwarded — FO4 foliage alpha-dithered distance fading absent.

**FO4-D6-010**: FaceGen NIF truncation (1 235 / 1 238 truncated FO4 NIFs) tracked but no dedicated issue filed — Should be formally tracked and linked to FO4-D5-002 resolution.

**FO4-D6-013**: `BSBoneLODExtraData` has no parser — FO4 NPCs skip dynamic bone LOD fade-out.
