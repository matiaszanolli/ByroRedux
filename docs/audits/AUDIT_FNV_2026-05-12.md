## Findings

### FNV-D1-NEW-01 (= Dim 5's FNV-D5-NEW-04): per-block FNV baseline stale since #707 (2026-05-01)

- **Severity**: LOW (hygiene; net parse fidelity unchanged)
- **Dimension**: NIF Parser — FNV Regression Guard (also surfaced by Dim 5)
- **Location**: `crates/nif/tests/data/per_block_baselines/fallout_nv.tsv:47` (committed 2026-04-24 in `a9c7bc9`)
- **Status**: NEW
- **Description**: The per-game block-histogram test `per_block_baseline_fallout_nv` fails on current `main`:
  ```
  PARSED shrank: NiPSysBlock 17933 -> 17887  (filter or dispatch loss?)
  ```
  Root cause confirmed by regenerating with `BYROREDUX_REGEN_BASELINES=1`. The diff:
  ```
  -NiPSysBlock                 17933   0
  +NiPSysBlock                 17887   0
  +NiPSysColorModifier            46   0
  ```
  Total preserved (17933 = 17887 + 46). Commit `ff23881` / #707 (2026-05-01) gave `NiPSysColorModifier` a dedicated parser so its authored colour curve could reach `ParticleEmitter`. The baseline TSV was never regenerated.
- **Evidence**:
  - `a9c7bc9` (2026-04-24) — "R3: capture per-block parsed/unknown baselines for all 7 games"
  - `ff23881` (2026-05-01) — "Fix #707: pipe NiPSysColorModifier authored colour curve through to ParticleEmitter"
  - Dispatch site: `blocks/mod.rs:859` → `"NiPSysColorModifier" => Ok(Box::new(particle::NiPSysColorModifier::parse(stream)?))`
  - Doc comment at `particle.rs:179-189` cites the redirect from the old `NiPSysBlock` catch-all.
- **Impact**: Latent. The CI guard intended to catch dispatch-redistribution regressions has been silently failing for ~11 days. Any real regression that landed in this window would have appeared on top of the existing failure and could have been dismissed as "the test is already failing." **Cross-game spillover** (Dim 5 finding): 5 sibling games also fail `per_block_baselines`:
  - **FO4 / FO76 / Skyrim SE/LE**: same dedicated-dispatch class (`NiTimeController N → 0`) — likely same #707-shaped redirect into dedicated parsers.
  - **Oblivion**: 8 UNKNOWN-grew rows (NiMaterialProperty 1→48, NiNode 5→72, NiSourceTexture 1→48, NiStringExtraData 4→65, NiTexturingProperty 1→48, NiTriStripsData 0→37, NiVertexColorProperty 0→1) — categorically different, may indicate a real parser regression.
  - **Starfield**: BSLightingShaderProperty 0→1036 UNKNOWN; parsed 189801→188765 — distinct category.
- **Suggested Fix**: 
  1. For FNV / FO4 / FO76 / Skyrim (dispatch redirect, benign): `BYROREDUX_REGEN_BASELINES=1 cargo test --release -p byroredux-nif --test per_block_baselines -- --ignored` and commit the resulting TSV diffs.
  2. For Oblivion and Starfield (UNKNOWN-grew patterns): **do not regen blindly** — investigate as separate findings to confirm whether the parser is genuinely losing coverage. Out of scope for FNV but flagged here for the broader NIF audit roadmap.
