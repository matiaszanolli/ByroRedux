## Dimension Findings

### Dimension 1 — FO3 Rendering Path (Inline Shaders) — HEALTHY

Re-verified directly against current code (`crates/nif/src/shader_flags.rs:534-645`):
`ShaderFlags::LegacyFo3Fnv { sf1, sf2 }` remains the FO3/FNV variant, distinct
from the FO4 u32-pair layout; its own unit test
(`shader_flags.rs` — asserts `matches!(flags, ShaderFlags::LegacyFo3Fnv { .. })`)
still passes. No commit since the 2026-07-02 audit touched
`crates/nif/src/import/material/`, `byroredux/src/material_translate.rs`, or
`crates/core/src/ecs/components/material.rs`, so the single-`translate_material`-
boundary, `EmissiveSource::Material` mapping, and Disney-BSDF gate (provably 0
for FO3 — no BGSM authored) all stand as previously verified.

**Existing: #1856** (OPEN, filed from the 2026-07-02 audit as FO3-D1-01) —
`WaterShaderProperty.water_shader_flags` (the FO3/FNV non-BS branch) still
dead-ends at `MaterialInfo` with no `ImportedMesh`/`Material` consumer. Confirmed
still present at `crates/nif/src/import/material/walker.rs:962-964` (unchanged
since last audit). Not re-reported.

### Dimension 6 — BSA v104 + Real-Data / DDS — HEALTHY

**Existing: #1542** (OPEN) — uncompressed 16/24-bpp DDS (FO3 font/HUD
placeholders) still rejected at `dds.rs:258-264`, confirmed unchanged. Not

New since 2026-07-02: `1748e148` (Fix #1819, "classify SpeedTree placeholder
billboard PBR at import time") directly hardens FO3's `.spt` foliage path —
`placeholder_billboard_mesh` (`crates/spt/src/import/mod.rs`) now sets explicit

FO3/FNV/Oblivion SpeedTree content that a prior audit (2026-06 series, filed as
part of the SpeedTree audit trail) surfaced. Confirmed fixed and tested
(`crates/spt/src/import/mod.rs` test module, `Some(0.0)`/`Some(0.85)` pinned).

### Dimension 7 — FO3 Animation / NPC Spawn + Scripting Gap — HEALTHY

resolution — confirmed not a regression.

B-spline `FLT_MAX` sentinel, `AnimationClipRegistry` case-insensitive dedup, and
