Clean parse rate is still 100 % across all four games — block_sizes recovery
keeps every NIF reaching the import pipeline. But the `NiUnknown` headcount
hides two distinct classes of defect:

1. **Undispatched block types** — real types that nif.xml defines and that the
   game authors actually shipped, but which the dispatch table routes straight
   to the fallback arm. `block_size` skips them, the original `type_name` is
   preserved, and downstream import code never sees the data.
2. **Parse failures on dispatched types** — the dispatch has an arm, the
   parser starts, then returns `Err` and `block_sizes` recovery swallows the
   error. Skyrim SE `bhkRigidBody` loses 12,866 blocks this way (~58 % of the
   NiUnknown pool on SE).

## Stream-drift evidence (Dim 3)

Every FO3 and FNV archive run emits the same recurring warning bucket:

```
[WARN byroredux_nif] Block N 'SkyShaderProperty': expected XX bytes, consumed 54. Adjusting position.
```

`SkyShaderProperty` (49 on FNV, ~similar on FO3) is aliased to
`BSShaderPPLightingProperty::parse` in `blocks/mod.rs:268-276`. The parse
under-consumes by 13-28 bytes per block — `block_sizes` recovery re-anchors
the outer stream, but the sky-specific scroll/tiling fields never reach the
struct. Dozens of sky NIFs silently lose their per-variant fields. See
NIF-05 below.

No other stream-drift warning bucket appears across the four archives.

## Findings

### CRITICAL


**NIF-01 — Skyrim SE `bhkRigidBody` parse fails on 12,866 blocks (58 % of SE NiUnknown pool)**

- **Severity:** CRITICAL
- **Dimension:** Dim 1 (Block Parsing) × Dim 5 (Coverage Gaps)
- **Game Affected:** Skyrim SE (bsver 100)
- **Location:** `crates/nif/src/blocks/collision.rs:210-` (`BhkRigidBody::parse`),
  recovery at `crates/nif/src/lib.rs:293-316`
- **Problem:** The dispatch arm at `blocks/mod.rs:628` routes both `bhkRigidBody`
  and `bhkRigidBodyT` through `BhkRigidBody::parse`. On Skyrim SE (bsver 100)
  the parser returns `Err`, the outer loop logs a warning, seeks past the
  block via `block_size`, and inserts a `NiUnknown`. Every Havok rigid body
  in `Skyrim - Meshes0.bsa` is lost — mass, inertia, collision filter,
  constraint refs all gone. Physics, ragdoll, and havok-driven animation on
  Skyrim scenes all downstream-degrade.
- **Evidence:** `/tmp/audit/nif/skyrimse_unk.out` — 9,772 `bhkRigidBody` +
  3,094 `bhkRigidBodyT` in the NiUnknown bucket. FNV/FO3 parse these cleanly
  (not in their unknown histograms); Oblivion has 6.
- **Suggested fix:** Bisect `BhkRigidBody::parse` with a SE-only test
  fixture. The three `bsver <= 34` / `bsver >= 83` / `bsver < 130` gates at
  lines 223, 243, 305 all hit the Skyrim path; the cInfo size on SE is
  specifically 144 bytes (bsver 100) vs 152 on FO4 (bsver 130) per nif.xml
  `bhkRigidBodyCInfo`. Likely a gated field that Skyrim has but the current
  `bsver >= 83` arm over- or under-consumes.


**NIF-02 — `NiAdditionalGeometryData` missing from dispatch (4,039 FO3+FNV blocks)**

- **Severity:** CRITICAL
- **Dimension:** Dim 5 (Coverage Gaps) × Dim 4 (Import Pipeline)
- **Game Affected:** FO3, FNV
- **Location:** `crates/nif/src/blocks/mod.rs` (no dispatch arm);
  `crates/nif/src/blocks/tri_shape.rs` (no parser)
- **Problem:** `NiAdditionalGeometryData` carries per-vertex auxiliary
  channels that nif.xml defines as tangents, bitangents, blend weights, and
  optional skin-bone IDs — i.e. the channels the renderer needs for
  normal-mapped specular on FO3/FNV architecture. 4,039 blocks in the two
  archives fall into `NiUnknown` and the geometry side of the pipeline
  sees zero additional data.
- **Evidence:** `/tmp/audit/nif/fnv_unk.out:1` (2,308), `/tmp/audit/nif/fo3_unk.out:1`
  (1,731). Zero on Skyrim (BSTriShape replaced it) and Oblivion (predates it).
- **Suggested fix:** Add `NiAdditionalGeometryData` parser (nif.xml line
  4721). Structure is { num_vertices, num_block_infos, block_infos[],
  blocks[] }. Wire the tangent/bitangent channels into the same `MeshHandle`
  vertex-attribute slot used by BSTriShape today so the shader sees a
  unified tangent source. ~80 LOC parser + dispatch arm.


**NIF-03 — `NiBoolTimelineInterpolator` missing from dispatch (8,450 blocks across 3 games)**

- **Severity:** CRITICAL
- **Dimension:** Dim 5 (Coverage Gaps)
- **Game Affected:** FO3, FNV, Skyrim SE
- **Location:** `crates/nif/src/blocks/mod.rs:505` (has `NiBoolInterpolator`
  but not the Timeline variant); `crates/nif/src/blocks/interpolator.rs`
- **Problem:** `NiBoolTimelineInterpolator` (nif.xml: subclass of
  `NiBoolInterpolator` with an extra `Timeline: TimeBool100` field) drives
  binary visibility/enabled animations on a huge fraction of Skyrim SE
  content — 6,796 blocks, second-most-common unknown after `bhkRigidBody`.
  Every one silently animates nothing. Renderer sees the affected objects
  as permanently visible (or permanently hidden, depending on default).
- **Evidence:** SE 6,796 + FNV 1,118 + FO3 536 from the three unknown
  sweeps.
- **Suggested fix:** Thin extension of `NiBoolInterpolator::parse` with the
  extra `TimeBool100` byte. Dispatch arm next to the existing
  `NiBoolInterpolator =>` at `blocks/mod.rs:505`. ~20 LOC.

**NIF-05 — `SkyShaderProperty` alias under-consumes 13-28 bytes per block (48-49 FNV + 49 FO3 warnings)**

- **Severity:** HIGH
- **Dimension:** Dim 3 (Stream Position) × Dim 2 (Version Handling)
- **Game Affected:** FO3, FNV (also Oblivion sky meshes)
- **Location:** `crates/nif/src/blocks/mod.rs:268-276` (alias block for
  `"BSShaderPPLightingProperty" | "SkyShaderProperty" | ...`)
- **Problem:** `SkyShaderProperty` inherits `BSShaderLightingProperty` +
  adds 4 scroll vectors (per nif.xml line 6284 — `Sky Scroll X`, `Sky
  Scroll Y`, `Sky Distance`, `Sky Z Distance` plus optional `Flag`).
  Aliasing to `BSShaderPPLightingProperty::parse` consumes only the
  texture_set_ref + refraction + parallax fields, missing the
  sky-specific tail. `block_sizes` keeps the outer stream aligned so the
  bug is silent at parse time. Sky NIFs render with default scroll and
  distance — static clouds, wrong horizon fade.
- **Evidence:** Recurring warning bucket in `/tmp/audit/nif/fnv_err.log:21-49`
  and `/tmp/audit/nif/fo3_err.log:16-36` — 6+ distinct block offsets per
  run, all `SkyShaderProperty` with `consumed 54, expected 42-82`.
- **Suggested fix:** Split `SkyShaderProperty` out of the alias block
  (same structural move as `TileShaderProperty` got in #455 and
  `WaterShaderProperty`/`TallGrassShaderProperty` in #474). Parser reads
  PPLighting base + 4 f32 + optional Flag. ~60 LOC with unit test.


**NIF-06 — `bhkBlendController` missing from dispatch (1,427 FNV+FO3 blocks)**

- **Severity:** HIGH
- **Dimension:** Dim 5 (Coverage Gaps)
- **Game Affected:** FO3, FNV
- **Location:** `crates/nif/src/blocks/mod.rs` (no dispatch arm)
- **Problem:** `bhkBlendController` drives blend weights between multiple
  Havok animations (nif.xml: `NiTimeController` subclass with a single
  `Keys: NiFloatInterpolator` ref). Affects ragdoll blending and
  animation layering on FO3/FNV NPCs.
- **Evidence:** FNV 845, FO3 582.
- **Suggested fix:** Thin wrapper around `NiSingleInterpController::parse`
  (same shape) + dispatch arm. ~15 LOC.


**NIF-07 — `BSNiAlphaPropertyTestRefController` missing from dispatch (751 SE blocks)**

- **Severity:** HIGH
- **Dimension:** Dim 5 (Coverage Gaps)
- **Game Affected:** Skyrim SE
- **Location:** `crates/nif/src/blocks/mod.rs`
- **Problem:** Animates the alpha-test threshold on `NiAlphaProperty`
  (dissolve effects, fade transitions, ghost-reveal VFX). 751 blocks on
  Skyrim SE silently hold a fixed threshold.
- **Evidence:** `/tmp/audit/nif/skyrimse_unk.out`: 751.

**NIF-08 — `NiFloatExtraData` + `NiFloatExtraDataController` missing (SE: 1,492, cross-game: 1,657)**

- **Severity:** HIGH
- **Dimension:** Dim 5 (Coverage Gaps)
- **Game Affected:** FO3, FNV, Skyrim SE
- **Location:** `crates/nif/src/blocks/mod.rs:326-332` (NiExtraData arm
  covers String/Binary/Integer variants, not Float)
- **Problem:** `NiFloatExtraData` is a generic float-valued metadata tag,
  widely used for tool-authored engine hooks (FOV multipliers, scale
  overrides, wetness levels). The controller subtype animates it.
  1,312 + 180 blocks on SE; also present on FNV/FO3.
- **Evidence:** All three unknown sweeps.
- **Suggested fix:** Extend the existing `NiExtraData::parse` enum-style
  multi-type match to accept `"NiFloatExtraData"` (it's identical in shape
  to `NiIntegerExtraData` but with `f32` payload). ~5 LOC + test.

**NIF-09 — Oblivion NiUnknown pool is 429 blocks spread across 32 already-dispatched types**

- **Severity:** MEDIUM
- **Dimension:** Dim 1 (Block Parsing) × Dim 3 (Stream Position)
- **Game Affected:** Oblivion
- **Location:** `crates/nif/src/lib.rs:332-376` (runtime-size-cache +
  oblivion_skip_sizes recovery)
- **Problem:** Every entry in `/tmp/audit/nif/obl_unk.out` maps to a type
  that IS in the dispatch table (NiNode, NiTriShape, NiTransformData,
  NiPSysSpawnModifier, etc.). Parse fails → runtime-size-cache skips the
  block → NiUnknown. Oblivion has no `block_sizes`, so these only
  recover if a sibling NIF successfully parsed the same type first. The
  per-type counts are tiny (≤ 49) so the real-world impact is bounded,
  but it means Oblivion-specific layout quirks exist across 32 types.
- **Evidence:** `/tmp/audit/nif/obl_unk.out` — 32 distinct type names,
  top bucket is NiNode at 49.
- **Suggested fix:** Deferred until Oblivion coverage rework is in scope.
  Capture the 429 failing instances, group by type, diff against
  successful parses to isolate the Oblivion-specific tail.
