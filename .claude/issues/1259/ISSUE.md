# PERF-D3-NEW-04: Blend-pipeline cache pre-population walks every batch every frame with no steady-state short-circuit

**Labels**: bug, renderer, medium, performance

## Severity
**MEDIUM**

## Status
**NEW**

## Location
- `crates/renderer/src/vulkan/context/draw.rs:1721-1737`

## Description
The pre-loop at `draw.rs:1721` iterates every batch and for each `PipelineKey::Blended { src, dst, wireframe }` does a `HashMap::contains_key((src, dst, wireframe))`. After the first few cell-load frames every (src, dst, wireframe) combo is cached and `contains_key` always returns `true`. On Riverwood with M blended batches per frame (estimated 300-500 — Skyrim hair, leaves, banners, smoke all blend), that's M hash lookups every frame producing zero work in steady state.

Cheap individually, but visible in CPU-bound regimes where M is large.

## Evidence
```rust
// draw.rs:1721-1737
for batch in &batches {
    if let PipelineKey::Blended { src, dst, wireframe } = batch.pipeline_key {
        // Normalize cache key against the device-cap gate so a
        // disabled-wireframe device hits the same slot it would
        // for a regular opaque blend. Matches the gate in
        // `get_or_create_blend_pipeline`. #869.
        let wireframe = wireframe && self.device_caps.fill_mode_non_solid_supported;
        if !self.blend_pipeline_cache.contains_key(&(src, dst, wireframe)) {
            if let Err(e) = self.get_or_create_blend_pipeline(src, dst, wireframe) {
                log::error!(
                    "Failed to create blend pipeline (src={src}, dst={dst}): {e}; \
                     draws using this combo will fall back to opaque pipeline"
                );
            }
        }
    }
}
```

The loop runs unconditionally regardless of whether new (src, dst) combos have appeared this frame.

## Trigger Conditions
Any frame with one or more `PipelineKey::Blended` batches — i.e. every Skyrim / FNV / FO3 frame with any alpha-blended content (hair, foliage, banners, smoke, particles).

## Impact
At Riverwood scale (estimated 300-500 blended batches given Skyrim's heavy use of NiAlphaProperty), saves perhaps 50-100 µs/frame. Not the headline win, but free.

## Suggested Fix
Two options:

**Option (a) — minimal change**: track the live (src, dst, wireframe) triples seen in `build_render_data` (or in `collect_static_mesh_draws`) into a small `HashSet` field on `VulkanContext`. Only run the pre-population walk when the seen-set differs from the cached-set. Reset the seen-set comparison after every cell load. ~30 LOC.

**Option (b) — structural**: hoist the pre-population to `cell_loader::spawn` — every blend combo is determined by NIF/BGSM properties known at import time, so the pipeline can be created at cell-load instead of per-frame. Cleaner but touches more files.
