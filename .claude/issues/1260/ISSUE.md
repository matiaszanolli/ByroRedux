# PERF-D3-NEW-05: Off-frustum draws pay full per-command build cost before being skipped at in_raster gate

**Labels**: bug, renderer, low, performance

## Severity
**LOW**

## Status
**NEW**

## Location
- `crates/renderer/src/vulkan/context/draw.rs:1425-1538`

## Description
Every `DrawCommand` — including `in_raster=false` (frustum-culled) and `is_water=true` — runs through the full per-command processing in the build loop:

- mesh registry lookup (line 1426)
- model matrix non-uniform-scale detection: 3 dot products + 2 abs subtracts (line 1438-1444)
- flag assembly (lines 1466-1494)
- GpuInstance struct construction + push to vec (lines 1501-1520)
- is_caustic_source check (line 1474)

Only after all that does the `if !draw_cmd.in_raster || draw_cmd.is_water { continue; }` at line 1536 skip batch formation. For frustum-culled draws this work IS required by the #516 RT-only contract (the GpuInstance entry must exist for TLAS-hit shaders) — that's fine. But the order matters: the non-uniform-scale check + flag bits could be skipped, since the off-frustum entry's flags are never read by the rasterizer (only the model matrix is needed by the RT path for TLAS instance positioning).

At Riverwood scale, the sort key places `rt_only=1` draws at the END (per `render/mod.rs:165-178`), but the build loop walks the **sorted** order, so the off-frustum tail is the last few thousand iterations. Roughly: if the frustum kept 5000 of 12,277 draws in-raster, the other ~7,000 pay the full build cost.

## Evidence
```rust
// draw.rs:1425-1538 — full build per command, skip only at line 1536
for draw_cmd in draw_commands {
    let Some(mesh) = self.mesh_registry.get(draw_cmd.mesh_handle) else { continue; };
    // ... model matrix dot products, flag assembly, is_caustic_source ...
    gpu_instances.push(GpuInstance { /* full payload */ });
    if !draw_cmd.in_raster || draw_cmd.is_water {
        continue;  // <-- skipped here, but build cost paid
    }
    // batch formation below
}
```

## Trigger Conditions
Any frame with non-zero frustum-culled draw count. Scales linearly with off-screen radius — radius-5 exterior grids will see substantially more.

## Impact
Saves the per-command CPU cost on (total_commands − in_raster_count) draws per frame. At ~50-100 ns per skipped non-uniform-scale check + flag assembly, on ~7,000 off-frustum draws, ~350-700 µs/frame at Riverwood scale. Small absolute win, but it scales with off-screen radius — for radius-5 grids it becomes more visible.

## Suggested Fix
Two options:

**Option (a) — minimal change**: split the loop into two phases: (1) construct GpuInstance for every draw (the #516 SSBO requirement) but with a minimal model-matrix-only payload for `in_raster=false`; (2) only run flag assembly + non-uniform-scale check for in-raster draws. ~20 LOC.

**Option (b) — sort-aware bailout**: since the `rt_only` draws cluster at the end of the sorted list, the loop can bail out of the full-build path once it crosses the `rt_only=1` boundary and switch to a thin GpuInstance-only path. Slightly fewer branches per iteration; same algorithmic shape.
