# SUN-N4: Sun disc renders without elevation gate — visible below horizon at sunset/sunrise

Labels: bug renderer medium 
State: OPEN

## Severity
**MEDIUM** — Visual artifact when looking at a flat horizon at sunrise/sunset with no terrain occlusion.

## Status
**NEW**

## Location
`crates/renderer/shaders/composite.frag:184-218`

## Description
The sun disc draws when `cos_angle > sun_size - 0.002`. There's no check that `dir.y > 0` (elevation positive) or that `sun_direction.y > 0` (sun above horizon). At sunset / sunrise / through the night, when the sun is below horizon (`sun_direction.y < 0`), the disc still draws additively over below-horizon sky pixels.

Below-horizon sky pixels go through the `elevation < 0` branch (line 107) which mixes `horizon → sky_lower`. The disc add at line 217 then layers on top, painting a visible sun on the "ground tint" region.

The visual effect requires (a) flat horizon view (no terrain occlusion at low elevation), and (b) `sun_intensity > 0` (otherwise disc=0). The sunrise/sunset window has both: terrain may be 1000+ units away with nothing covering the immediate horizon, and `sun_intensity` is ramping but non-zero.

## Evidence
```glsl
// composite.frag:182-218 — disc drawn unconditionally when cos_angle is high
float cos_angle = dot(dir, sun_direction);
float sun_edge_start = sun_size - 0.002;
if (cos_angle > sun_edge_start) {
    // ... compute disc, sample sprite ...
    sky += disc_color * sun_intensity * disc;   // ← no `dir.y > 0` gate
}
```

## Trigger Conditions
- Exterior cell with `is_exterior > 0.5 && depth >= 0.9999` (sky branch active)
- `sun_direction.y < 0` AND `sun_intensity > 0` (sunrise/sunset transition)
- View direction has `dir.y < 0` AND `cos_angle > sun_size - 0.002` (looking down at where the sun "is")
- No depth-occluding terrain in the look-down path

## Impact
Visual artifact when looking at a flat horizon at sunrise or sunset with no terrain occlusion. Real exterior content (Mojave's open wasteland, Skyrim's tundra) has enough mountain/cliff geometry to occlude the horizon line — the sky branch only renders when `depth >= 0.9999`, so terrain depth-tests out the disc. The bug is visible primarily on:
- Open flat exteriors with no nearby terrain
- Indoor exterior cells (DLC interior with sky visible through skylights)
- Camera elevated above mountains looking at sunset

## Suggested Fix
Two options:
1. Gate the disc add on `dir.y > 0`:
   ```glsl
   if (cos_angle > sun_edge_start && dir.y > 0.0) {
       // ...
       sky += disc_color * sun_intensity * disc;
   }
   ```
   Cleanest; matches the cloud-layer gates at lines 130, 147, 158, 169.

2. Multiply `disc` by `smoothstep(0, 0.05, dir.y)` for a soft horizon fade, mirroring the cloud `horizon_fade`.

Option 1 is simpler and matches existing cloud gating.
