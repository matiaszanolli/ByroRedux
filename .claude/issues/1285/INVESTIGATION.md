# Investigation — #1285 MAT_FLAG_* bits 5-9 bypass generated-header lockstep

## Premise — CONFIRMED
`MAT_FLAG_*` bits 5-9 (Disney BSDF / SSS / model-space-normals: PBR_BSDF,
TRANSLUCENCY, MODEL_SPACE_NORMALS, TRANSLUCENCY_THICK_OBJECT,
TRANSLUCENCY_MIX_ALBEDO) lived in two unsynchronized places:
- Rust: `vulkan/material.rs::material_flag::*` (literals, authoritative).
- Shader: hand-written `#define`s in `triangle.frag` (were at :207-211).

No lockstep test tied them. Bits 0-4 already flow through the generated header
(`shader_constants_data.rs` → build.rs → `shader_constants.glsl` → `triangle.frag`
`#include`) and are pinned by `generated_header_contains_all_defines` +
`material_flag_bits_match_material_consts` (#1190). triangle.frag:9 already
`#include`s the generated header.

## Design choice — matched the bits-0-4 pattern (NOT the issue's alias variant)
The issue suggested making `material.rs` bits 5-9 *aliases* of `shader_constants_data`.
I instead matched the **established, tested bits-0-4 model**: a mirror literal in
`shader_constants_data.rs` pinned equal to the `material.rs` authoritative literal
by `material_flag_bits_match_material_consts`. Reasons:
- Consistency — bits 0-4 and 5-9 now use one idiom in the same module.
- The pin test stays *meaningful* (catches literal drift); aliases would make it
  trivially-true (`assert_eq!(X, X)`).
- Smaller, safer diff — `material.rs` needs no change (the literals were already there).
- Achieves the issue's actual goal/risk-closure identically: bits 5-9 now flow
  through the generated header and are pinned by both lockstep tests.
- Path note: the issue's suggested `crate::shader_constants_data::` path is wrong
  anyway — `shader_constants_data.rs` is `include!`'d into `crate::shader_constants`,
  not its own module.

## Changes (5 files)
1. `shader_constants_data.rs` — added `MAT_FLAG_PBR_BSDF` (1<<5) … `_TRANSLUCENCY_MIX_ALBEDO`
   (1<<9) mirrors; refreshed the stale "add once Phase 2b lands" comment.
2. `build.rs` — emit the 5 new `#define`s into `shader_constants.glsl`.
3. `shader_constants.rs` — extended `generated_header_contains_all_defines` (+5
   table rows) and `material_flag_bits_match_material_consts` (+5 asserts).
4. `triangle.frag` — removed the hand-written `#define`s; they now come from the
   `#include`. Replaced with a comment forbidding re-adding them.
5. `shader_constants.glsl` — regenerated by build.rs (tracked file; +5 lines).

## Verification
